## 0. 들어가며 (Motivation)

지금까지 우리는 Transformer 를 **밑에서 위로** 쌓아 왔습니다 — [[Day-032]] 어텐션, [[Day-033]] self-attention·인코더 블록, [[Day-034]] 마스크·인코더/디코더·LN. 부품을 다 만졌으니, 이제 그 부품들이 **처음 제안된 그 논문** 을 직접 펼칠 차례입니다.

이 논문(2017, NeurIPS)은 딥러닝 역사에서 손꼽히는 전환점입니다. 제목 그대로 *"주의(attention)만 있으면 된다"* — 그때까지 시퀀스 모델의 표준이던 **순환(RNN/LSTM, [[Day-030]]·[[Day-031]])과 합성곱(CNN)을 전부 버리고**, 어텐션만으로 최고 성능의 기계번역기를 만들 수 있음을 보였습니다. BERT·GPT·T5, 그리고 오늘날의 모든 LLM 이 이 8쪽짜리 논문의 후손입니다.

그런데 "유명한 논문"을 그냥 처음부터 끝까지 읽으면 길을 잃기 쉽습니다. 그래서 [[Day-012]] 에서 배운 **3-pass 독해법** 을 씁니다 — (1) 제목·초록·결론·그림만 훑어 **큰 그림**, (2) 본문을 읽되 증명은 건너뛰고 **핵심 메커니즘**, (3) 필요한 부분을 **직접 재현**. 오늘은 pass 1~2 를 충실히 하고, pass 3 의 맛보기로 논문의 두 설계 결정을 코드로 재현합니다.

> We built the Transformer bottom-up over Day-032~034; today we open the paper that introduced it — *Attention Is All You Need* (2017). It threw out recurrence and convolution and showed attention alone beats them at translation. We read it with the 3-pass method ([[Day-012]]) and partially reproduce two of its design choices (sinusoidal positional encoding, the 1/√d_k scaling) to see *why* — the "understand + partial reproduction" depth of the paper track.

### 1.2 Pass 1 — 문제와 기여 (초록·서론·결론만)

**문제.** RNN/LSTM([[Day-030]]·[[Day-031]])은 시퀀스를 **왼쪽부터 한 스텝씩** 처리합니다(`h_t` 는 `h_{t-1}` 에 의존). 이 순차성은 두 가지를 막습니다: ① **병렬화 불가** — 문장 길이만큼 순차 계산이라 GPU 를 못 채웁니다. ② **장거리 의존성** — 멀리 떨어진 두 단어를 잇는 경로가 길어 신호가 흐려집니다([[Day-031]] 의 기울기 소실 문제).

**기여(초록의 주장).** 논문은 순환과 합성곱을 **완전히 제거** 하고 어텐션만으로 만든 **Transformer** 를 제안합니다. 결과(§6, pass 1 에서 그림·표만 봐도 드러남): WMT 2014 영↔독 번역에서 **28.4 BLEU**(당시 최고, 앙상블 포함 기존 기록 경신), 영→불 **41.8 BLEU** 를 **8×P100 GPU 로 3.5일** 이라는, 기존 대비 훨씬 적은 학습량으로 달성. "적은 비용으로 더 좋게"가 핵심 메시지입니다.

> [!tip] Pass 1 의 5질문([[Day-012]])에 답해보기
> - **무슨 문제?** 시퀀스 모델의 순차성(병렬화·장거리 의존성 한계).
> - **핵심 아이디어?** 순환/합성곱을 버리고 self-attention 만으로.
> - **어떻게 검증?** WMT'14 번역 BLEU + 학습 비용(FLOPs).
> - **결과?** SOTA BLEU 를 훨씬 적은 비용으로.
> - **한계·다음?** (pass 2 에서) — 이 논문 자체는 번역에 한정, 이후 BERT/GPT 로 확장.

### 1.3 Pass 2 — 핵심 메커니즘 (§3, 우리가 만진 것과 매핑)

이제 §3 을 읽되, 증명·하이퍼파라미터 세부는 건너뛰고 **메커니즘** 에 집중합니다. 반가운 건, 우리가 이미 코드로 만든 것들이라는 점입니다.

**(a) §3.2.1 Scaled Dot-Product Attention.** 논문의 식 (1):

$$
\text{Attention}(Q,K,V) = \operatorname{softmax}\!\left(\frac{QK^\top}{\sqrt{d_k}}\right)V
$$

[[Day-032]] 에서 만진 바로 그 식입니다. 핵심은 **`1/√d_k`** — 논문 각주 4 는 *"d_k 가 크면 내적이 크기가 커져 softmax 를 기울기가 극히 작은 영역으로 밀어 넣는다"* 고 설명합니다. 그래서 스케일링합니다(→ 4부 실습에서 숫자로 확인).

**(b) §3.2.2 Multi-Head Attention.** 하나의 큰 어텐션 대신, `d_model` 을 `h` 개로 쪼개 **여러 부분공간에서 병렬로** 어텐션한 뒤 이어 붙입니다([[Day-033]] 의 멀티헤드). base 모델은 `h=8`, `d_k=d_v=d_model/h=64`.

**(c) §3.3 Position-wise FFN.** 각 위치에 **독립적으로** 적용되는 2층 MLP: `FFN(x)=max(0, xW₁+b₁)W₂+b₂`, 내부 차원 `d_ff=2048`([[Day-033]] 의 FFN).

**(d) §3.5 Positional Encoding.** self-attention 은 순서를 모릅니다(집합처럼 봄). 그래서 위치 정보를 **더해** 줍니다. 논문은 **학습 없는 사인/코사인** 을 씁니다:

$$
PE_{(pos,2i)}=\sin\!\Big(\frac{pos}{10000^{2i/d_{model}}}\Big),\quad
PE_{(pos,2i+1)}=\cos\!\Big(\frac{pos}{10000^{2i/d_{model}}}\Big)
$$

차원 `i` 마다 다른 파장의 파동을 겹쳐, 각 위치에 고유한 "지문"을 부여합니다(→ 4부 재현).

> [!note] Pass 2 에서 챙길 §4 — "왜 self-attention 인가"
> 논문 §4 는 self-attention·순환·합성곱을 **세 기준** 으로 비교합니다: ① 층당 **계산 복잡도**, ② **병렬화 가능한 연산량**(순차 연산 수), ③ **장거리 의존성 경로 길이**. self-attention 은 두 위치를 잇는 경로가 **O(1)**(순환은 O(n))이라 장거리 학습이 쉽고, 순차 연산이 **O(1)** 이라 병렬화가 완벽합니다. 대가는 층당 복잡도가 시퀀스 길이에 **O(n²)** — 오늘날 "긴 문맥" 문제의 뿌리입니다.

### 1.4 실험·결과 (§6, Table 2·3)

- **Table 2 (번역 성능·비용):** Transformer(big)가 영→독 **28.4 BLEU**, 영→불 **41.8 BLEU** 로 기존 단일·앙상블 모델을 능가하면서 학습 FLOPs 는 훨씬 적음.
- **Table 3 (ablation, 논문에서 가장 배울 게 많은 표):** 설계 결정들을 하나씩 흔들어 본 표입니다 — 헤드 수 `h`(너무 적거나 많으면 나빠짐), `d_k` 축소(품질 하락), 모델 크기·드롭아웃, 그리고 **학습된 위치 임베딩 vs sinusoidal(성능 거의 동일 → 그래서 외삽이 되는 sinusoidal 채택)**. ablation 을 읽는 눈이 곧 논문을 비판적으로 읽는 힘입니다([[Day-012]]).

> [!warning] 논문을 읽을 때 흔한 함정
> "BLEU 숫자가 높다"만 보고 넘어가면 절반만 읽은 것입니다. 진짜 통찰은 **Table 3(ablation)** 과 **§4(왜 self-attention)** 에 있습니다 — *어떤 결정이 성능에 얼마나 기여했는가*. 결과 표(무엇을 달성)보다 ablation·분석(왜 그렇게 설계)에서 배울 게 많습니다.

### 4.2 1부 — Sinusoidal Positional Encoding 재현 (§3.5 식 그대로)

논문 §3.5 의 식을 **그대로** 옮깁니다:

```python
def positional_encoding(max_len, d_model):
    pos = np.arange(max_len)[:, None]
    i = np.arange(d_model)[None, :]
    div = np.power(10000.0, (2 * (i // 2)) / d_model)   # 차원별 파장
    angle = pos / div
    pe = np.zeros((max_len, d_model))
    pe[:, 0::2] = np.sin(angle[:, 0::2])                # 짝수=sin
    pe[:, 1::2] = np.cos(angle[:, 1::2])                # 홀수=cos
    return pe
```

출력(재현 가능, d_model=16):

```
  PE shape = (50, 16)  (max_len=50, d_model=16)
  값 범위: min=-1.000, max=1.000  → [-1, 1] 유계 확인
  pos=0 의 인코딩(앞 8차원) = [0. 1. 0. 1. 0. 1. 0. 1.]

  [상대 위치 성질] pos=10 과 pos=10+k 의 정규화 내적:
    k= 0: cos유사도 = +1.000
    k= 1: cos유사도 = +0.936
    k= 2: cos유사도 = +0.796
    k= 5: cos유사도 = +0.767
    k=10: cos유사도 = +0.456
    k=20: cos유사도 = +0.722
```

두 가지가 논문의 주장을 뒷받침합니다. ① **유계** — 모든 값이 `[-1,1]`(sin·cos 이니까)이라, 임베딩에 더해도 폭주하지 않습니다. ② **거리 정보** — 가까운 위치일수록 인코딩이 비슷하고(k 작을수록 유사도↑), 멀어질수록 감소합니다. 다만 **완벽히 단조롭진 않습니다**(k=20 이 k=10 보다 되레 높음) — 여러 주파수의 파동이 겹친 결과이고, `d_model` 이 작을수록 두드러집니다. 논문이 sinusoidal 을 택한 이유는 Table 3 에서 학습된 위치 임베딩과 **성능이 거의 같으면서**, 학습 때 못 본 **더 긴 길이로 외삽(extrapolate)** 이 가능하다고 기대했기 때문입니다.

### 4.3 2부 — 왜 `1/√d_k` 인가 (§3.2.1 각주 재현)

논문 각주의 주장: *q, k 성분이 평균0·분산1 이면 내적 `q·k = Σqᵢkᵢ` 의 분산은 `d_k`* 입니다. 큰 `d_k` 에서 점수가 커지면 softmax 가 포화됩니다. 무작위 벡터로 실측합니다:

```python
for d_k in [8, 64, 512]:
    q = np.random.randn(20000, d_k); k = np.random.randn(20000, d_k)
    dot = np.sum(q * k, axis=1)                 # 내적
    scaled = dot / np.sqrt(d_k)                 # 스케일링 후
```

출력:

```
  d_k=  8:  내적 분산=    7.9 (이론값 8)   |   스케일 후 분산=0.983 (~=1)
  d_k= 64:  내적 분산=   63.7 (이론값 64)   |   스케일 후 분산=0.996 (~=1)
  d_k=512:  내적 분산=  515.0 (이론값 512)  |   스케일 후 분산=1.006 (~=1)

  [softmax 포화] 점수 벡터 = 표준정규 * sqrt(d_k), d_k=64:
    스케일 없음: max prob = 0.678  (1.0 에 가까움 = 포화)
    스케일 있음: max prob = 0.076  (완만한 분포 = 기울기 살아있음)
```

**논문의 각주가 숫자로 확인됩니다.** ① 내적 분산은 정확히 `d_k`(8→7.9, 64→63.7, 512→515) — `1/√d_k` 로 나누면 분산이 **1 로 복원** 됩니다. ② 스케일이 없으면 점수가 커져 softmax 최대확률이 한쪽으로 쏠리고(포화), 스케일이 있으면 분포가 완만해 [[Day-027]] 의 **기울기가 살아 있습니다**. "왜 √d_k 로 나누나?"에 대한 논문 한 줄이, 두 줄짜리 실험으로 손에 잡힙니다.

### 4.4 3부 — base 모델 설정 재현 (Table 3)

```
  d_model=512, heads h=8, d_ff=2048, N(층)=6
  → d_k = d_v = d_model/h = 64
  MHA 투영 파라미터/층 = 4·d_model^2 = 1,048,576
  FFN 파라미터/층      = 2·d_model·d_ff = 2,097,152
  인코더 한 층 합(대략) = 3,145,728
```

논문의 base 설정(`d_model=512, h=8, d_ff=2048, N=6`)을 그대로 넣어, 우리가 [[Day-033]]·[[Day-034]] 에서 만진 부품들의 **규모** 를 확인합니다. 한 층에서 어텐션 투영(`4·d_model²`)보다 FFN(`2·d_model·d_ff`)이 파라미터를 더 먹는다는 점도 눈에 들어옵니다.

### 4.5 (실험) 직접 흔들어 보기

- **PE 파장 바꾸기**: `10000` 을 `100` 으로 줄이면 파장이 짧아져 인접 위치 구분이 더 촘촘해집니다 — 1부 유사도 곡선이 어떻게 변하는지 보세요.
- **스케일 지우기**: 2부에서 `scaled = dot`(나눗셈 제거)로 두면 `d_k=512` 에서 softmax 가 얼마나 극단적으로 포화되는지 관찰하세요.

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **왜 순환을 버렸나.** 논문 §4 의 세 기준(계산 복잡도·병렬 가능 연산·경로 길이)으로, self-attention 이 RNN([[Day-030]])보다 나은 점과 **대가(O(n²))** 를 각각 설명해 보세요.
> 2. **√d_k 의 필요성.** 4.3 실측에서 스케일을 없애면 softmax 가 포화됩니다. 이것이 [[Day-027]] 의 기울기 소실과 어떻게 연결되나요? 왜 하필 `√d_k`(분산이 아니라 표준편차)로 나눌까요?
> 3. **sinusoidal vs 학습된 임베딩.** Table 3 에서 둘의 성능이 거의 같았는데, 논문은 왜 sinusoidal 을 택했을까요? 4.2 에서 본 "유계·거리 인코딩" 성질과 '외삽' 개념을 엮어 답해 보세요.

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **Vaswani et al. (2017) *Attention Is All You Need* 는 순환·합성곱을 완전히 버리고 어텐션만으로 시퀀스 변환기(Transformer)를 만들어, WMT'14 번역에서 SOTA BLEU(영→독 28.4)를 훨씬 적은 학습 비용으로 달성한 전환점 논문이다.** [[Day-012]] 의 3-pass 로 읽으면 — pass 1: 문제(RNN 순차성)·기여(어텐션만으로)·결과(BLEU·비용); pass 2: §3.2.1 `softmax(QKᵀ/√d_k)V`([[Day-032]]), §3.2.2 멀티헤드(h=8,d_k=64, [[Day-033]]), §3.5 sinusoidal PE, §4 self-attention 이 경로길이 O(1)·병렬 O(1)이나 복잡도 O(n²); Table 3 ablation 이 진짜 통찰. 부분 재현으로 논문의 두 설계 결정을 숫자로 확인했다 — ① sinusoidal PE 는 유계(`[-1,1]`)이며 거리 정보를 담고(완전 단조는 아님), ② 내적 분산이 실측으로 `d_k`(64→63.7) 여서 `1/√d_k` 가 이를 1로 복원해 softmax 포화(무스케일 max prob 0.68 → 스케일 0.08)와 기울기 소실을 막는다. 다음 편([[Day-036]])은 이 이해 위에서 🛠️ **B3 빌드** — NumPy MLP+역전파부터 미니 Transformer 손코딩으로 잇는다.
