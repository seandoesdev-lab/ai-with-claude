## 0. 들어가며 (Two ways to draw the same boundary)

[[Day-010]] 에서 우리는 감정 분류기를 만들며 한 가지 분류기 — **로지스틱 회귀(logistic regression)** — 를 썼습니다. 로지스틱 회귀는 "벡터 $x$ 가 주어졌을 때 라벨이 1일 확률"을 **직접** 모델링합니다. 이런 부류를 **판별 모델(discriminative model)** 이라 부릅니다: 경계선(decision boundary)을 곧장 긋는 데만 집중하죠.

오늘은 정반대 철학의 분류기를 만듭니다. **"각 클래스가 어떤 단어를 얼마나 잘 만들어내는가"** 를 먼저 배우고, 그걸 **베이즈 정리** 로 뒤집어 분류하는 — **생성 모델(generative model)** 인 **나이브 베이즈** 입니다.

> *질문: "이 문장은 긍정인가?"를 풀 때, 경계선을 직접 긋는 대신 — "긍정 글쓴이라면 이 단어들을 쓸 확률"과 "부정 글쓴이라면 쓸 확률"을 각각 계산해 더 그럴듯한 쪽을 고르면 어떨까?*

놀랍게도 이 단순한 발상은 [[Day-011]] 에서 만든 **언어모델(단어의 확률을 세는 기계)** 과 한 뿌리입니다. 나이브 베이즈는 사실상 **"클래스마다 작은 언어모델을 하나씩 두고, 어느 모델이 이 문장을 더 잘 설명하는지 겨루는"** 분류기입니다. 그래서 [[Day-011]] 의 카운트·스무딩이 오늘 그대로 재등장합니다 — *같은 도구, 다른 과제*.

> Day-010 used a **discriminative** classifier (logistic regression draws the boundary directly). Today we build a **generative** one — **Naïve Bayes** — which learns *how each class generates words*, then flips it with **Bayes' theorem**. It's essentially one tiny [[Day-011]] language model per class, competing to explain the sentence.

## 1. 핵심 개념 (Core Concepts)

### 1.1 베이즈 정리 — 조건을 뒤집기

우리가 알고 싶은 건 **문장(단어들)이 주어졌을 때 클래스의 확률** $P(c \mid \text{words})$ 입니다. 그런데 이건 직접 세기 어렵습니다("이 정확한 문장이 긍정일 확률"은 데이터에 거의 안 나오니까 — [[Day-011]] 의 희소성 문제 그대로). **베이즈 정리** 는 이걸 *셀 수 있는 방향* 으로 뒤집습니다:

$$ P(c \mid w_1,\dots,w_n) = \frac{P(c)\;P(w_1,\dots,w_n \mid c)}{P(w_1,\dots,w_n)} $$

분모 $P(w_1,\dots,w_n)$ 는 클래스와 무관(모든 $c$ 에 똑같이 나눠짐)하므로 **분류에는 무시** 해도 됩니다. 따라서:

$$ \hat{c} = \arg\max_{c}\; \underbrace{P(c)}_{\text{사전확률 prior}}\;\underbrace{P(w_1,\dots,w_n \mid c)}_{\text{우도 likelihood}} $$

### 1.2 "나이브(naïve)" 한 가정 — 단어들이 서로 독립

문제는 여전히 $P(w_1,\dots,w_n \mid c)$ — *단어 전체 조합* 의 확률 — 을 셀 수 없다는 것입니다. 여기서 나이브 베이즈의 **순진한(naïve) 가정** 이 등장합니다:

> **클래스가 주어지면, 단어들은 서로 독립이다.** (conditional independence)

즉 단어 조합의 확률을 **개별 단어 확률의 곱** 으로 쪼갭니다:

$$ P(w_1,\dots,w_n \mid c) \approx \prod_{i=1}^{n} P(w_i \mid c) $$

이건 명백히 *틀린* 가정입니다("San"과 "Francisco"는 독립이 아니죠). 하지만 **분류 결정** 에는 놀랍도록 잘 통합니다 — 정확한 확률이 아니라 *어느 클래스가 더 큰가* 의 **순위** 만 맞으면 되기 때문입니다(생각거리 1). 그래서 나이브 베이즈는 "틀렸지만 유용한" 모델의 교과서적 예입니다.

### 1.3 각 확률을 카운트로 추정 — [[Day-011]] 의 재방문

이제 셀 것은 두 가지뿐입니다.

- **사전확률(prior)** $P(c)$ = (클래스 $c$ 의 문서 수) / (전체 문서 수).
- **우도(likelihood)** $P(w \mid c)$ = 클래스 $c$ 안에서 단어 $w$ 가 나온 빈도 비율. 이게 바로 [[Day-011]] 의 **유니그램 언어모델** 입니다:

$$ P(w \mid c) = \frac{\text{count}(w, c)}{\sum_{w'} \text{count}(w', c)} $$

### 1.4 0 확률의 재앙과 라플라스 스무딩 — 또 [[Day-011]]

만약 단어 $w$ 가 클래스 $c$ 의 학습 데이터에 **한 번도** 안 나왔다면 $P(w\mid c)=0$ 이 됩니다. 그러면 곱셈 $\prod P(w_i\mid c)$ 가 **통째로 0** 이 되어, 다른 단어가 아무리 그 클래스를 가리켜도 **단어 하나가 거부권(veto)** 을 행사합니다. [[Day-011]] 에서 본 그 문제 그대로죠.

해법도 똑같습니다 — **라플라스(add-α) 스무딩**: 모든 카운트에 $\alpha$(보통 1)를 더합니다.

$$ P(w \mid c) = \frac{\text{count}(w,c) + \alpha}{\sum_{w'}\big(\text{count}(w',c)+\alpha\big)} = \frac{\text{count}(w,c)+\alpha}{\big(\sum_{w'}\text{count}(w',c)\big) + \alpha V} $$

($V$ = 어휘 크기.) 이러면 본 적 없는 단어도 0 대신 **아주 작은 확률** 을 받아 거부권이 사라집니다.

### 1.5 로그 공간에서 계산 — 언더플로우 방지

수십 개 확률(각각 0~1)을 곱하면 컴퓨터의 부동소수점이 **0으로 언더플로우** 합니다. 그래서 [[Day-011]] 의 퍼플렉서티 계산처럼 **로그(log)** 를 취해 곱셈을 덧셈으로 바꿉니다:

$$ \log P(c \mid \text{words}) \;\propto\; \log P(c) + \sum_{i=1}^{n} \log P(w_i \mid c) $$

$\arg\max$ 는 단조증가 함수인 로그를 취해도 안 바뀌므로, 우리는 이 **로그 점수가 가장 큰 클래스** 를 고르면 됩니다.

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 두 명의 글쓴이
> 방에 두 사람이 있습니다 — 영화를 **사랑하는 낙천가(긍정)** 와 **혹평가(부정)**. 각자 자기만의 "단어 주머니(bag of words)"를 갖고 있습니다: 낙천가의 주머니엔 `great·wonderful·amazing` 이 잔뜩, 혹평가의 주머니엔 `awful·boring·terrible` 이 잔뜩.
>
> 이제 누군가 쪽지("an awful and boring film")를 건넵니다. 나이브 베이즈는 묻습니다: *"이 쪽지를, 낙천가가 자기 주머니에서 단어를 하나씩 뽑아 썼을 확률이 클까, 혹평가가 썼을 확률이 클까?"* 각 단어를 **독립적으로 뽑는다**고 가정하고(그래서 '나이브'), 각자의 주머니 구성으로 확률을 곱해 더 그럴듯한 글쓴이를 범인으로 지목합니다.
>
> **스무딩의 비유.** 쪽지에 낙천가가 **한 번도 안 쓴 단어**(`marvelous`)가 있으면? 스무딩이 없으면 낙천가는 "난 그 단어 절대 안 써 → 확률 0 → 내가 쓴 쪽지 아님"이라며 **무조건 탈락**합니다. 라플라스 스무딩은 모두에게 "혹시 모르니 모든 단어를 한 번씩은 써봤다고 치자"며 **작은 가능성** 을 깔아줍니다.
>
> Two writers — an optimist and a critic — each with their own bag of words. Naïve Bayes asks which writer was more likely to have drawn this note, word by word (independently — hence *naïve*). Smoothing stops a single never-seen word from vetoing a writer outright.

## 3. 한 장의 그림 (One Picture)

```text
        "an awful and boring film"
                  │
        ┌─────────┴──────────────────────────────┐
        ▼                                         ▼
  [긍정(pos) 언어모델]                      [부정(neg) 언어모델]
  logP(pos)                                 logP(neg)
  + logP(awful|pos)  ← 작음(스무딩값)        + logP(awful|neg)  ← 큼
  + logP(boring|pos) ← 작음                  + logP(boring|neg) ← 큼
  + logP(film|pos)                           + logP(film|neg)
        │                                         │
        ▼                                         ▼
     로그 점수 합산  ───────  argmax  ───────►  더 큰 쪽 = 예측 클래스
                                                  (여기선 부정)

   각 P(w|c) 는 카운트로 추정 + 라플라스 스무딩 (count+α)/(N_c+αV)   ← Day-011 그대로
```

> One tiny language model per class; sum log-probabilities; pick the larger. Each `P(w|c)` is a smoothed count — exactly the [[Day-011]] machinery.

### 4.2 나이브 베이즈를 from scratch 로 — `naive_bayes.py`

아래를 **`naive_bayes.py`** 로 저장합니다(이 노트 옆에 함께 제공). 1절의 수식이 그대로 코드가 됩니다: `fit` 에서 사전확률·우도를 **카운트 + 스무딩** 으로 추정 → `log_scores` 에서 **로그 덧셈** → `argmax`. 데이터는 [[Day-010]] 과 **똑같은** 감정 문장 24개로, *분류기만 바꿔* 공정 비교합니다.

```python
# naive_bayes.py — 나이브 베이즈(Multinomial NB)로 감정 분류 (순수 파이썬, 외부 라이브러리 0)
import sys, math
from collections import Counter
sys.stdout.reconfigure(encoding="utf-8")  # Windows 콘솔 한글 깨짐 방지

# 0) Day-010 과 '같은' 작은 감정 데이터셋 (1=긍정, 0=부정)
pos = [
    "this movie was great and i loved it", "an amazing film with a wonderful story",
    "the acting was brilliant and moving", "a fantastic and enjoyable film overall",
    "i really liked this great movie", "a wonderful and amazing story i loved",
    "brilliant acting and a fantastic story", "i loved the wonderful and great film",
    "an enjoyable and brilliant movie overall", "the story was amazing and i really liked it",
    "a great film with wonderful acting", "i enjoyed this amazing and brilliant movie",
]
neg = [
    "this movie was terrible and i hated it", "an awful film with a boring story",
    "the acting was bad and dull", "a dreadful and boring film overall",
    "i really disliked this terrible movie", "an awful and boring story i hated",
    "bad acting and a dreadful story", "i hated the awful and terrible film",
    "a boring and dreadful movie overall", "the story was terrible and i really disliked it",
    "a bad film with awful acting", "i disliked this terrible and dreadful movie",
]

def split(items): return items[:-2], items[-2:]   # 각 클래스 끝 2개를 테스트로 (Day-005)
pos_tr, pos_te = split([(t, 1) for t in pos])
neg_tr, neg_te = split([(t, 0) for t in neg])
train, test = pos_tr + neg_tr, pos_te + neg_te
X_train, y_train = [t for t, _ in train], [y for _, y in train]
X_test,  y_test  = [t for t, _ in test],  [y for _, y in test]

class MultinomialNB:
    def __init__(self, alpha=1.0): self.alpha = alpha   # alpha=0 → 스무딩 없음
    def fit(self, docs, labels):
        self.classes = sorted(set(labels))
        self.vocab = sorted({w for d in docs for w in d.split()}); V = len(self.vocab)
        N = len(docs); self.logprior = {}; self.loglik = {c: {} for c in self.classes}
        for c in self.classes:
            c_docs = [d for d, y in zip(docs, labels) if y == c]
            self.logprior[c] = math.log(len(c_docs) / N)            # log P(class)
            counts = Counter(w for d in c_docs for w in d.split())
            denom = sum(counts.values()) + self.alpha * V           # (Σcount) + αV
            for w in self.vocab:
                numer = counts[w] + self.alpha
                self.loglik[c][w] = math.log(numer / denom) if numer > 0 else float("-inf")
        return self
    def log_scores(self, doc):
        scores = {}
        for c in self.classes:
            s = self.logprior[c]
            for w in doc.split():
                if w in self.loglik[c]: s += self.loglik[c][w]   # 학습 어휘 밖 단어는 무시
            scores[c] = s
        return scores
    def predict(self, doc):
        s = self.log_scores(doc); return max(s, key=s.get)
    def predict_proba_pos(self, doc):
        s = self.log_scores(doc); m = max(s.values())
        exps = {c: math.exp(v - m) for c, v in s.items()}        # 수치 안정 softmax
        return exps.get(1, 0.0) / sum(exps.values())

def metrics(name, y_true, y_pred):
    tp = sum(1 for t,p in zip(y_true,y_pred) if t==1 and p==1)
    fp = sum(1 for t,p in zip(y_true,y_pred) if t==0 and p==1)
    fn = sum(1 for t,p in zip(y_true,y_pred) if t==1 and p==0)
    tn = sum(1 for t,p in zip(y_true,y_pred) if t==0 and p==0)
    acc=(tp+tn)/len(y_true); prec=tp/(tp+fp) if tp+fp else 0; rec=tp/(tp+fn) if tp+fn else 0
    f1=2*prec*rec/(prec+rec) if prec+rec else 0
    print(f"\n===== {name} =====")
    print(f"  accuracy={acc:.3f}  precision={prec:.3f}  recall={rec:.3f}  f1={f1:.3f}")

nb = MultinomialNB(alpha=1.0).fit(X_train, y_train)
metrics("MultinomialNB (alpha=1.0, 라플라스 스무딩)", y_test, [nb.predict(d) for d in X_test])

# 로그우도비 log P(w|pos) - log P(w|neg) 로 '가장 긍정/부정' 단어 보기
ratio = {w: nb.loglik[1][w] - nb.loglik[0][w] for w in nb.vocab}
print("긍정어 top5:", sorted(ratio, key=ratio.get, reverse=True)[:5])
print("부정어 top5:", sorted(ratio, key=ratio.get)[:5])

for s in ["a wonderful and amazing film i loved", "a boring and awful movie i hated"]:
    p = nb.predict_proba_pos(s); print(f"  P(pos)={p:.3f} -> {'긍정' if p>=0.5 else '부정'} | {s}")
```

> 💡 노트 옆의 `.py` 파일에는 **스무딩을 끈(alpha=0) 경우 0 확률이 어떻게 점수를 −inf 로 만드는지** 보여주는 데모까지 들어 있습니다.


### 4.3 이 결과를 정직하게 읽기 (관찰 포인트)

1. **NB 도 1.00 으로 완승 — TF-IDF 와 어깨를 나란히.** [[Day-010]] 에서 TF-IDF+로지스틱 회귀가 1.00, 자가학습 임베딩이 0.50 이었죠. 나이브 베이즈도 **1.00** 입니다. 작고 키워드가 또렷한 데이터에서는 *카운트 기반 방법(TF-IDF·NB)* 이 강하다는 [[Day-010]] 의 교훈을 한 번 더 확인합니다.
2. **그런데 일반화는 NB 가 훨씬 자신 있다.** 새 문장에서 [[Day-010]] 임베딩은 `0.527 / 0.467`(거의 찍기)였지만, NB 는 **`0.985 / 0.012`** — 방향도 맞고 **확신(margin)도 두껍습니다.** 왜? NB 는 24문장 전체의 단어 통계를 *직접* 우도로 쓰는 반면, Day-010 임베딩은 그 적은 데이터로 벡터를 *먼저 학습* 하느라 신호가 묽어졌기 때문입니다. *데이터가 적을수록 단순 카운트 모델이 유리* 합니다.
3. **−inf 데모 = 0 확률의 재앙을 눈으로.** 스무딩을 끄면 `log_scores('the acting was dull')` 의 긍정 점수가 **−inf** 가 됩니다 — `dull` 이 긍정 데이터에 없어 $P(\text{dull}\mid\text{pos})=0$ 이기 때문. 단어 하나가 거부권을 행사하는 [[Day-011]] 의 문제 그대로이고, 라플라스 스무딩이 정확히 이걸 막습니다.
4. **로그우도비 = 해석 가능성(interpretability).** `amazing·brilliant·great·loved·wonderful` 이 가장 긍정, `boring·terrible·awful·dreadful·hated` 이 가장 부정으로 깔끔히 분리됩니다. NB 는 **"왜 그렇게 분류했는지"를 단어 단위로 설명** 할 수 있는 투명한 모델입니다 — 블랙박스 신경망과 대비되는 큰 장점.

### 4.4 (실험) 직접 흔들어 보기

- **`alpha` 를 바꿔 보기**: `alpha=0.01`, `0.1`, `1`, `10` 으로 돌려 새 문장의 `P(pos)` 가 어떻게 0.5 쪽으로 밀려나는지(스무딩이 셀수록 확신이 약해짐) 관찰하세요. 스무딩은 *일반화 ↔ 자신감* 의 손잡이입니다.
- **부정어 함정 재현**: 학습 데이터에 "this movie is **not** good" 류를 넣고, `not good` 을 NB 가 어떻게 처리하는지 보세요. 단어를 독립으로 보는 NB 는 [[Day-010]] 의 평균 임베딩처럼 **어순·부정** 을 못 살립니다(생각거리 2). → 바이그램(bigram) NB 로 확장하면?
- **sklearn 과 대조**: `uv add scikit-learn` 후 `from sklearn.naive_bayes import MultinomialNB` 와 `CountVectorizer` 로 같은 데이터를 돌려, 우리 from-scratch 구현과 결과가 일치하는지 검증하세요(재현성 — [[Day-012]] Pass 3 의 정신).

## 5. 용어집 (Glossary)

| 용어 | English | 한 줄 뜻 |
|------|---------|---------|
| 나이브 베이즈 | Naïve Bayes | 단어 조건부 독립을 가정해 베이즈 정리로 분류하는 생성 모델 |
| 베이즈 정리 | Bayes' theorem | $P(c\mid x)=P(c)P(x\mid c)/P(x)$ — 조건을 뒤집는 공식 |
| 사전확률 | prior | 데이터를 보기 전 클래스 확률 $P(c)$ |
| 우도 | likelihood | 클래스가 그 단어들을 낼 확률 $P(\text{words}\mid c)$ |
| 사후확률 | posterior | 단어를 본 뒤의 클래스 확률 $P(c\mid\text{words})$ |
| 조건부 독립 | conditional independence | 클래스가 정해지면 단어들이 서로 무관하다는 가정 |
| 생성/판별 모델 | generative / discriminative | 데이터 생성을 모델링 vs 경계선을 직접 모델링 |
| 라플라스 스무딩 | Laplace (add-α) smoothing | 모든 카운트에 α를 더해 0 확률을 막음 |
| 로그우도비 | log-likelihood ratio | $\log P(w\mid c_1)-\log P(w\mid c_2)$, 단어의 클래스 편향 |
| 언더플로우 | underflow | 작은 확률을 곱하다 0으로 뭉개지는 현상(→ 로그로 회피) |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **"틀렸지만 유용한" 가정**: 단어 독립 가정은 명백히 거짓("New"와 "York"은 독립이 아님)인데, 왜 나이브 베이즈는 *분류* 에서 잘 작동할까요? 힌트: 분류는 정확한 확률값이 아니라 클래스 간 **순위(argmax)** 만 맞으면 됩니다. 확률 추정치 자체는 얼마나 믿을 수 있을까요?
> 2. **평균이 죽인 것을 NB 도 죽인다**: [[Day-010]] 의 평균 임베딩이 "not good"의 부정을 못 살렸듯, 단어 독립 NB 도 똑같이 못 살립니다. 두 모델이 *같은 한계* 를 공유하는 이유는 무엇이며, 이를 풀려면 무엇이 필요할까요? (이 한계가 Phase 3 의 **어텐션·트랜스포머** 로 가는 동기입니다.)
> 3. **생성 vs 판별**: 같은 데이터에서 [[Day-010]] 의 로지스틱 회귀(판별)와 오늘의 NB(생성)가 둘 다 1.00 을 냈습니다. 데이터가 **아주 적을 때** 와 **아주 많을 때**, 둘 중 어느 쪽이 더 유리할지 추측하고 그 이유를 생각해 보세요. (힌트: NB 는 가정이 강해 *적은 데이터* 에 강하고 빨리 수렴합니다.)

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **나이브 베이즈** 는 클래스마다 작은 [[Day-011]] 식 언어모델을 두고, **베이즈 정리** 로 $P(c\mid\text{words})\propto P(c)\prod P(w\mid c)$ 를 계산해 **로그 점수가 가장 큰 클래스** 를 고르는 *생성형* 분류기다. 각 $P(w\mid c)$ 는 **카운트 + 라플라스 스무딩**(0 확률 방지)으로 추정하고, 언더플로우를 막으려 **로그 공간** 에서 더한다. [[Day-010]] 과 같은 데이터에서 NB 는 **정확도 1.00** 으로 TF-IDF 와 동급이면서, 새 문장 일반화는 **0.985/0.012** 로 임베딩 베이스라인보다 훨씬 자신 있었다 — *작고 또렷한 데이터에서 단순 카운트 모델의 힘* 을 재확인. 단, 단어 독립 가정 탓에 [[Day-010]] 평균 임베딩과 **같은 부정·어순의 한계** 를 공유한다(→ Phase 3 트랜스포머의 동기). 이로써 Phase 1 의 분류 줄기가 완성되고, 다음은 🛠️ **B1 빌드 마일스톤** 이다.
