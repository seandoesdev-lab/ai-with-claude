> 어제 [[Day-031]] 의 LSTM·GRU 는 게이트로 장기 기억을 크게 개선했지만, 두 가지 근본 한계가 남았습니다 — ① **한 스텝씩 순차 처리** 라 병렬화가 어렵고, ② 시퀀스 전체가 **하나의 벡터로 압축** 되어 정보 병목(bottleneck)이 생깁니다. 오늘은 이 병목을 뚫는 **Attention(주의)** 을 배웁니다. 핵심 발상은 하나 — 출력의 각 위치가 입력의 **모든 위치를 직접 참조** 하고, 그중 **관련 있는 곳에 가중치(주의)** 를 두어 정보를 끌어오는 것입니다. 이를 **질의(Query)·키(Key)·값(Value)** 세 역할로 정식화하면 그 유명한 한 줄, **`Attention(Q,K,V) = softmax(QKᵀ/√d)·V`** 가 됩니다. 실습에서는 **학습 없이도** 이 식을 손으로 구현해 (1) 주의 가중치가 확률분포(합=1)임을, (2) 질의와 '닮은' 키에 주의가 쏠리는 **정렬(alignment)** 을, (3) 왜 하필 `√d` 로 나누는지를 숫자로 확인합니다. 이 발상이 곧 [[Day-033]] 의 **Self-Attention** 과 Transformer([[Phase-3]] 정점, 📄 *Attention Is All You Need*)로 이어집니다.

## 0. 들어가며 (Motivation)

[[Day-030]]·[[Day-031]] 의 순환망(RNN/LSTM/GRU)으로 **번역** 을 한다고 해봅시다. 전형적 구조는 **seq2seq(인코더-디코더)** 입니다 — 인코더 RNN 이 입력 문장 *"나는 프랑스에서 자랐다"* 를 한 스텝씩 읽어 **마지막 은닉 상태 하나** 로 요약하고(이걸 **컨텍스트 벡터** 라 부릅니다), 디코더 RNN 이 그 벡터 하나만 보고 *"I grew up in France"* 를 한 단어씩 생성합니다.

문제는 명백합니다 — **문장이 아무리 길어도 정보를 딱 하나의 고정 크기 벡터에 욱여넣어야** 합니다. 20단어든 100단어든 같은 크기의 벡터로. 이것이 **정보 병목(information bottleneck)** 입니다. 긴 문장일수록 앞부분 정보가 그 하나의 벡터에서 희미해지고, 디코더는 "지금 이 단어를 낼 때 입력의 **어느 부분** 을 봐야 하는지" 를 알 길이 없습니다.

2015년 Bahdanau 등이 낸 해법이 **Attention** 입니다: 인코더의 **모든** 은닉 상태를 버리지 말고 전부 들고 있다가, 디코더가 각 단어를 낼 때마다 **"지금 필요한 입력 위치가 어디인지" 를 매번 다시 계산** 해 그곳들의 정보를 **가중 평균** 으로 끌어오자는 것이죠. 하나의 벡터로 압축하는 대신, **필요할 때마다 원본 전체를 다시 들여다본다** — 이 한 가지 아이디어가 병목을 풀고, 나아가 순환 자체를 없애는 Transformer([[Day-033]])로 가는 문을 엽니다.

> The seq2seq encoder crams a whole sentence into one fixed vector — a bottleneck. Attention keeps *all* encoder states and, for each output word, recomputes *which input positions matter* and pulls them in as a weighted average. Instead of compressing into one vector, it looks back at the whole source whenever needed.

## 1. 핵심 개념 (Core Concepts)

### 1.1 질의·키·값 (Query, Key, Value) — 도서관 비유로 시작

Attention 은 **"검색"** 의 미분 가능한(differentiable) 버전입니다. [[Day-024]] 의 벡터 공간 모델에서 질의 벡터와 문서 벡터의 **유사도** 로 순위를 매겼던 걸 떠올려 보세요 — 여기서도 똑같은 발상입니다. 세 역할로 나눕니다:

- **질의 (Query) `q`**: "나는 지금 **무엇을 찾고 있나**" — 디코더의 현재 상태.
- **키 (Key) `k`**: "각 위치는 **무엇에 대한 것인가**" — 검색을 위한 색인표(라벨).
- **값 (Value) `v`**: "각 위치가 **실제로 담은 내용**" — 찾으면 가져올 실체.

과정은: ① 질의를 모든 키와 견줘 **유사도 점수** 를 낸다 → ② 점수를 `softmax` 로 **가중치(합=1)** 로 바꾼다 → ③ 그 가중치로 값들을 **평균** 낸다. 딱딱한 검색은 "가장 비슷한 하나만" 고르지만(argmax), Attention 은 `softmax` 로 **부드럽게 섞어** 미분이 가능하게 만듭니다 — 그래서 역전파([[Day-027]])로 학습됩니다.

### 1.2 Scaled Dot-Product Attention — 오늘의 한 줄

가장 널리 쓰이는 형태는 유사도를 **내적(dot product)** 으로 재는 것입니다. 질의 행렬 `Q`(Lq×d), 키 `K`(Lk×d), 값 `V`(Lk×d)에 대해:

$$
\text{Attention}(Q,K,V) = \operatorname{softmax}\!\left(\frac{QK^\top}{\sqrt{d}}\right)V
$$

한 줄씩 뜯어보면:

| 단계 | 수식 | shape | 의미 |
|------|------|-------|------|
| ① 점수 | `S = QKᵀ / √d` | (Lq, Lk) | 질의·키 유사도(내적), √d 로 스케일 |
| ② 가중치 | `A = softmax(S, dim=-1)` | (Lq, Lk) | 각 질의마다 키들에 대한 **확률분포**(행 합=1) |
| ③ 출력 | `out = A · V` | (Lq, d) | 값들의 **주의 가중 평균** |

핵심은 **행렬곱 두 번과 softmax 한 번** 이 전부라는 것 — 순환(recurrence)이 **없습니다**. 그래서 모든 위치를 **한 번에(병렬로)** 처리할 수 있고, 이것이 RNN 대비 Transformer 의 결정적 속도 이점이 됩니다.

### 1.3 왜 `√d` 로 나누나 (Scaling)

`d` 차원 벡터 두 개의 내적은, 각 성분이 평균 0·분산 1이면 **분산이 대략 `d`** 가 됩니다(성분곱 `d`개의 합). `d` 가 커질수록 점수 `QKᵀ` 의 스케일이 커지고, `softmax` 에 큰 값이 들어가면 분포가 **거의 원-핫으로 포화(saturate)** 됩니다 — 한 곳만 1, 나머지는 0. 포화된 `softmax` 는 **기울기가 거의 0** 이라([[Day-031]] 의 게이트 포화와 같은 병) 학습이 멈춥니다. `√d` 로 나누면 분산이 다시 **1 근처** 로 돌아와 분포가 부드럽게 유지됩니다. 4부에서 `d=8→512` 로 키우며 **엔트로피가 어떻게 무너지는지** 를 숫자로 봅니다.

### 1.4 정렬 (Alignment) — Attention 이 '해석 가능'한 이유

가중치 행렬 `A` 는 그 자체로 의미가 있습니다 — **"출력의 i번째가 입력의 j번째를 얼마나 봤나"** 를 담은 **정렬(alignment) 지도** 입니다. 번역에서 이 지도를 그려 보면 *"cats" → "고양이"* 처럼 대응되는 단어끼리 밝게 빛납니다. RNN 의 은닉 벡터는 속을 들여다보기 어려웠지만, Attention 가중치는 **모델이 어디를 봤는지 눈으로 확인** 할 수 있습니다. 2부 실습에서 이 정렬을 막대그래프로 직접 봅니다.

### 1.5 이 자리는 어디인가 — Self-Attention 으로 가는 문턱

오늘의 Attention 은 **디코더(질의)가 인코더(키·값)를 보는** cross-attention 입니다. 여기서 질문 하나 — **질의·키·값을 모두 같은 시퀀스에서 뽑으면** 어떻게 될까요? 그러면 문장이 **자기 자신의 모든 단어를 서로 참조** 하게 됩니다("The animal didn't cross the street because **it** was tired" 에서 *it* 이 *animal* 을 보게). 이것이 **Self-Attention** 이고, 순환을 **완전히 버리고** 이것만 쌓아 만든 것이 Transformer 입니다 — 바로 다음 편 [[Day-033]] 의 주제입니다.

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 형광펜을 든 번역가
> [[Day-031]] 의 seq2seq 가 "**원문을 통째로 외운 뒤 책을 덮고** 번역하는 사람" 이었다면, Attention 은 "**원문을 펼쳐 놓고, 지금 쓸 단어에 필요한 부분만 형광펜으로 칠해 가며** 번역하는 사람" 입니다. *"grew up"* 을 쓸 차례엔 원문의 *"자랐다"* 에 형광펜(높은 가중치)을 긋고, *"France"* 를 쓸 땐 *"프랑스"* 에 긋죠. **어느 단어를 쓰느냐에 따라 매번 다른 곳** 을 칠합니다(질의마다 다른 가중치). 통째로 외우면(하나의 벡터) 긴 문장에서 앞부분을 잊지만, 원문을 펼쳐 두고 그때그때 골라 보면(모든 위치 참조) 아무리 길어도 필요한 정보를 정확히 집어냅니다. 그리고 그 형광펜 자국(가중치 지도)을 보면 "이 사람이 어디를 보고 번역했는지" 를 우리가 알 수 있습니다(정렬).
>
> If seq2seq was a translator who memorizes the source, shuts the book, then translates, Attention is one who keeps the source open and highlights just the words needed for the current output word — a different highlight each time. Nothing gets forgotten in a long sentence, and the highlight marks (attention weights) let us see exactly where the model looked.

## 3. 한 장의 그림 (One Picture)

```text
   seq2seq (Day-031)                      Attention (Day-032)
   ──────────────────                     ─────────────────────────────────────
   입력 ▶[인코더 RNN]▶ [context]  ▶디코더   입력 ▶[인코더]▶ h_1 h_2 h_3 h_4  (모두 보존!)
             하나의 벡터에 압축                                │  │  │  │  = Keys / Values
             → 긴 문장 병목                                    ▼  ▼  ▼  ▼
                                          디코더 상태 q ─▶ 유사도(q·k)  →  softmax  →  가중치
                                                              [0.07, 0.07, 0.79, 0.07]
                                                                          │
                                              out = Σ (가중치 × Value)  ◀─┘  '가중 평균'

   Attention(Q,K,V) = softmax( Q·Kᵀ / √d ) · V
   ┌─────────────────────────────────────────────────────────────────────────┐
   │ 하나의 벡터로 압축(RNN) → 병목  |  모든 위치를 직접 참조 + 가중 평균 → 병목 해소 │
   └─────────────────────────────────────────────────────────────────────────┘
```

> Instead of squeezing the source into one context vector, Attention keeps every encoder state as Keys/Values and, for each decoder query, computes softmax similarity weights and takes a weighted average of the Values.

### 4.2 1부 — Scaled Dot-Product Attention 을 손으로 (발췌)

1.2절의 식을 그대로 코드로 옮깁니다. **세 줄** 이 전부입니다:

```python
def scaled_dot_product_attention(Q, K, V):
    d = Q.shape[-1]
    scores = Q @ K.transpose(-2, -1) / (d ** 0.5)   # ① 질의·키 유사도, √d 스케일
    attn = F.softmax(scores, dim=-1)                # ② 확률분포로 (행 합=1)
    out = attn @ V                                  # ③ 값의 주의 가중 평균
    return out, attn
```

출력(시드 고정):

```
  질의 1개 → 키 4개에 대한 주의 가중치(softmax, 합=1):
    [0.411, 0.22, 0.096, 0.272]  합 = 1.0
  출력 = 이 가중치로 값 V 를 섞은 벡터, shape=(1, 8)
```

가중치가 **음수 없이 합이 정확히 1** — 즉 "키 4개에 주의를 어떻게 배분할지" 의 확률분포입니다. 출력은 이 비율로 값 `V` 를 섞은 한 벡터입니다.

### 4.3 2부 — 정렬(alignment) 을 눈으로 (발췌)

질의가 3번째 위치("cats")와 닮도록 세팅하면, 주의가 그곳으로 쏠립니다:

```python
q2 = torch.tensor([[0.2, 0.2, 5.0, 0.2]])   # 3번째 축이 큼 → 위치2(cats)와 내적이 큼
out2, attn2 = scaled_dot_product_attention(q2, K2, V2)
```

출력:

```
  질의가 3번째 위치(cats)를 가리킬 때 주의 가중치:
    I     0.071 ██
    love  0.071 ██
    cats  0.786 ███████████████████████████████
    .     0.071 ██
  → 출력은 거의 V[2]='cats'의 내용([0.0, 0.0, 30.0, 0.0])에 수렴: [0.71, 1.43, 23.58, 2.85]
```

**해석.** 질의와 가장 닮은 키('cats')가 주의의 **78.6%** 를 가져가고, 출력은 그 위치의 값 `[·,·,30,·]` 에 거의 수렴합니다. 이것이 번역에서 *"이 출력 단어는 저 입력 단어를 본다"* 는 **정렬** 이고, 사람이 눈으로 확인할 수 있는 Attention 의 해석 가능성입니다.

### 4.4 3부 — 왜 √d 로 나누나 (발췌)

차원 `d` 를 8→512 로 키우며, 스케일링 유무에 따라 주의 분포의 **엔트로피**(고를수록 큼)를 잽니다:

```python
raw    = F.softmax(Qd @ Kd.T, dim=-1)              # 스케일링 없음
scaled = F.softmax(Qd @ Kd.T / dim ** 0.5, dim=-1)  # √d 스케일링
```

실행 결과(재현 가능):

```
  d=   8 | 스케일 X 엔트로피 3.34 bits (max p=0.284) | 스케일 O 엔트로피 4.20 bits (max p=0.108)
  d=  64 | 스케일 X 엔트로피 0.24 bits (max p=0.967) | 스케일 O 엔트로피 3.94 bits (max p=0.170)
  d= 512 | 스케일 X 엔트로피 0.03 bits (max p=0.997) | 스케일 O 엔트로피 3.45 bits (max p=0.258)
```

**해석.** 스케일링을 **안 하면**(스케일 X) 차원이 커질수록 한 점수만 폭주해 `softmax` 가 `d=512` 에서 **max p=0.997**(거의 원-핫), 엔트로피 **0.03 bits** 로 완전히 포화됩니다 — 이 상태의 기울기는 사실상 0이라 학습이 죽습니다. `√d` 로 나누면(스케일 O) 같은 `d=512` 에서도 max p=0.258, 엔트로피 3.45 bits 로 분포가 **부드럽게 유지** 됩니다. 이것이 식에 `√d` 가 들어간 이유이자, 이름이 ***Scaled** Dot-Product Attention* 인 까닭입니다.

### 4.5 (실험) 직접 흔들어 보기

- **질의 방향 바꾸기**: 2부의 `q2` 에서 큰 성분의 위치를 바꿔(`[5.0, 0.2, 0.2, 0.2]`) 주의가 다른 단어로 옮겨 가는지 확인하세요 — 질의가 정렬을 결정합니다.
- **스케일링 제거의 대가**: 1부 함수에서 `/ (d ** 0.5)` 를 지우고 `d` 를 크게 해보면, 주의가 한 곳으로만 쏠려 "모든 위치를 보는" 이점이 사라집니다.
- **딱딱한 검색과 비교**: `softmax` 대신 `argmax`(가장 큰 하나만 1)로 바꾸면 미분이 끊겨 학습이 안 됩니다 — 왜 '부드러운' softmax 여야 하는지 체감할 수 있습니다.

## 5. 용어집 (Glossary)

| 용어 | English | 한 줄 뜻 |
|------|---------|---------|
| 주의 | attention | 질의와 닮은 위치의 값을 가중 평균으로 끌어오는 메커니즘 |
| 질의 | query `Q` | "지금 무엇을 찾나" — 유사도 계산의 기준(디코더 상태) |
| 키 | key `K` | "각 위치는 무엇에 대한 것인가" — 검색용 색인/라벨 |
| 값 | value `V` | "각 위치가 담은 실제 내용" — 찾으면 가져올 실체 |
| 스케일 내적 주의 | scaled dot-product attention | `softmax(QKᵀ/√d)V` — 오늘의 표준 형태 |
| 정렬 | alignment | 출력 위치가 입력 위치를 얼마나 봤나(가중치 지도) |
| 컨텍스트 벡터 | context vector | seq2seq 에서 입력을 압축한 하나의 벡터(병목의 원인) |
| 정보 병목 | information bottleneck | 긴 시퀀스를 고정 크기 벡터에 욱여넣어 생기는 손실 |
| 포화 | saturation | softmax 가 거의 원-핫이 되어 기울기가 0에 가까워짐 |
| seq2seq | sequence-to-sequence | 인코더-디코더로 시퀀스를 시퀀스로 변환(번역 등) |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **병목의 크기.** seq2seq 의 컨텍스트 벡터가 512차원이라면, 100단어 문장이든 5단어 문장이든 같은 512차원에 담깁니다. Attention 은 이 제약을 어떻게 없앴나요? (힌트: 저장하는 것이 '하나의 요약'에서 '무엇'으로 바뀌었나.)
> 2. **왜 softmax 인가.** 가장 닮은 키 하나만 고르는 `argmax` 대신 `softmax` 로 '부드럽게' 섞는 이유는 무엇일까요? [[Day-027]] 의 역전파와 연결해 생각해 보세요.
> 3. **Q=K=V 라면?** 오늘은 질의(디코더)와 키·값(인코더)이 서로 다른 시퀀스였습니다. 만약 셋을 모두 **같은 문장** 에서 뽑으면 무슨 일이 벌어질까요? (이것이 [[Day-033]] Self-Attention 의 씨앗입니다.)

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **Attention 은 seq2seq([[Day-031]])의 '하나의 벡터로 압축' 병목을, '모든 입력 위치를 직접 참조해 관련 있는 곳을 가중 평균으로 끌어오는' 방식으로 푼 메커니즘이다.** 입력을 **질의(Query)·키(Key)·값(Value)** 세 역할로 나누고, 질의와 키의 유사도(내적)를 `softmax` 로 가중치화한 뒤 값들을 섞는다 — 그 한 줄이 **`Attention(Q,K,V) = softmax(QKᵀ/√d)·V`** 이다. `√d` 로 나누는 것은 큰 차원에서 점수가 커져 softmax 가 포화(기울기 소멸)되는 것을 막기 위함이며(실습: `d=512`에서 스케일 없으면 max p=0.997·엔트로피 0.03, 있으면 max p=0.258·엔트로피 3.45), 가중치 행렬 자체가 "어느 출력이 어느 입력을 봤나" 를 보여주는 **정렬(alignment) 지도** 라 해석 가능성도 얻는다. 순환이 없어 **모든 위치를 병렬 처리** 할 수 있고, 질의·키·값을 같은 시퀀스에서 뽑는 **Self-Attention** 으로 확장해 순환을 완전히 대체한 것이 다음 편 [[Day-033]] 의 Transformer 다.
