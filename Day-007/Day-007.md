## 0. 들어가며 (From tokens to numbers)

[[Day-006]] 에서 우리는 `"the cat sat on the mat"` 같은 문장을 `["the", "cat", "sat", ...]` 토큰 리스트로 쪼갰습니다. 하지만 모델도 검색엔진도 **글자**가 아니라 **숫자**를 먹습니다. 그래서 다음 관문은 — **"토큰 → 숫자(벡터)"** 입니다.

가장 오래되고 단순한 답이 오늘의 주제입니다. **문서를 단어들의 "빈도 표(frequency table)"로 본다.** 순서·문법은 버리고, "어떤 단어가 몇 번 나왔는가"만 셉니다. 놀랍게도 이 단순한 표현만으로도 스팸 분류·문서 검색·주제 분류가 꽤 잘 됩니다.

> Computers eat **numbers**, not characters. The oldest, simplest bridge from tokens to numbers is to treat each document as a **table of word counts** — throwing away order and grammar. Despite its simplicity, this powers surprisingly good spam filters, search, and topic classifiers.

그리고 한 걸음 더 — 모든 단어가 똑같이 중요하진 않습니다. "the" 는 거의 모든 문서에 나오니 **변별력(discriminative power)** 이 없고, "mat" 처럼 드문 단어가 그 문서를 잘 대표합니다. 이 직관을 수식으로 만든 것이 **TF-IDF** 입니다. 오늘 이걸 손으로 계산해 [[Day-001]] 의 `naive_search` 를 진짜 검색 점수로 키웁니다.

## 1. 핵심 개념 (Core Concepts)

### 1.1 Bag-of-Words (단어 가방)

문서를 **단어들이 든 가방**으로 봅니다. 가방을 흔들어 순서를 섞어 버리고, **"무슨 단어가 몇 개 들었나"** 만 셉니다.

1. **어휘(vocabulary)** 만들기 — 말뭉치(corpus) 전체의 서로 다른 단어를 모아 정렬하고, 각 단어에 **인덱스(차원)** 를 부여합니다.
2. **벡터화** — 각 문서를, 어휘 크기만큼 긴 벡터로 표현합니다. `i` 번째 칸 = `i` 번째 단어가 그 문서에 나온 **횟수(count)**.

여러 문서를 쌓으면 **문서-단어 행렬(document-term matrix, DTM)** 이 됩니다. 행=문서, 열=단어.

> Bag-of-Words discards order and represents each document as a vector of word **counts** over a fixed **vocabulary**. Stacking documents gives a **document-term matrix** (rows = docs, columns = terms).

**한계**: ① 순서·문법 소실 ("dog bites man" = "man bites dog"), ② 차원이 어휘 크기만큼 커지고 대부분 0 인 **희소(sparse)** 벡터, ③ 단어의 **의미(meaning)** 를 모름 (`cat`·`kitten` 은 완전히 다른 차원). ③은 Phase 1 후반의 **임베딩(embedding, [[Day-008]] 예정)** 이 풀어 줍니다.

### 1.2 Term Frequency (TF, 단어 빈도)

한 문서 안에서 어떤 단어가 **자주** 나오면 그 문서의 주제와 관련이 깊을 것입니다. 가장 단순한 TF 는 **그냥 등장 횟수**입니다.

- **원시 빈도(raw count)**: `tf(t, d)` = 단어 `t` 가 문서 `d` 에 나온 횟수.
- 긴 문서가 유리해지지 않도록 **정규화**하기도 합니다: `tf = count / 문서_길이`, 또는 로그 스케일 `1 + log(count)` (**sublinear TF**).

### 1.3 Inverse Document Frequency (IDF, 역문서빈도)

문제는 "the·is·of" 같은 단어가 거의 모든 문서에 나와 TF 만으로는 **흔한 단어가 점수를 독식**한다는 것. 그래서 **여러 문서에 두루 나오는 단어일수록 깎아내립니다.**

- **문서 빈도(document frequency, df)**: 단어 `t` 가 **나타난 문서의 수** (한 문서에 10번 나와도 df 는 1).
- **IDF**: 전체 문서 수 `N` 을 `df` 로 나눈 뒤 로그:

$$ \text{idf}(t) = \log\frac{N}{\text{df}(t)} $$

`df` 가 크면(흔한 단어) IDF 가 0 에 가까워지고, `df` 가 작으면(희귀어) IDF 가 커집니다. (실무에선 0 나눗셈·극단값을 막으려 **스무딩(smoothing)** 을 더합니다: `log((1+N)/(1+df)) + 1` — 4.4 의 scikit-learn 기본값.)

> **IDF** down-weights terms that appear in *many* documents. Using **document frequency** (how many docs contain the term, not total occurrences) means ubiquitous words like "the" get an IDF near zero.

### 1.4 TF-IDF = TF × IDF

두 직관을 곱합니다:

$$ \text{tfidf}(t, d) = \text{tf}(t, d) \times \text{idf}(t) $$

- **이 문서에 자주 나오고(높은 TF) + 다른 문서엔 드문(높은 IDF)** 단어가 가장 큰 가중치를 받습니다 → 그 문서를 **잘 대표하는** 단어.
- "the" 처럼 어디나 나오는 단어는 TF 가 높아도 IDF 가 0 에 가까워 **눌립니다.**

이렇게 만든 TF-IDF 벡터를 [[Day-004]] 의 **코사인 유사도**로 비교하면 — 그게 바로 고전적 **문서 검색(retrieval)** 이고, Phase 2 의 **BM25** 로 진화하는 출발점입니다.

> **TF-IDF = TF × IDF.** High when a term is frequent *in this doc* but rare *across the corpus* — exactly the terms that characterize a document. Compare TF-IDF vectors with cosine similarity and you have classic document retrieval, the ancestor of BM25.

## 2. 직관 / 비유 (Intuition)

**파티에서 사람 구별하기 (Telling people apart at a party):**
- 모두가 입은 흰 티셔츠("the", "is", "of")는 **누가 누구인지 구별해 주지 못합니다** — IDF 가 낮습니다.
- 단 한 사람만 쓴 **빨간 모자**("mat", "log")는 그 사람을 단번에 가리킵니다 — IDF 가 높습니다.
- 그리고 그 사람이 빨간 모자를 **여러 개** 쓰고 있다면(높은 TF) 더더욱 확실하죠. **TF-IDF = "얼마나 자주" × "얼마나 특이한가".**

**가방 비유 (The bag):**
- BoW 는 문장을 **가방에 쏟아 단어를 세는 것**입니다. `"dog bites man"` 과 `"man bites dog"` 의 가방은 **똑같습니다** — 순서를 잃었으니까. 이것이 BoW 의 힘(단순·빠름)이자 약점(어순·문맥 소실)입니다.

> Common words are like white T-shirts everyone wears — useless for telling documents apart (low IDF). A rare word is a red hat that pinpoints one document (high IDF). TF-IDF = "how often" × "how distinctive". And BoW is literally dumping words into a bag: "dog bites man" and "man bites dog" give the *same* bag.

## 3. 한 장의 그림 (One picture to remember)

```
문서들 (documents)              어휘(vocab) 차원으로 벡터화
 D0 "the cat sat on the mat"      ┌─ and are cat ... mat ... the ─┐
 D1 "the dog sat on the log"   →  │ 각 칸 = 그 단어의 빈도(count) │   ← Bag-of-Words (DTM)
 D2 "cats and dogs are friends"   └──────────────────────────────┘
 D3 "the cat and the dog play"          │
                                         ▼   × IDF (흔한 단어일수록 ×0에 가깝게)
                                  TF-IDF 가중 벡터 (weighted)
                                         │
                                         ▼   코사인 유사도 (Day-004)
                                  질의(query)와 가장 가까운 문서 = 검색 결과
```

> documents → count words over a fixed vocabulary (BoW / DTM) → re-weight by IDF (TF-IDF) → compare to a query with cosine similarity → ranked search results.

### 4.2 BoW·TF-IDF·검색을 손으로 — `bow_tfidf.py`

아래를 **`bow_tfidf.py`** 로 저장합니다(옆에 함께 제공). 어휘 만들기 → BoW 행렬 → IDF → TF-IDF → 코사인 검색까지 한 번에 봅니다.

```python
# bow_tfidf.py — Bag-of-Words & TF-IDF를 표준 라이브러리만으로 직접 구현
import sys, re, math
from collections import Counter

sys.stdout.reconfigure(encoding="utf-8")   # Windows 콘솔 한글 깨짐 방지

corpus = [
    "the cat sat on the mat",     # D0
    "the dog sat on the log",     # D1
    "cats and dogs are friends",  # D2
    "the cat and the dog play",   # D3
]

def tokenize(text):
    return re.findall(r"[a-z]+", text.lower())

docs = [tokenize(d) for d in corpus]

# 1) 어휘(vocabulary): 등장한 모든 단어를 정렬해 인덱스(차원) 부여
vocab = sorted({t for toks in docs for t in toks})
idx = {t: i for i, t in enumerate(vocab)}
print("어휘 크기(vocab size):", len(vocab))
print("어휘:", vocab)

# 2) Bag-of-Words 벡터: 순서를 버리고 '빈도'만 센다
def bow(tokens):
    v = [0] * len(vocab)
    for term, c in Counter(tokens).items():
        v[idx[term]] = c
    return v

bow_matrix = [bow(toks) for toks in docs]
print("\n[BoW] D0 =", bow_matrix[0])
print("[BoW] D3 =", bow_matrix[3])

# 3) IDF = log10(N / df).  흔한 단어일수록 작아진다.
N = len(corpus)
df = Counter()
for toks in docs:
    for term in set(toks):       # 문서당 1번만 세기 = document frequency
        df[term] += 1
idf = {t: math.log10(N / df[t]) for t in vocab}
print("\n[IDF] (df=등장 문서 수, 흔할수록 idf 작다)")
for t in vocab:
    print(f"  {t:<8} df={df[t]}  idf={idf[t]:.4f}")

# 4) TF-IDF = tf * idf
def tfidf(tokens):
    tf = Counter(tokens)
    return {t: tf[t] * idf[t] for t in tf}

print("\n[TF-IDF] D0 의 단어별 가중치 (클수록 그 문서를 잘 대표)")
for t, w in sorted(tfidf(docs[0]).items(), key=lambda kv: -kv[1]):
    print(f"  {t:<8} tf={Counter(docs[0])[t]}  tfidf={w:.4f}")

# 5) 코사인 유사도로 문서 검색(ranking)  ← Day-004 와 연결
def cosine(a, b):
    common = set(a) & set(b)
    dot = sum(a[t] * b[t] for t in common)
    na = math.sqrt(sum(w * w for w in a.values()))
    nb = math.sqrt(sum(w * w for w in b.values()))
    return dot / (na * nb) if na and nb else 0.0

query = "cat and dog"
q_vec = tfidf(tokenize(query))
doc_vecs = [tfidf(toks) for toks in docs]
scores = [(i, cosine(q_vec, dv)) for i, dv in enumerate(doc_vecs)]
scores.sort(key=lambda x: -x[1])

print(f'\n[검색] 질의(query) = "{query}"')
for i, s in scores:
    print(f"  D{i}  score={s:.4f}  | {corpus[i]}")
```

- `cat`·`dog`·`and` 의 IDF 가 똑같이 0.3010 인 이유: 셋 다 df=2(두 문서에 등장)이기 때문입니다.

### 4.3 더 큰 문서에서 핵심어 뽑기 (실전 감각)

TF-IDF 의 대표적 쓰임은 **문서의 키워드 자동 추출**입니다. 위 `bow_tfidf.py` 의 코퍼스를 아래처럼 바꿔 다시 돌려 보세요(코드 수정 없이 `corpus` 만 교체):

```python
corpus = [
    "machine learning models learn patterns from data",
    "deep learning is a kind of machine learning with neural networks",
    "search engines rank documents by relevance to a query",
    "information retrieval ranks documents using term frequency",
]
```

각 문서에서 tfidf 가 가장 큰 단어를 뽑으면, D0 은 `models`·`patterns`, D2 는 `search`·`engines`·`rank`·`relevance`·`query` 처럼 **그 문서만의 주제어**가 위로 올라옵니다. 반면 두 문서에 모두 나오는 `learning`·`documents` 는 IDF 때문에 점수가 깎입니다. **TF-IDF 가 "주제어 탐지기"로 작동**하는 걸 직접 보세요.

### 4.4 (보너스) 표준 도구로 — `scikit-learn` (선택)

원리를 이해한 지금, 실무 표준 도구를 만나 봅니다. scikit-learn 의 `TfidfVectorizer` 는 토큰화·어휘·IDF·**L2 정규화**를 한 번에 해 줍니다.

```powershell
uv add scikit-learn
```

```python
# sklearn_tfidf.py (선택) — 표준 라이브러리로 같은 일을
import sys
from sklearn.feature_extraction.text import CountVectorizer, TfidfVectorizer
from sklearn.metrics.pairwise import cosine_similarity

sys.stdout.reconfigure(encoding="utf-8")

corpus = [
    "the cat sat on the mat",
    "the dog sat on the log",
    "cats and dogs are friends",
    "the cat and the dog play",
]

# (1) Bag-of-Words 행렬
cv = CountVectorizer()
bow = cv.fit_transform(corpus)
print("어휘:", cv.get_feature_names_out())
print("BoW 행렬(밀집):\n", bow.toarray())

# (2) TF-IDF 행렬 + 코사인 검색
tv = TfidfVectorizer()        # 기본: smooth idf + L2 정규화
X = tv.fit_transform(corpus)
q = tv.transform(["cat and dog"])
sims = cosine_similarity(q, X)[0]   # 질의 vs 각 문서
ranked = sorted(range(len(corpus)), key=lambda i: -sims[i])
print('\n[검색] query = "cat and dog"')
for i in ranked:
    print(f"  D{i}  score={sims[i]:.4f}  | {corpus[i]}")
```


scikit-learn 의 IDF 는 **스무딩**(`log((1+N)/(1+df))+1`)과 **L2 정규화**를 쓰므로 점수의 *절댓값*은 4.2 와 다릅니다. 하지만 **순위는 똑같습니다 — `D3` 가 1위.** 우리가 손으로 이해한 개념이 이 검증된 구현 뒤에 그대로 들어 있습니다. (`CountVectorizer` 의 기본 토크나이저는 한 글자 토큰을 버리는 등 우리 정규식과 미세하게 다를 수 있으니, `get_feature_names_out()` 으로 어휘를 꼭 확인하세요.)

## 5. 용어집 (Glossary)

| 용어 | 영어 | 한 줄 뜻 |
|------|------|---------|
| 단어 가방 | Bag-of-Words (BoW) | 순서를 버리고 단어 빈도만으로 문서를 표현 |
| 문서-단어 행렬 | document-term matrix (DTM) | 행=문서, 열=단어, 값=빈도인 행렬 |
| 단어 빈도 | term frequency (TF) | 한 문서 안에서 단어가 나온 횟수(또는 정규화값) |
| 문서 빈도 | document frequency (df) | 그 단어가 **등장한 문서의 수** |
| 역문서빈도 | inverse document frequency (IDF) | log(N/df) — 흔한 단어를 깎는 가중치 |
| TF-IDF | term frequency–inverse df | TF × IDF, 문서를 대표하는 단어에 큰 값 |
| 스무딩 | smoothing | 0 나눗셈·극단값을 막는 보정(+1 등) |
| 희소 벡터 | sparse vector | 대부분의 값이 0 인 고차원 벡터 |
| 정규화 | normalization (L2) | 벡터 길이를 1로 맞춤(코사인과 직결) |
| 키워드 추출 | keyword extraction | TF-IDF 상위 단어로 문서 주제어 뽑기 |
| 베이스라인 | baseline | 비교 기준이 되는 단순·강력한 방법 |

## 6. 생각거리 (Reflection)

스스로 답해 보세요 (정답보다 사고 과정이 목적):

1. **순서를 잃는 대가**: BoW 에서 `"dog bites man"` 과 `"man bites dog"` 의 벡터는 완전히 같습니다. 이 정보 손실이 **검색**에서는 큰 문제가 안 되지만 **번역·질의응답**에서는 치명적인 이유는 무엇일까요? (이 한계가 Phase 3 의 **어텐션(attention)** 으로 가는 동기입니다.)
2. **왜 df 이고 총빈도가 아닌가**: IDF 는 단어의 *총 등장 횟수*가 아니라 *등장한 문서 수(df)* 를 씁니다. 한 문서에 "cat" 이 100번 나오는 것과, 100개 문서에 1번씩 나오는 것은 IDF 관점에서 전혀 다릅니다. 왜 "몇 개 문서에 퍼졌는가"가 단어의 **변별력**을 더 잘 나타낼까요?
3. **lemmatization 의 효과 예측**: 4.2 에서 `D2 "cats and dogs are friends"` 가 질의 "cat and dog" 에 꼴찌였습니다. [[Day-006]] 의 표제어 추출로 `cats→cat`, `dogs→dog` 를 적용하면 D2 의 순위는 어떻게 바뀔까요? 예측한 뒤, `corpus` 를 직접 고쳐 확인해 보세요.

## 7. 오늘의 한 줄 요약 (One-line takeaway)

> **Bag-of-Words** 는 순서를 버리고 단어 빈도만으로 문서를 벡터로 만들고, **TF-IDF = TF × IDF** 는 "이 문서에 자주(TF) + 다른 문서엔 드문(IDF)" 단어에 큰 가중치를 줘 문서의 **대표어**를 드러낸다. 이 벡터를 [[Day-004]] 의 **코사인 유사도**로 비교하면 작동하는 **문서 검색**이 되고, 이것이 Phase 2 의 **BM25** 와 Phase 5 의 **밀집 검색(dense retrieval)** 으로 진화하는 출발선이다. 단순하지만 지금도 강력한 베이스라인이다.
