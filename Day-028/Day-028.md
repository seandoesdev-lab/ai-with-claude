## 0. 들어가며 (Motivation)

지난 편([[Day-027]])의 마지막 생각거리를 떠올려 봅시다: *"층이 50개인 Transformer 라면 미분을 사람이 다 유도할 수 있을까?"* 답은 **아니오** 입니다. 우리가 손으로 적은 `dz2 = (a2 - y)/N`, `dz1 = (dz2 @ W2.T) * sigmoid_grad(a1)` 같은 식은 **층·연산이 바뀔 때마다 새로 유도** 해야 하고, 실수 한 번이면 조용히 틀립니다.

그래서 현대 딥러닝은 미분을 **자동으로** 계산합니다. 그 도구가 **PyTorch** — 연구·산업 양쪽에서 가장 널리 쓰이는 딥러닝 프레임워크입니다. PyTorch 의 마법은 단순합니다: **당신은 순전파(forward)만 적으면 된다.** 그러면 PyTorch 가 그 계산 과정을 **계산 그래프(computational graph)** 로 몰래 기록해 두었다가, `loss.backward()` 를 부르는 순간 [[Day-027]] 에서 우리가 손으로 하던 연쇄법칙 역전파를 **전부 자동** 으로 수행합니다. 오늘 우리는 이 도구를 손에 쥐고, 이후 RNN·Attention·Transformer([[Phase-3]]) 로 나아갈 발판을 만듭니다.

> Last time we derived backprop *by hand*. That does not scale to a 50-layer Transformer. Modern deep learning computes gradients **automatically**. PyTorch's core idea: you only write the **forward pass**; it silently records a **computational graph**, and `loss.backward()` replays the entire chain-rule backward pass from [[Day-027]] for you.

## 1. 핵심 개념 (Core Concepts)

### 1.1 텐서(tensor) = NumPy 배열 + 두 가지 초능력

**텐서** 는 [[Day-003]] 에서 배운 NumPy 다차원 배열과 거의 똑같습니다 — 인덱싱·브로드캐스팅·행렬곱(`@`) 모두 동일하게 씁니다. 다른 점은 **두 가지 초능력** 입니다:

| 초능력 | 무엇 | 왜 중요한가 |
|--------|------|-------------|
| **① GPU 실행** | `.to("cuda")` 로 텐서를 GPU 로 옮기면 수천 개 코어가 병렬로 계산 | 대형 모델 학습을 수십~수백 배 빠르게 |
| **② autograd(자동미분)** | `requires_grad=True` 를 켠 텐서의 모든 연산을 그래프로 기록 → 기울기 자동 계산 | 손으로 역전파를 유도할 필요가 없어짐 |

즉 **텐서 ≈ "미분 가능하고 GPU 에서 도는 NumPy 배열"** 이라고 이해하면 충분합니다.

### 1.2 계산 그래프(computational graph) — autograd 가 몰래 그리는 지도

`requires_grad=True` 인 텐서로 연산을 하면, PyTorch 는 결과 텐서에 **"이 값이 어떤 연산으로, 어떤 입력에서 나왔는지"** 를 매달아 둡니다. 이렇게 순전파를 따라 자동으로 쌓이는 **연산의 방향 그래프** 가 계산 그래프입니다. 이것이 [[Day-027]] 에서 우리가 머릿속으로 그렸던 $x \to z_1 \to a_1 \to z_2 \to \hat y \to L$ 사슬을 **PyTorch 가 실행 중에 자동으로 기록** 한 것입니다.

### 1.3 `loss.backward()` — 역전파 한 줄

스칼라 손실 `loss` 에서 `loss.backward()` 를 부르면, PyTorch 는 계산 그래프를 **출력에서 입력 방향으로 거꾸로** 훑으며([[Day-027]] 의 backward 와 정확히 같은 방향) 각 잎(leaf) 텐서의 편미분을 계산해 그 텐서의 **`.grad`** 속성에 채웁니다. 우리가 어제 손으로 적은 `dW1, dW2 …` 가 이제 `W1.grad, W2.grad …` 로 **자동** 으로 나타나는 것입니다.

$$
\underbrace{\texttt{dz2, dW2, dz1, dW1 = ...}}_{\text{Day-027: 손으로 유도}}
\qquad\Longrightarrow\qquad
\underbrace{\texttt{loss.backward()}}_{\text{Day-028: 한 줄}}
$$

### 1.4 학습 스텝의 3대 관용구 (반드시 외울 패턴)

PyTorch 학습 루프는 [[Day-027]] 의 4단계(① forward ② loss ③ backward ④ update)를 그대로 코드화하되, 딱 세 줄의 관용구를 씁니다:

```python
grad 초기화   # ① 기울기는 누적되므로 매 스텝 0으로 (안 하면 이전 기울기와 더해짐!)
loss.backward()   # ② 역전파 자동 수행 → .grad 채움
파라미터 갱신   # ③ θ ← θ − η·grad (경사하강, Day-014·027 과 같은 규칙)
```

> [!warning] 초보가 가장 많이 하는 실수 — grad 누적
> PyTorch 는 `.grad` 를 **덮어쓰지 않고 더합니다(accumulate).** 매 스텝 `grad.zero_()`(또는 `optimizer.zero_grad()`)로 초기화하지 않으면, 기울기가 스텝마다 쌓여 학습이 폭주합니다. "backward 전에 grad 를 비운다" 를 습관으로 만드세요.

### 1.5 `torch.no_grad()` — "이건 미분하지 마"

파라미터를 갱신하는 `θ -= lr*θ.grad` 는 **학습 대상이 아니라 그냥 값 갱신** 이므로, 이 연산까지 그래프에 기록되면 낭비이자 오류입니다. `with torch.no_grad():` 블록 안에서 하면 autograd 가 그래프 기록을 잠시 꺼서 안전·효율적으로 갱신합니다. (평가·추론 때도 씀 — 기울기가 필요 없으니까)

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 영수증을 모아 두는 자동 회계사 (autograd)
> [[Day-027]] 에서 우리는 회사의 오차를 부서별로 **손수 계산기를 두드려** 책임 배분(기울기)을 했습니다. PyTorch 의 autograd 는 **자동 회계사** 입니다. 당신이 돈을 쓸 때(=순전파 연산)마다 회계사가 **영수증(계산 그래프)** 을 조용히 모아 둡니다. 결산일에 "총손실이 얼마야?" 하고 `backward()` 를 외치면, 회계사가 모아 둔 영수증을 **역순으로 되짚어** 각 부서·개인이 손실에 기여한 몫(`.grad`)을 순식간에 정산해 줍니다. 당신은 **돈 쓰는 규칙(forward)만** 정하면 되고, 정산(backward)은 회계사가 알아서 합니다. 단, 매 분기 시작에 **장부를 초기화(`zero_()`)** 하지 않으면 지난 분기 숫자가 더해져 엉망이 됩니다.
>
> Autograd is an automatic accountant: every forward operation quietly files a receipt (the graph). Call `backward()` and it retraces the receipts in reverse to settle each parameter's share of the loss into `.grad`. You only define how money is spent (forward); it does the settlement — but you must clear the ledger (`zero_()`) each round.

## 3. 한 장의 그림 (One Picture)

```text
   당신이 적는 것 = 순전파(forward)만
   ┌─────────────────────────────────────────────────────────────┐
   │  a1 = sigmoid(X @ W1 + b1)                                    │
   │  ŷ  = sigmoid(a1 @ W2 + b2)      ← PyTorch 가 이 연산들을      │
   │  loss = BCE(ŷ, y)                  계산 그래프로 몰래 기록      │
   └─────────────────────────────────────────────────────────────┘
                    │  loss.backward()  ← 한 줄
                    ▼
   ◀════ autograd 가 그래프를 거꾸로 훑어 .grad 자동 계산 ════
        W1.grad   b1.grad   W2.grad   b2.grad
        (= Day-027 에서 손으로 유도하던 dW1, db1, dW2, db2 그 자체)
                    │  with torch.no_grad():  θ -= lr * θ.grad
                    ▼
   경사하강 갱신 (Day-014·027 과 같은 규칙)   ── 이 한 바퀴 = 1 step
   ───────────────────────────────────────────────────────────────
   Day-027: forward + [손으로 짠 backward] + update
   Day-028: forward +   loss.backward()    + update   (backward 자동화)
```

> You write only the forward pass; PyTorch records it as a graph. `loss.backward()` walks the graph in reverse and fills each parameter's `.grad` — exactly the `dW1, dW2 …` we derived by hand in [[Day-027]]. Then gradient descent (same rule) updates the weights.

### 4.2 1부 — autograd 로 미분 자동 계산 (발췌)

```python
import torch

# requires_grad=True → 이 텐서로 이어지는 연산이 그래프로 기록된다
w = torch.tensor(2.0, requires_grad=True)
b = torch.tensor(1.0, requires_grad=True)
x = torch.tensor(3.0)

y = w * x + b        # 순전파
z = y ** 2           # z = (w*x + b)^2  (스칼라 손실 흉내)
z.backward()         # ← 역전파 자동! (Day-027 의 손계산 대체)

print(w.grad)        # dz/dw = 2*(w*x+b)*x = 2*7*3 = 42.0  (자동으로 맞음)
print(b.grad)        # dz/db = 2*(w*x+b)   = 14.0
```

$z=(wx+b)^2$ 를 손으로 미분하면 $\partial z/\partial w = 2(wx+b)\cdot x = 2\cdot 7\cdot 3 = 42$. autograd 가 이 값을 **유도 없이** 뱉어 냅니다.

### 4.3 2부 — XOR MLP 를 PyTorch 로 (발췌)

```python
X = torch.tensor([[0.,0.],[0.,1.],[1.,0.],[1.,1.]])
Y = torch.tensor([[0.],[1.],[1.],[0.]])          # XOR

H = 4                                            # Day-027 과 동일 구조
W1 = torch.randn(2, H, requires_grad=True); b1 = torch.zeros(1, H, requires_grad=True)
W2 = torch.randn(H, 1, requires_grad=True); b2 = torch.zeros(1, 1, requires_grad=True)
params = [W1, b1, W2, b2]
loss_fn = torch.nn.BCELoss()                     # Day-027 의 이진 교차엔트로피

def forward(x):
    a1 = torch.sigmoid(x @ W1 + b1)
    return torch.sigmoid(a1 @ W2 + b2)           # 예측 확률 ŷ

for epoch in range(1, 8001):
    loss = loss_fn(forward(X), Y)                # ① forward → ② loss
    for p in params:                             # (grad 초기화 — 누적 방지)
        if p.grad is not None: p.grad.zero_()
    loss.backward()                              # ③ 역전파 자동!
    with torch.no_grad():                        # ④ 경사하강 갱신
        for p in params: p -= 0.5 * p.grad
```

### 4.4 결과를 정직하게 읽기 (관찰 포인트)

1. **autograd = 손계산과 정확히 일치.** 1부에서 `dz/dw=42.0, dz/db=14.0` 은 손으로 유도한 값과 소수점까지 같고, 수치미분(41.96)과도 일치합니다 — autograd 를 **믿어도 된다** 는 증거.
2. **`loss.backward()` 한 줄이 [[Day-027]] 전체를 대체.** 어제의 `dz2, dW2, dz1, dW1` 네 줄 유도가 오늘은 사라졌습니다. 우리는 **순전파(`forward`)만** 적었을 뿐인데 기울기가 `W1.grad …` 로 자동으로 나타났습니다.
3. **같은 문제, 같은 결말 — 정확도 1.00.** [[Day-027]] 의 NumPy 손코딩과 초기화·시드가 달라 손실 숫자는 다르지만(500 epoch 에서 0.20 vs 0.69 등 초기 수렴 양상 차이), 결국 XOR 를 **완전히 학습** 합니다. *도구가 바뀌어도 원리(4단계 순환)는 그대로* 임을 확인하세요.
4. **손실 곡선은 여전히 계단형.** epoch 100(0.69)에서 한동안 헤매다 500~1000 사이 급락(0.20→0.03) — [[Day-027]] 에서 본 plateau→급락 현상이 PyTorch 에서도 재현됩니다.

### 4.5 (실험) 직접 흔들어 보기

- **grad 초기화 빼 보기**: `p.grad.zero_()` 두 줄을 지우면 기울기가 누적돼 손실이 발산(NaN)합니다 — 1.4 절 경고를 몸으로 체감.
- **`no_grad()` 빼 보기**: 갱신을 `with torch.no_grad():` 없이 하면 "leaf 텐서를 제자리 수정" 오류가 납니다. autograd 가 왜 갱신을 그래프에서 빼야 하는지 이해하는 계기.
- **`torch.nn.Linear` 로 바꿔 보기**: 다음 편 예고 — 위 `forward` 를 `nn.Linear(2,4)`, `nn.Linear(4,1)` 로 바꾸면 `W1=torch.randn(...)` 수동 선언이 사라집니다. 이것이 [[Day-029]] 의 `nn.Module`.
- **[[Day-027]] 와 나란히 실행**: `backprop.py`(NumPy)와 `autograd_intro.py`(PyTorch)를 함께 돌려, 손코딩 backward 와 autograd 가 **같은 일** 을 함을 눈으로 대조하세요.

| 텐서 | tensor | GPU 실행·자동미분이 가능한 다차원 배열 (NumPy 배열의 확장) |
| 자동미분 | autograd | 순전파를 기록해 두었다 역전파로 기울기를 자동 계산하는 엔진 |
| 계산 그래프 | computational graph | 순전파 연산들의 방향 그래프 — backward 가 거꾸로 훑는 지도 |
| requires_grad | requires_grad | 이 텐서에 대한 기울기를 추적할지 켜는 플래그 |
| grad | .grad | backward 가 채워 넣는, 그 텐서에 대한 손실의 기울기 |
| backward | backward() | 계산 그래프를 역순으로 훑어 모든 `.grad` 를 채우는 호출 |
| grad 누적 | gradient accumulation | `.grad` 가 덮이지 않고 더해지는 성질 → 매 스텝 초기화 필요 |
| no_grad | no_grad | 그래프 기록을 잠시 꺼 갱신·추론을 안전·효율적으로 하는 문맥 |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **왜 순전파만 적으면 되나.** PyTorch 가 순전파 중에 계산 그래프를 기록해 두기 때문에 backward 가 가능합니다. 그렇다면 `if`/`for` 로 매 스텝 **구조가 달라지는** 신경망도 미분할 수 있을까요? (힌트: PyTorch 는 실행할 때마다 그래프를 새로 그립니다 — "define-by-run")
> 2. **grad 를 왜 더하게(누적) 만들었을까.** 매번 덮어쓰는 게 편할 텐데 PyTorch 는 굳이 누적을 기본으로 합니다. 배치를 여러 조각으로 나눠 기울기를 모으는 상황(gradient accumulation)을 떠올리면 왜 유용한지 보입니다 — 어떤 경우일지 상상해 보세요.
> 3. **손계산 vs 자동.** [[Day-027]] 의 `backprop.py` 와 오늘의 `autograd_intro.py` 는 같은 XOR 를 풉니다. 그런데도 [[Day-027]] 에서 굳이 **손으로** 역전파를 유도해 본 것이 왜 가치가 있었을까요? (도구를 블랙박스로 두지 않는 것의 의미)

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **PyTorch 의 두 기둥은 텐서(tensor)와 autograd 다.** 텐서는 NumPy 배열([[Day-003]])에 *GPU 실행* 과 *자동미분* 을 더한 자료구조이고, autograd 는 순전파를 **계산 그래프** 로 기록해 두었다가 **`loss.backward()` 한 줄** 로 [[Day-027]] 의 연쇄법칙 역전파 전체를 자동 수행해 각 파라미터의 `.grad` 를 채운다. 학습 루프는 [[Day-027]] 의 4단계(forward→loss→backward→update)를 그대로 따르되, 세 관용구 — **grad 초기화 → `loss.backward()` → `no_grad()` 로 갱신** — 를 쓴다(grad 는 누적되므로 매 스텝 초기화 필수). [[Day-027]] 의 XOR MLP 를 PyTorch 로 다시 짜, 손코딩 backward 없이 **정확도 1.00** 을 재현했다(1부에서 autograd 가 손계산·수치미분과 일치함도 확인). 🛠️ 이 autograd 는 Phase 3 마무리 **B3 빌드(미니 Transformer)** 의 토대이며, 다음 편 [[Day-029]] 는 `nn.Module`·`torch.optim` 으로 모델과 갱신을 캡슐화한다.
