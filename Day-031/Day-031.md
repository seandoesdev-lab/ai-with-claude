> 어제 [[Day-030]] 의 바닐라 RNN 은 순환식 `h_t = tanh(Wₕₕ·h_{t-1} + …)` 에서 **`Wₕₕ` 를 매 시점 반복 곱** 하기 때문에, 시퀀스가 길면 기울기가 뒤로 갈수록 **0으로 소멸(vanishing gradient)** 해 **먼 과거를 잊습니다**(장기 의존성 학습 실패). 오늘은 이 한계를 **게이트(gate)** 로 푸는 두 아키텍처, **LSTM(Long Short-Term Memory)** 과 **GRU(Gated Recurrent Unit)** 를 배웁니다. 핵심 아이디어는 하나 — 상태를 매 시점 **곱셈으로 덮어쓰지 말고**, **덧셈으로 갱신되는 별도의 "기억 통로"(cell state `c_t`)** 를 두고, **무엇을 잊고(forget)·새로 쓰고(input)·내보낼지(output)** 를 신경망이 스스로 학습한 게이트로 조절하는 것입니다. 실습에서는 **학습 없이도** 기울기 소실을 직접 관측해(RNN 은 `e-3 → e-34` 로 소멸, forget 게이트를 연 LSTM 은 살아남음), "왜 게이트가 먼 과거를 지키는가" 를 숫자로 확인합니다. 이 게이트 발상은 곧 [[Day-032]] 의 **Attention** 과 Transformer([[Phase-3]])로 이어집니다.

## 0. 들어가며 (Motivation)

[[Day-030]] 의 마지막에서 예고한 대로, 바닐라 RNN 에는 결정적 약점이 있습니다. 순환식이 상태를 이렇게 갱신하죠:

$$h_t = \tanh(W_{xh}\,x_t + W_{hh}\,h_{t-1} + b)$$

`h_{t-1}` 이 다음 스텝에서 `W_{hh}` 로 곱해지고, 그 결과가 또 곱해지고… **역전파(BPTT, [[Day-027]]·[[Day-028]])** 때 기울기는 이 과정을 거꾸로 타면서 `W_{hh}` 와 `tanh'` 을 **T번 반복 곱** 합니다. 곱해지는 값이 1보다 작으면 기울기는 **지수적으로 0에 수렴**(vanishing)하고, 1보다 크면 **폭발**(exploding)합니다. 그래서 RNN 은 "**문장 앞부분의 단서를 문장 끝까지 기억**" 하는 일을 잘 못합니다 — 예: *"나는 프랑스에서 자랐다 … (긴 문장) … 그래서 나는 유창하게 **[프랑스어]** 를 한다"* 에서 빈칸을 채우려면 아주 앞의 "프랑스" 를 끝까지 기억해야 하는데, 바닐라 RNN 은 그때쯤이면 그 신호가 사라집니다.

해결의 실마리는 [[Day-030]] 4.2절에서 이미 봤습니다 — **`h_t` 안에는 과거가 `tanh` 와 `W_{hh}` 의 반복 곱으로 포개져 있어 뒤로 갈수록 희미해진다**는 것. 그렇다면 **곱셈으로 덮어쓰는 대신, 정보가 거의 손실 없이 흐르는 "덧셈 통로" 를 따로 만들면** 어떨까요? 그것이 바로 LSTM 의 **셀 상태(cell state)** 이고, "무엇을 그 통로에 남기고 지울지" 를 정하는 스위치가 **게이트** 입니다.

> The vanilla RNN multiplies by `W_{hh}` at every step, so gradients shrink (vanish) or blow up (explode) across long sequences — it forgets the distant past. The fix: keep a separate **cell state** that is updated by **addition**, and learn **gates** that decide what to keep, write, and read.

## 1. 핵심 개념 (Core Concepts)

### 1.1 게이트(gate) 란 무엇인가

**게이트** 는 `[0, 1]` 사이 값을 내는 작은 신경망(보통 `sigmoid`)입니다. 이 값을 어떤 벡터에 **원소별로 곱하면**, `0` 인 곳은 정보를 **막고**(닫힌 문), `1` 인 곳은 그대로 **통과**(열린 문)시킵니다. 즉 게이트는 "**정보의 흐름을 조절하는 밸브**" 입니다. 결정적으로 이 밸브의 개폐 정도(`sigmoid` 안의 가중치)를 **데이터로부터 학습** 합니다 — 규칙을 사람이 짜는 게 아니라, "이런 맥락에선 기억을 유지, 저런 맥락에선 지움" 을 스스로 배웁니다.

### 1.2 LSTM — 셀 상태와 세 개의 게이트

LSTM 은 은닉 상태 `h_t` 외에 **셀 상태(cell state) `c_t`** 를 하나 더 들고 다닙니다. `c_t` 는 "장기 기억을 실어 나르는 컨베이어 벨트" 이고, 세 게이트가 그 위에서 작동합니다:

$$
\begin{aligned}
f_t &= \sigma(W_f x_t + U_f h_{t-1} + b_f) &&\text{(forget: 무엇을 잊을까)}\\
i_t &= \sigma(W_i x_t + U_i h_{t-1} + b_i) &&\text{(input: 무엇을 새로 쓸까)}\\
\tilde{g}_t &= \tanh(W_g x_t + U_g h_{t-1} + b_g) &&\text{(candidate: 새로 쓸 후보 내용)}\\
o_t &= \sigma(W_o x_t + U_o h_{t-1} + b_o) &&\text{(output: 무엇을 내보낼까)}\\[4pt]
c_t &= f_t \odot c_{t-1} + i_t \odot \tilde{g}_t &&\textbf{(셀 상태: 유지 + 새로 쓰기 — 덧셈!)}\\
h_t &= o_t \odot \tanh(c_t) &&\text{(은닉 상태: 셀을 걸러 출력)}
\end{aligned}
$$

여기서 `⊙` 는 원소별 곱입니다. 가장 중요한 줄은 **`c_t = f_t ⊙ c_{t-1} + i_t ⊙ g̃_t`** — 셀 상태가 **곱셈이 아니라 덧셈** 으로 갱신됩니다. `f_t ≈ 1`(잊지 않기), `i_t ≈ 0`(새로 안 쓰기)이면 `c_t ≈ c_{t-1}` 로 **과거가 거의 그대로 다음으로 흘러갑니다**. 이 "덧셈 통로" 덕에 기울기도 잘 보존됩니다(1.4절에서 숫자로 확인).

### 1.3 GRU — 더 가벼운 사촌

**GRU(2014)** 는 LSTM 을 단순화한 버전입니다. 셀 상태를 따로 두지 않고 `h_t` 하나로 합치며, 게이트도 **두 개** 만 씁니다:

$$
\begin{aligned}
r_t &= \sigma(W_r x_t + U_r h_{t-1}) &&\text{(reset: 과거를 얼마나 무시할까)}\\
z_t &= \sigma(W_z x_t + U_z h_{t-1}) &&\text{(update: 과거 vs 새 정보 비율)}\\
\tilde{h}_t &= \tanh(W_h x_t + U_h (r_t \odot h_{t-1}))\\
h_t &= (1 - z_t)\odot h_{t-1} + z_t \odot \tilde{h}_t &&\textbf{(과거와 새 후보의 가중 평균 — 역시 덧셈 구조)}\\
\end{aligned}
$$

`update` 게이트 `z_t` 가 "과거 `h_{t-1}` 을 얼마나 유지하고 새 후보 `h̃_t` 를 얼마나 반영할지" 를 **가중 평균** 으로 섞습니다. 게이트가 하나 적어 **파라미터가 더 적고 더 빠르며**, 많은 과제에서 LSTM 과 **비슷한 성능** 을 냅니다(3부에서 파라미터 수 비교: RNN 304 < GRU 912 < LSTM 1216).

### 1.4 왜 게이트가 기울기 소실을 막나 — 핵심 직관

셀 상태의 갱신식 `c_t = f_t ⊙ c_{t-1} + …` 를 시간축으로 역전파하면, 먼 과거로 흐르는 기울기의 핵심 인자는 **forget 게이트들의 곱** `∏ f_t` 입니다. `f_t ≈ 1` 로 유지되면 이 곱은 **1 근처에 머물러 기울기가 소멸하지 않습니다**. 반면 바닐라 RNN 의 대응 인자는 `∏ (W_{hh}·\text{diag}(\tanh'))` 인데, `tanh'≤1` 이고 반복 곱이라 **지수적으로 0** 이 됩니다. 즉 LSTM 은 "기울기가 지나갈 **거의 손실 없는 덧셈 고속도로**" 를 하나 깐 셈입니다(그래서 원논문 제목이 *Long Short-Term Memory* — "길게 지속되는 단기 기억"). 이 차이를 4부에서 **학습 없이** 숫자로 직접 봅니다.

### 1.5 PyTorch — `nn.LSTM` · `nn.GRU`

[[Day-030]] 의 `nn.RNN` 과 **인터페이스가 사실상 동일** 합니다. `nn.RNN` 을 `nn.LSTM`/`nn.GRU` 로 바꾸기만 하면 됩니다:

```python
self.rnn = nn.LSTM(input_size=1, hidden_size=16, batch_first=True)
out, (h_n, c_n) = self.rnn(x)   # LSTM 만 (h_n, c_n) 튜플로 반환 (셀 상태 c_n 추가)
# GRU/RNN 은:  out, h_n = self.rnn(x)
```

- `out` : 모든 시점의 은닉 `h_1…h_L`, `h_n` : 마지막 은닉, `c_n` : 마지막 셀 상태(LSTM 전용)
- 학습 루프([[Day-029]] 의 `zero_grad → backward → step` 세 줄)와 손실은 **그대로** 재사용 — 모델 한 줄만 바뀝니다.

### 1.6 이 자리는 어디인가 — Attention 으로 가는 문턱

LSTM·GRU 는 **마지막 순환형(recurrent) 아키텍처** 입니다. 게이트로 장기 기억을 크게 개선했지만, 여전히 **한 스텝씩 순차 처리** 해야 하고(병렬화가 어렵고), 아주 긴 문맥에선 정보가 하나의 벡터에 병목됩니다. 이 한계를 "**모든 위치를 한 번에 참조**" 하는 방식으로 뛰어넘는 것이 다음 편 [[Day-032]] 의 **Attention** 이고, 그것만으로 시퀀스를 처리하는 것이 Transformer 입니다.

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 컨베이어 벨트 위의 세 개의 밸브
> [[Day-030]] 의 RNN 이 "**메모지 한 장**" 이었다면, LSTM 은 공장의 **컨베이어 벨트(셀 상태 `c_t`)** 를 하나 깐 것입니다. 벨트는 처음부터 끝까지 물건(기억)을 거의 손실 없이 실어 나릅니다. 벨트 위엔 세 개의 밸브가 있습니다: **forget 밸브** 는 벨트 위 오래된 물건을 버릴지 정하고(닫으면 유지), **input 밸브** 는 새 물건을 벨트에 올릴지 정하며, **output 밸브** 는 지금 벨트에서 무엇을 꺼내 볼지 정합니다. 밸브를 여닫는 규칙은 공장장이 짜는 게 아니라 **데이터로 학습** 됩니다 — "인용부호가 열렸으니 닫힐 때까지 이 사실을 유지" 같은 규칙을 스스로 익히죠. 벨트가 곱셈이 아니라 **덧셈** 으로 굴러가기에, 앞에서 올린 물건이 뒤까지 **또렷이** 도착합니다(=기울기가 소멸하지 않음). GRU 는 이 설계를 간소화해 밸브를 **둘** 로 줄인 실용형입니다.
>
> If the RNN was a single sticky note, the LSTM adds a **conveyor belt** (cell state) that carries memory across the whole sequence with little loss, plus three learned **valves** (forget/input/output). Because the belt moves by **addition, not multiplication**, what you place early still arrives clearly at the end — gradients don't vanish. GRU is the streamlined two-valve version.

## 3. 한 장의 그림 (One Picture)

```text
   RNN (Day-030)                 LSTM (Day-031)  — 셀 상태 c_t 라는 '덧셈 고속도로'
   ───────────────               ─────────────────────────────────────────────────
   h_{t-1} ─▶(× Whh, tanh)─▶ h_t     c_{t-1} ─────▶(× f_t)──(＋ i_t·g̃_t)─────▶ c_t   ← 덧셈 갱신
     매 스텝 '곱셈'으로 덮어씀              │  forget            input↑            │
     → 반복 곱 → 기울기 소멸               │                                     ▼
                                        h_{t-1} ─▶[f,i,g̃,o 게이트 계산]     h_t = o_t·tanh(c_t)
                                        x_t ────▶                              ▲ output

   c_t = f_t·c_{t-1} + i_t·g̃_t     (f≈1 이면 c_t≈c_{t-1} : 과거가 거의 그대로 흐른다)
   h_t = o_t·tanh(c_t)
   ┌──────────────────────────────────────────────────────────────────────────┐
   │ 곱셈 반복(RNN) → 소멸 |  덧셈 통로 + 게이트(LSTM/GRU) → 먼 과거 보존       │
   └──────────────────────────────────────────────────────────────────────────┘
```

> The cell state `c_t` is an additive "highway": with the forget gate ≈ 1, `c_t ≈ c_{t-1}`, so the distant past flows forward almost intact — unlike the RNN's repeated multiplication.

### 4.2 1부 — LSTM 셀을 손으로 한 스텝 (발췌)

1.2절의 식을 그대로 코드로 옮겨, 게이트 4개와 셀 상태 `c_t` 를 직접 계산합니다.

```python
def lstm_step(x_t, h_prev, c_prev):
    f = torch.sigmoid(Wf @ x_t + Uf @ h_prev + bf)   # forget: 무엇을 잊을까
    i = torch.sigmoid(Wi @ x_t + Ui @ h_prev + bi)   # input : 무엇을 새로 쓸까
    g = torch.tanh(   Wg @ x_t + Ug @ h_prev + bg)   # candidate: 새로 쓸 후보
    o = torch.sigmoid(Wo @ x_t + Uo @ h_prev + bo)   # output: 무엇을 내보낼까
    c = f * c_prev + i * g       # ← 셀 상태: '덧셈'으로 갱신 (오늘의 핵심)
    h = o * torch.tanh(c)        # ← 은닉 상태: 셀을 걸러 출력
    return h, c, f
```

출력(시드 고정 `torch.manual_seed(0)`):

```
  c_0 = [0.0, 0.0, 0.0] (빈 장기기억)
  x_1=1 | forget f_1= [0.68, 0.46, 0.25] | c_1= [0.038, 0.355, -0.21]
  x_2=0 | forget f_2= [0.5,  0.53, 0.49] | c_2= [0.035, 0.138, -0.191]
  x_3=1 | forget f_3= [0.69, 0.48, 0.25] | c_3= [0.071, 0.389, -0.278]
```

`c_t` 는 매 스텝 **이전 셀 `c_{t-1}` 에 forget 을 곱해 남기고, 새 후보를 더해** 갱신됩니다 — `c_t = f_t·c_{t-1} + i_t·g̃_t`. RNN 처럼 통째로 덮어쓰지 않는다는 점이 핵심입니다.

### 4.3 2부 — 기울기 소실을 '학습 없이' 눈으로 (발췌)

가장 중요한 실험입니다. 무작위 초기화된 RNN/LSTM 에 길이 `L` 시퀀스를 넣고, **마지막 시점 출력** 에서 손실을 역전파해 **첫 시점 입력이 받는 기울기 크기** 를 잽니다. 이 값이 크면 "먼 과거까지 학습 신호가 도달" 하는 것이고, 0에 가까우면 소실입니다.

```python
def grad_to_first(cell, L, forget_bias=None):
    net = {"RNN": nn.RNN, "LSTM": nn.LSTM}[cell](1, 16, batch_first=True)
    if cell == "LSTM" and forget_bias is not None:
        # forget 게이트 편향을 크게 → sigmoid(f)≈1 : '기억 유지'로 열어 둠
        for nm, p in net.named_parameters():
            if "bias" in nm:
                n = p.shape[0] // 4; p.data[n:2*n].fill_(forget_bias)  # 순서 i,f,g,o
    x = torch.randn(1, L, 1, requires_grad=True)
    out, _ = net(x); out[:, -1, :].sum().backward()
    return x.grad[0, 0, 0].abs().item()          # '첫 시점' 입력이 받는 기울기
```

실행 결과(재현 가능, `torch.manual_seed(0)`):

```
   L | 바닐라 RNN | LSTM(forget 열림)
  ---+------------+------------------
   10 | 1.86e-03  | 1.28e-01
   20 | 3.89e-07  | 1.41e-01
   40 | 5.29e-12  | 8.51e-02
   80 | 5.67e-23  | 3.52e-01
  120 | 3.46e-34  | 5.57e-01
```

**해석.** 바닐라 RNN 은 길이가 10→120 으로 늘수록 첫 시점 기울기가 `e-3` 에서 `e-34` 로 **지수적으로 소멸** 합니다 — 이게 바로 "먼 과거를 학습하지 못하는" 기울기 소실입니다. 반면 forget 게이트를 열어 둔(`b_f=5`, `sigmoid(5)≈0.993`) LSTM 은 길이와 **거의 무관하게** 기울기를 `e-1` 수준으로 **살려 둡니다**. 이것이 1.4절 "forget 게이트들의 곱 `∏ f_t ≈ 1` 이 덧셈 고속도로를 만든다" 를 숫자로 확인한 것입니다. (참고: 학습된 장기기억 LSTM 은 이렇게 forget 게이트를 열어 두는 상태에 **스스로 도달** 하므로, 실전에서 `b_f` 를 1로 초기화하는 것이 흔한 요령입니다.)

### 4.4 3부 — 게이트 비용: 파라미터 수 (발췌)

```
  RNN  |   304 params | 게이트 없음(1개 변환)
  GRU  |   912 params | 게이트 3개분(reset·update·후보)
  LSTM |  1216 params | 게이트 4개분(forget·input·output·후보)
```

게이트가 늘수록 파라미터도 늡니다(RNN < GRU < LSTM, 대략 게이트 수에 비례). GRU 가 LSTM 보다 가벼운 이유가 여기 있습니다 — 성능이 비슷하다면 GRU 가 실용적 선택이 되곤 합니다.

### 4.5 (실험) 직접 흔들어 보기

- **forget 게이트를 닫아 보기**: 2부의 `forget_bias=5.0` 을 `-5.0`(sigmoid≈0.007)으로 바꾸면 LSTM 도 기울기가 **소멸** 합니다 — 게이트가 열려 있어야 기억이 보존됨을 반증으로 확인.
- **기본 초기화 확인**: `forget_bias=0.0`(sigmoid=0.5)이면 LSTM 도 `0.5^L` 로 서서히 줄어듭니다. "학습이 forget 게이트를 여는 법을 배워야" 장기기억이 생긴다는 뜻.
- **직접 학습**: [[Day-030]] 의 "첫 비트 기억하기" 를 `nn.LSTM`/`nn.GRU` 로 바꿔(모델 한 줄) 길이 `L` 을 키우며 정확도를 비교해 보세요. (단, 이 단일-비트 과제는 워낙 쉬워 RNN 도 꽤 잘 풉니다 — 게이트의 진짜 이점은 **여러 방해 신호 사이에서** 먼 정보를 지킬 때 드러납니다.)

## 5. 용어집 (Glossary)

| 용어 | English | 한 줄 뜻 |
|------|---------|---------|
| 장단기 메모리 | LSTM | 셀 상태 + 3게이트로 장기 기억을 지키는 순환망 |
| 게이트 순환 유닛 | GRU | 게이트 2개로 LSTM 을 단순화한 경량 순환망 |

| 셀 상태 | cell state `c_t` | 덧셈으로 갱신되는 장기 기억 통로("컨베이어 벨트") |
| 망각 게이트 | forget gate | 이전 셀 상태를 얼마나 유지/삭제할지 (`c_t=f·c_{t-1}+…`) |
| 입력 게이트 | input gate | 새 후보 정보를 셀에 얼마나 쓸지 |
| 출력 게이트 | output gate | 셀 상태에서 무엇을 은닉 `h_t` 로 내보낼지 |
| 갱신/재설정 게이트 | update/reset gate | GRU 의 두 게이트(과거·새 정보 비율 / 과거 무시 정도) |
| 기울기 소실 | vanishing gradient | 반복 곱으로 먼 과거의 학습 신호가 0으로 사라짐 |
| 장기 의존성 | long-term dependency | 멀리 떨어진 입력들 사이의 관계(RNN 이 놓치기 쉬움) |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **덧셈 vs 곱셈.** RNN 은 상태를 `tanh(W·h)` 로 **곱해** 갱신하고 LSTM 은 셀을 `f·c + i·g̃` 로 **더해** 갱신합니다. 왜 "덧셈" 이 기울기 소실을 완화할까요? (힌트: `c_t` 를 `c_{t-1}` 로 미분하면 무엇이 남나 — `∂c_t/∂c_{t-1}=f_t`.)
> 2. **게이트가 하는 일.** forget 게이트가 항상 `1`, input 게이트가 항상 `0` 이면 LSTM 은 무엇을 하는 기계가 될까요? 반대로 forget 이 항상 `0` 이면? 각각 어떤 기억 특성을 갖나요?
> 3. **LSTM vs GRU.** GRU 는 셀 상태를 없애고 게이트를 2개로 줄였는데도 성능이 비슷한 경우가 많습니다. 파라미터가 적다는 것은 어떤 상황(데이터 크기·속도·과적합)에서 장점이 될까요? 반대로 LSTM 의 별도 셀 상태가 유리할 상황은?

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **LSTM·GRU 는 "곱셈으로 덮어쓰는" 바닐라 RNN([[Day-030]])의 기울기 소실을, "덧셈으로 갱신되는 기억 통로 + 학습형 게이트" 로 푼 순환망이다.** LSTM 의 심장은 셀 상태 갱신식 **`c_t = f_t·c_{t-1} + i_t·g̃_t`** 하나로, forget·input·output 세 게이트가 "무엇을 잊고·쓰고·내보낼지" 를 데이터로부터 학습한다. forget 게이트가 열려 있으면(`f≈1`) 셀 상태가 거의 그대로 흐르고, 역전파 기울기의 핵심 인자 `∏ f_t` 가 1 근처에 머물러 **먼 과거까지 학습 신호가 도달** 한다 — 실습에서 학습 없이도 RNN 은 첫 시점 기울기가 `e-3→e-34` 로 소멸한 반면 forget 을 연 LSTM 은 `e-1` 로 살아남음을 확인했다. **GRU** 는 셀 상태를 없애고 게이트를 update·reset **둘** 로 줄인 경량판으로(파라미터 RNN 304 < GRU 912 < LSTM 1216) 대개 비슷한 성능을 낸다. LSTM·GRU 는 **마지막 순환형 아키텍처** 이며, 순차 처리·정보 병목의 한계를 "모든 위치를 한 번에 보는" 방식으로 넘어서는 것이 다음 편 [[Day-032]] 의 **Attention**, 그리고 Transformer 다.
