## 0. 들어가며 (Motivation)

[[Day-028]] 의 XOR 학습 코드를 다시 떠올려 봅시다. 우리는 이렇게 썼습니다:

```python
W1 = torch.randn(2, H, requires_grad=True); b1 = torch.zeros(1, H, requires_grad=True)
W2 = torch.randn(H, 1, requires_grad=True); b2 = torch.zeros(1, 1, requires_grad=True)
params = [W1, b1, W2, b2]
...
for p in params:                    # grad 초기화
    if p.grad is not None: p.grad.zero_()
loss.backward()
with torch.no_grad():               # 갱신
    for p in params: p -= lr * p.grad
```

층이 2개라서 그나마 견딜 만했습니다. 하지만 **Transformer 는 층이 수십~수백 개** 이고, 각 층마다 가중치·편향이 여러 개입니다. 이걸 전부 손으로 `randn` 선언하고 `params` 리스트에 담고, `for` 문으로 초기화·갱신하면 — 실수 하나로 파라미터를 빠뜨리거나 갱신을 잊게 됩니다.

PyTorch 는 이 반복 작업을 **두 개의 표준 도구** 로 걷어냅니다. **`nn.Module`** 은 "모델이 어떤 파라미터를 갖는지" 를 자동으로 관리하고, **`torch.optim`** 은 "그 파라미터를 어떻게 갱신하는지" 를 자동으로 처리합니다. 오늘 이후 우리가 쓰는 모든 신경망은 이 골격 위에 올라갑니다.

> In [[Day-028]] we hand-declared every weight and looped to zero grads and update. That does not scale to a 50-layer Transformer. PyTorch removes this boilerplate with two standard tools: `nn.Module` (owns and registers parameters automatically) and `torch.optim` (owns the update rule). The loop collapses to `zero_grad() → backward() → step()`.

## 1. 핵심 개념 (Core Concepts)

### 1.1 `nn.Module` — 모델을 하나의 클래스로

`nn.Module` 은 모든 PyTorch 모델의 **부모 클래스** 입니다. 우리 모델을 `nn.Module` 을 상속한 클래스로 만들고, `__init__` 에서 **층(layer)** 을 만들고 `forward` 에서 **순전파** 를 적으면 끝입니다.

```python
class XORNet(nn.Module):
    def __init__(self, hidden=8):
        super().__init__()                 # 반드시 먼저! (파라미터 등록 장치를 켠다)
        self.fc1 = nn.Linear(2, hidden)    # Day-028 의 W1(2xH)+b1
        self.fc2 = nn.Linear(hidden, 1)    # Day-028 의 W2(Hx1)+b2

    def forward(self, x):
        h = torch.tanh(self.fc1(x))
        return self.fc2(h)                 # logit (sigmoid 전)
```

핵심은 **`nn.Linear(in, out)`** 입니다. 이 한 줄이 [[Day-028]] 의 `W = randn(in,out)` 과 `b = zeros(out)` 를 **자동으로 만들고 `requires_grad=True` 로 등록** 합니다. 즉 우리가 손으로 가중치를 선언할 필요가 사라집니다. ($y = xW + b$ 를 계산하는 **완전연결층(fully-connected / linear layer)** 입니다.)

### 1.2 `self.` 로 붙이면 자동 등록된다

`nn.Module` 의 마법은 **속성 등록** 에 있습니다. `self.fc1 = nn.Linear(...)` 처럼 층을 `self.` 로 붙여 두면, `nn.Module` 이 그 안의 모든 파라미터를 **자동으로 추적** 합니다. 그래서 우리는 [[Day-028]] 의 `params = [W1, b1, W2, b2]` 리스트를 손으로 만들 필요 없이, **`model.parameters()`** 한 번으로 전부 꺼낼 수 있습니다.

```python
for name, p in model.named_parameters():
    print(name, tuple(p.shape))
# fc1.weight (8, 2)   fc1.bias (8,)
# fc2.weight (1, 8)   fc2.bias (1,)
```

> [!warning] `super().__init__()` 을 빠뜨리면
> `__init__` 첫 줄의 `super().__init__()` 을 잊으면 파라미터 등록 장치가 켜지지 않아, `nn.Linear` 를 `self.` 로 붙여도 `model.parameters()` 가 **비어 있게** 됩니다. 그러면 최적화기가 갱신할 대상이 없어 학습이 전혀 안 됩니다. "nn.Module 서브클래스의 `__init__` 첫 줄은 항상 `super().__init__()`" 를 규칙으로 삼으세요.

### 1.3 `torch.optim` — 갱신 규칙을 최적화기에 위임

[[Day-028]] 에서 우리는 갱신을 손으로 `p -= lr * p.grad` 로 적었습니다. 이것이 바로 [[Day-014]]·[[Day-027]] 의 **경사하강(gradient descent)** 규칙 $\theta \leftarrow \theta - \eta\nabla$ 입니다. `torch.optim` 은 이 규칙을 **객체 하나에 위임** 합니다:

```python
optimizer = torch.optim.SGD(model.parameters(), lr=0.5)   # Day-028 과 정확히 같은 규칙
# 또는
optimizer = torch.optim.Adam(model.parameters(), lr=0.05) # 적응형 학습률 — 대체로 더 빠름
```

- **SGD** 는 우리가 [[Day-028]] 에서 손으로 짠 규칙과 **완전히 동일** 합니다 (`p -= lr*grad`).
- **Adam** 은 파라미터마다 학습률을 **자동 조절** 하는 개선판입니다. 실무 기본값으로 가장 널리 쓰이며, XOR 같은 작은 문제도 SGD 보다 빠르게 수렴합니다.

### 1.4 표준 학습 루프 — 세 줄 관용구

[[Day-028]] 의 세 관용구(grad 초기화 → backward → 갱신)가 최적화기로 이렇게 표준화됩니다:

| 단계 | [[Day-028]] (수동) | Day-029 (표준) |
|------|-------------------|----------------|
| grad 초기화 | `for p: p.grad.zero_()` | `optimizer.zero_grad()` |
| 역전파 | `loss.backward()` | `loss.backward()` (동일) |
| 파라미터 갱신 | `with no_grad(): p -= lr*p.grad` | `optimizer.step()` |

```python
for epoch in range(2000):
    logits = model(X)              # ① 순전파
    loss = loss_fn(logits, Y)      # ② 손실
    optimizer.zero_grad()          # ③ grad 초기화 (누적 방지 — Day-028 경고 그대로)
    loss.backward()                #    역전파 자동
    optimizer.step()               # ④ 갱신 위임
```

`optimizer` 는 생성 시 넘긴 `model.parameters()` 를 기억하고 있어서, `zero_grad()` 와 `step()` 이 **그 파라미터 전부** 에 한 번에 적용됩니다. 층이 100개여도 코드는 그대로 세 줄입니다.

### 1.5 `BCEWithLogitsLoss` — sigmoid 를 손실에 흡수

[[Day-028]] 에서는 `forward` 끝에 `sigmoid` 를 붙이고 `BCELoss` 를 썼습니다. 오늘은 `forward` 가 **logit(=sigmoid 전 값)** 을 그대로 내보내고, 손실로 **`nn.BCEWithLogitsLoss`** 를 씁니다. 이건 sigmoid + BCELoss 를 **하나로 합친** 것으로, 내부적으로 log-sum-exp 를 안정적으로 계산해 **수치적으로 더 안전** 합니다(아주 큰/작은 logit 에서 `log(0)` 폭주 방지). 실무에서 이진 분류의 표준 선택입니다. (추론 때만 `sigmoid` 를 씌워 확률로 읽습니다.)

### 1.6 `model.train()` / `model.eval()` 과 `state_dict`

- **`model.train()` / `model.eval()`**: 학습 모드·평가 모드를 전환합니다. Dropout·BatchNorm 같은 층은 두 모드에서 **다르게 동작** 하므로, 학습 땐 `train()`, 추론·평가 땐 `eval()` 을 부르는 습관이 중요합니다. (XOR MLP 엔 그런 층이 없어 결과는 같지만, 습관을 지금부터 들입니다.)
- **`state_dict()`**: 모델의 모든 파라미터를 이름→텐서 사전으로 반환합니다. `torch.save(model.state_dict(), "xor.pt")` 로 저장하고 `load_state_dict(...)` 로 복원 — **체크포인트·배포의 표준 통로** 입니다.

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 부품 상자(nn.Module)와 정비공(optimizer)
> [[Day-028]] 에서 우리는 자동차를 만들려고 **나사·볼트를 하나하나 봉투에 담아**(`params=[W1,b1,W2,b2]`) 관리하고, 정비도 **직접 렌치를 돌렸습니다**(`p -= lr*grad`). `nn.Module` 은 **부품이 조립된 엔진 블록** 입니다 — `nn.Linear` 라는 완제 부품을 끼우면 그 안의 나사(가중치)를 블록이 **알아서 목록화** 합니다(`model.parameters()`). `torch.optim` 은 **정비공** 입니다 — 엔진을 통째로 맡기면(`optimizer = Adam(model.parameters())`), "조여"(`step()`) 한 마디에 **모든 나사를 규칙대로** 조정합니다. 당신은 더 이상 나사를 세지도, 렌치를 직접 돌리지도 않습니다. 부품 상자에 부품을 끼우고(`__init__`), 조립 순서만 적고(`forward`), 정비공에게 "조여!" 라고만 하면 됩니다.
>
> `nn.Module` is a pre-assembled engine block that inventories its own screws (parameters); `torch.optim` is the mechanic you hand the whole engine to — one word (`step()`) and every screw is tightened by the rule. You stop counting screws and turning the wrench yourself.

## 3. 한 장의 그림 (One Picture)

```text
   Day-028 (수동)                         Day-029 (표준 골격)
   ─────────────────────────────         ─────────────────────────────
   W1,b1,W2,b2 = randn/zeros(...)   ==>   class Net(nn.Module):
   params = [W1,b1,W2,b2]                     self.fc1 = nn.Linear(2,8)   ← 가중치 자동 소유
                                              self.fc2 = nn.Linear(8,1)
                                          model = Net()
                                          model.parameters()  ← 전부 자동 수집

   학습 한 스텝:
     for p: p.grad.zero_()          ==>   optimizer.zero_grad()   ┐
     loss.backward()                ==>   loss.backward()         ├ 세 줄로 표준화
     with no_grad(): p -= lr*p.grad ==>   optimizer.step()        ┘
                                          (optimizer = Adam(model.parameters(), lr))

   ┌──────────── 결과는 동일: XOR 정확도 1.00 ────────────┐
   │  코드는 짧아지고, 층이 100개여도 루프는 그대로 세 줄  │
   └───────────────────────────────────────────────────────┘
```

> Same result (XOR accuracy 1.00), far less code. `nn.Module` auto-owns the parameters; `torch.optim` auto-updates them; the training step is always `zero_grad → backward → step`, no matter how deep the model.

### 4.2 1부 — nn.Module 로 캡슐화 (발췌)

```python
import torch
import torch.nn as nn

class XORNet(nn.Module):
    def __init__(self, hidden=8):
        super().__init__()
        self.fc1 = nn.Linear(2, hidden)
        self.fc2 = nn.Linear(hidden, 1)
    def forward(self, x):
        return self.fc2(torch.tanh(self.fc1(x)))  # logit

model = XORNet(hidden=8)
print(model)                                       # 구조 출력
for name, p in model.named_parameters():           # 파라미터 자동 수집
    print(name, tuple(p.shape))
```

### 4.3 2부 — torch.optim 표준 루프 (발췌)

```python
loss_fn = nn.BCEWithLogitsLoss()                    # sigmoid+BCE 를 하나로 (안정적)
optimizer = torch.optim.Adam(model.parameters(), lr=0.05)

for epoch in range(1, 2001):
    logits = model(X)
    loss = loss_fn(logits, Y)
    optimizer.zero_grad()                           # ③
    loss.backward()                                 #   ↓ 세 줄 관용구
    optimizer.step()                                # ④
```

실행하면 다음과 같은 형태로 출력됩니다 (시드 고정 `torch.manual_seed(7)` 으로 재현 가능):

### 4.4 3부 — nn.Sequential 로 클래스 없이

층을 단순히 **차례로 쌓기만** 한다면 클래스조차 필요 없습니다:

```python
seq = nn.Sequential(
    nn.Linear(2, 8),
    nn.Tanh(),
    nn.Linear(8, 1),
)
# 학습 루프(zero_grad -> backward -> step)는 2부와 완전히 동일
```

`nn.Sequential` 은 넘긴 층들을 **순서대로 통과** 시키는 컨테이너입니다. 분기·재사용이 없는 단순 적층 모델은 이 한 방식이 가장 간결합니다. (분기·조건이 있으면 1부처럼 `nn.Module` 클래스로 `forward` 를 직접 씁니다.)

### 4.5 (실험) 직접 흔들어 보기

- **SGD vs Adam**: `torch.optim.Adam(...)` 을 `torch.optim.SGD(model.parameters(), lr=0.5)` 로 바꿔 보세요. SGD 는 [[Day-028]] 의 수동 코드와 같은 규칙이라 수렴이 느려, 같은 2000 epoch 에서 loss 가 더 큽니다 — Adam 의 위력을 체감.
- **`super().__init__()` 지워 보기**: 1.2절 경고대로, 이 줄을 지우면 `model.parameters()` 가 비어 최적화기가 아무것도 갱신하지 못하고 loss 가 제자리입니다.
- **hidden 줄이기**: `XORNet(hidden=1)` 로 은닉 뉴런을 1개로 줄이면 XOR 를 못 풉니다([[Day-026]] 의 "비선형 경계엔 은닉층이 필요" 재확인).
- **저장·복원**: `torch.save(model.state_dict(), "xor.pt")` 로 저장 후, 새 `model2 = XORNet(); model2.load_state_dict(torch.load("xor.pt"))` 로 불러와 같은 예측이 나오는지 확인.

## 5. 용어집 (Glossary)

| 용어 | English | 한 줄 뜻 |
|------|---------|---------|
| 모듈 | nn.Module | 모든 PyTorch 모델의 부모 클래스 — 파라미터를 자동 등록·관리 |
| 선형층 | nn.Linear | $y=xW+b$ 를 계산하는 완전연결층 (가중치·편향 자동 생성) |
| 시퀀셜 | nn.Sequential | 층을 순서대로 통과시키는 컨테이너 (단순 적층용) |
| 최적화기 | optimizer | 기울기로 파라미터를 갱신하는 규칙을 캡슐화한 객체 |
| 확률적 경사하강 | SGD | `θ ← θ − η∇` — Day-028 의 수동 갱신과 동일한 규칙 |
| 아담 | Adam | 파라미터별 학습률을 적응적으로 조절하는 최적화기 (실무 기본값) |
| step | optimizer.step() | 모든 파라미터를 한 번에 규칙대로 갱신 |
| zero_grad | optimizer.zero_grad() | 모든 파라미터의 `.grad` 를 0으로 (누적 방지) |
| logit | logit | sigmoid/softmax 를 통과하기 *전* 의 원점수 |
| BCEWithLogitsLoss | BCEWithLogitsLoss | sigmoid+BCE 를 합친 수치적으로 안정적인 이진분류 손실 |
| 상태사전 | state_dict | 이름→파라미터 텐서 사전 — 저장·복원(체크포인트)의 통로 |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **왜 `self.` 로 붙여야 등록되나.** `nn.Linear` 를 지역 변수 `fc1 = nn.Linear(2,8)` 로 만들면 `model.parameters()` 에 안 잡힙니다. `nn.Module` 이 파라미터를 추적하는 방식(속성 대입을 가로채는 `__setattr__`)을 떠올리면, 왜 층을 리스트에 담을 땐 그냥 `[]` 대신 `nn.ModuleList` 를 써야 하는지도 자연스럽습니다 — 왜일까요?
> 2. **optimizer 는 무엇을 기억하나.** `Adam(model.parameters(), lr=0.05)` 를 만든 뒤 `optimizer.step()` 만 부르는데 어떻게 그 파라미터들을 갱신할까요? (힌트: 생성 시 파라미터 텐서들의 *참조* 를 붙들어 둡니다 — 그래서 `zero_grad`/`step` 이 model 없이도 동작.)
> 3. **SGD 와 손코딩의 관계.** [[Day-028]] 의 `p -= lr*p.grad` 와 `torch.optim.SGD(..., lr).step()` 은 정말 같은 일일까요? 같은 시드·같은 lr 로 두 방식을 각각 돌려 loss 곡선이 겹치는지 확인해 보면, 최적화기가 "마법" 이 아니라 우리가 손으로 하던 일을 **대신** 함을 확인할 수 있습니다.

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **PyTorch 의 표준 골격은 `nn.Module`(모델)과 `torch.optim`(갱신)이다.** 모델을 `nn.Module` 서브클래스로 만들고 `__init__` 에 `nn.Linear` 같은 층을 `self.` 로 붙이면, 가중치·편향이 `requires_grad=True` 로 **자동 생성·등록** 되어 [[Day-028]] 의 손선언(`W1,b1,W2,b2`)이 사라진다. `model.parameters()` 로 전부 꺼내 `torch.optim.SGD`(=Day-028 의 수동 규칙과 동일) 또는 `Adam`(적응형, 더 빠름)에 넘기면, 학습 루프가 **`optimizer.zero_grad() → loss.backward() → optimizer.step()`** 세 줄로 표준화된다(grad 는 여전히 누적되므로 `zero_grad` 필수). 손실은 sigmoid 를 흡수한 `BCEWithLogitsLoss` 로 수치 안정성을 얻고, `state_dict` 로 저장·복원한다. 같은 XOR 를 **더 짧은 코드로 정확도 1.00** 에 도달시켰다. 이 골격은 층이 수백 개인 모델에서도 그대로여서, 다음 편 [[Day-030]] 의 시퀀스 모델(RNN)과 Phase 3 마무리 🛠️ **B3 빌드(미니 Transformer)** 의 뼈대가 된다.
