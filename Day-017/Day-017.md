## 0. 들어가며 (Motivation)

[[Day-016]] 의 불린 검색은 "이 단어를 포함하는가/아닌가" 만 답했습니다. 하지만 `brilliant acting` AND 결과가 100개라면 — **어느 문서를 맨 위에 둬야 할까요?** 불린 검색에는 *순위* 가 없습니다. 이 질문이 오늘의 출발점입니다.

[[Day-015]] 에서 우리는 이미 한 가지 답을 봤습니다 — **TF-IDF 코사인 유사도**. 단어가 많이 나올수록(tf↑), 그리고 흔하지 않은 단어일수록(idf↑) 점수가 올라갔죠. 좋은 직관이지만 두 가지 빈틈이 있습니다.

1. **tf 가 선형으로 계속 오른다.** "great" 가 1번 나온 문서 대비 20번 나온 문서가 정말 20배 더 관련 있을까요? 직관적으로는 *아닙니다* — 처음 몇 번이 중요하고, 그 뒤로는 효용이 줄어듭니다(체감 효용 감소).
2. **긴 문서가 유리하다.** 문서가 길면 단어가 우연히 더 많이 등장합니다. 길이를 보정하지 않으면 장황한 문서가 부당하게 상위를 차지합니다.

**BM25**(Best Matching 25)는 바로 이 두 빈틈을 *명시적인 손잡이(parameter)* 로 메운, 1990년대 Okapi 시스템에서 나온 확률적 랭킹 함수입니다. 30년이 지난 지금도 Elasticsearch·Lucene 의 **기본 점수 함수** 이며, 최신 밀집 검색(dense retrieval)의 *강력한 베이스라인* 입니다.

> [!tip] 오늘의 한 발
> [[Day-015]] 가 "tf·idf 로 점수를 낼 수 있다" 였다면, 오늘은 "**현실적으로 잘 작동하는 점수**는 tf 를 *포화* 시키고 길이를 *정규화* 한다" 입니다. 표현(representation)에서 잘 다듬어진 **랭킹 함수(ranking function)** 로 한 칸 더.

## 1. 핵심 개념 (Core Concepts)

### 1.1 BM25 공식 — 전체 그림

질의 $Q = \{q_1, \dots, q_m\}$ 와 문서 $D$ 에 대해 BM25 점수는 **질의 단어별 점수의 합** 입니다:

$$ \text{BM25}(D, Q) = \sum_{q \in Q} \text{IDF}(q) \cdot \frac{f(q, D)\,(k_1 + 1)}{f(q, D) + k_1\Big(1 - b + b\,\dfrac{|D|}{\text{avgdl}}\Big)} $$

- $f(q, D)$ : 문서 $D$ 안에서 단어 $q$ 의 빈도(tf) — [[Day-016]] 포스팅의 `tf`.
- $|D|$ : 문서 $D$ 의 길이(단어 수), $\text{avgdl}$ : 컬렉션 평균 문서 길이.
- $\text{IDF}(q)$ : 단어 $q$ 의 역문서빈도(아래 1.4).
- $k_1, b$ : 두 개의 **튜닝 손잡이**(보통 $k_1 \in [1.2, 2.0]$, $b = 0.75$).

복잡해 보이지만, **세 조각** 으로 나눠 보면 각각이 0절의 빈틈 하나씩을 메웁니다. 하나씩 뜯어봅시다.

### 1.2 tf 포화 — $k_1$ 의 역할

tf 부분만 떼어내면(길이 정규화를 잠시 무시, 즉 $|D| = \text{avgdl}$):

$$ \frac{f \,(k_1 + 1)}{f + k_1} $$

이 함수는 $f$ 가 커질수록 **$k_1 + 1$ 에 점근(saturate)** 합니다. 즉 tf 가 1 → 2 → 3 으로 갈 때 점수는 크게 오르지만, 20 → 21 에서는 거의 안 오릅니다. **처음 몇 번의 등장이 가장 중요하고, 그 뒤는 체감 효용이 줄어든다** 는 직관을 수식으로 옮긴 것입니다.

- $k_1$ 이 **클수록** → 포화가 느려져 tf 에 더 민감(TF-IDF 의 선형에 가까워짐).
- $k_1 = 0$ 이면 → tf 가 1 이상이면 무조건 같은 점수(이진 BM, 등장 여부만 봄).

> [!tip] 한 줄
> $k_1$ 은 "tf 를 얼마나 믿을지" 의 손잡이다. TF-IDF 의 *무한정 선형 tf* 와 *등장 여부만 보는 이진* 사이를 매끄럽게 잇는다.

### 1.3 문서길이 정규화 — $b$ 의 역할

분모의 $k_1\big(1 - b + b\,\frac{|D|}{\text{avgdl}}\big)$ 가 길이를 보정합니다.

- 문서가 평균보다 **길면**($|D| > \text{avgdl}$) → 분모가 커져 → 점수가 **깎임**(긴 문서의 우연한 tf 이점 상쇄).
- 문서가 평균보다 **짧으면** → 분모가 작아져 → 점수가 **올라감**(짧은데도 단어가 나왔으면 더 의미 있음).
- $b = 1$ : 길이 정규화 **최대**. $b = 0$ : 정규화 **끔**(길이 무시).
- 보통 $b = 0.75$ — "어느 정도 보정하되 과하지 않게".

> [!warning] 왜 길이 보정이 필요한가
> 길이를 보정하지 않으면, 같은 주제를 짧고 정확히 쓴 문서보다 *장황하게 늘어놓은 문서* 가 단지 길다는 이유로 상위에 오릅니다. BM25 는 $b$ 로 이 편향을 끕니다.

### 1.4 IDF — [[Day-007]] idf 의 확률적 형제

BM25 의 idf 는 [[Day-007]] 의 idf 와 정신은 같지만(흔한 단어는 깎고 희소한 단어는 키운다) 식이 약간 다릅니다 — **확률적 IR** 에서 유도된 형태입니다:

$$ \text{IDF}(q) = \ln\!\left(1 + \frac{N - n(q) + 0.5}{n(q) + 0.5}\right) $$

- $N$ : 전체 문서 수, $n(q)$ : 단어 $q$ 를 포함한 문서 수(df, [[Day-016]] 의 `df`).
- `+0.5` 는 스무딩([[Day-011]]·[[Day-013]] 의 그 정신 — 극단값 방지), `+1` 은 음수 idf 방지용 보정.
- 의미: $q$ 가 **희소** 할수록($n(q)$ 작음) idf 가 크고, **흔할수록** 0 에 가까워집니다. `the` 처럼 거의 모든 문서에 든 단어는 idf ≈ 0 이라 점수 기여가 거의 없습니다 — [[Day-016]] 에서 본 "흔한 단어의 무력함" 이 *점수* 측면에서 재확인됩니다.

### 1.5 전체를 잇기 — 역색인에서 BM25 로

[[Day-016]] 의 흐름에 BM25 를 끼우면 완성형 검색이 됩니다:

```
질의 토큰화 → (역색인) 후보 추림 → 후보마다 BM25 점수 합산 → 상위 k개 정렬·반환
```

핵심은 **BM25 가 후보 전체가 아니라 질의 단어의 포스팅만 훑으며 계산된다** 는 점입니다. 단어별로 그 포스팅(=그 단어가 든 문서들)을 돌며 각 문서의 누적 점수에 기여분을 더하면 됩니다(term-at-a-time). [[Day-016]] 에서 `tf`·`df`·`문서길이` 를 미리 저장해 둔 것이 바로 이걸 위해서였습니다.

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 짠맛의 체감과 큰 접시
> 요리에 소금을 넣는다고 합시다. **처음 한 꼬집** 은 맛을 확 살립니다. 두 꼬집도 좋습니다. 그런데 열 꼬집째부터는? 더 짜지긴 해도 "맛있어지는 정도" 는 거의 안 늘고 오히려 물립니다 — 이것이 **tf 포화**($k_1$): 같은 단어의 반복은 처음 몇 번이 결정적이고 그 뒤는 효용이 준다.
>
> 또 같은 양의 소금이라도 **작은 접시** 에 넣으면 간이 딱 맞고, **거대한 접시** 에 넣으면 싱겁습니다. 그래서 큰 접시는 "소금이 있긴 한데 농도가 옅네" 라고 점수를 깎아야 공정합니다 — 이것이 **문서길이 정규화**($b$): 긴 문서에 단어가 우연히 많이 든 것을 농도(밀도)로 환산해 보정한다.
>
> First pinch of salt matters most (tf saturation, $k_1$); the same salt tastes weaker in a huge dish (length normalization, $b$). BM25 = TF-IDF that respects both diminishing returns and document size.

## 3. 한 장의 그림 (One Picture)

```text
   BM25(D,Q) = Σ over q in Q  of   IDF(q) × [ tf 포화·길이정규화 항 ]

            IDF(q)                tf 포화 + 길이 정규화
       (희소할수록 큼)        (처음 몇 번이 중요, 긴 문서는 깎임)
   ┌────────────────────┐   ┌──────────────────────────────────────┐
   │ ln(1 + (N-n+0.5)    │   │        f(q,D) · (k1+1)                │
   │          ───────── )│ × │ ──────────────────────────────────── │
   │           n+0.5     │   │  f(q,D) + k1·(1 - b + b·|D|/avgdl)    │
   └────────────────────┘   └──────────────────────────────────────┘
            │                          │            │
        Day-007 idf 의            tf 포화(k1)   길이 정규화(b)
        확률적 형제               1·2·3 은 크게,  긴 문서 = 분모↑
                                  20·21 은 거의 동일  → 점수↓

   tf 점수 곡선:
     점수 ▲           ___________  ← (k1+1) 에 점근(포화)
          │        _/
          │      _/
          │    _/
          │  _/
          │_/
          └──────────────────────►  tf (단어 등장 횟수)
            1  2  3 ...        20 21   (처음 몇 번이 가장 중요)
```

> Each query term contributes `IDF × (saturated, length-normalized tf)`. Sum over query terms; sort candidates descending.

### 4.2 BM25 를 직접 만들기 — `bm25.py`

아래를 **`bm25.py`** 로 저장합니다(이 노트 옆에 함께 제공). [[Day-016]] 의 같은 영화 리뷰 6편에 **역색인을 짓고 그 위에 BM25 랭킹** 을 얹어, 1절의 수식이 그대로 코드가 되는 것을 확인합니다. 마지막엔 $k_1$·$b$ 를 흔들어 tf 포화·길이 정규화의 효과를 *숫자로* 봅니다.

```python
# bm25.py — Day-017: 역색인 위에 BM25 랭킹 from scratch (표준 라이브러리만)
# 실행: uv run python bm25.py
import sys, math
from collections import defaultdict, Counter
sys.stdout.reconfigure(encoding="utf-8")  # Windows 콘솔 한글 깨짐 방지

# 0) Day-016 기반 컬렉션 — 영화 리뷰 6편.
#    'brilliant' 은 doc0 에만 등장(희소→idf 큼), doc5 는 'acting' 만 도배(tf 폭증).
#    이 설계로 "tf 도배가 1등을 보장하지 않는다"(tf 포화)를 숫자로 보여준다.
corpus = [
    "the movie had brilliant acting and a wonderful story",        # doc0  brilliant1 acting1
    "a boring and dull film with terrible acting",                 # doc1  acting1
    "an amazing soundtrack made this film wonderful",              # doc2
    "the plot was confusing but the visuals were truly amazing",   # doc3
    "i loved the wonderful music and the great cast",              # doc4
    "acting acting acting acting acting shown in this film",       # doc5  acting5 (도배)
]

def tokenize(text):
    return text.lower().split()   # Day-006 의 최소 토큰화

# 1) 색인 시간: 역색인 + 통계(df, 문서길이, avgdl) — Day-016 의 산물 재사용
class BM25Index:
    def __init__(self, k1=1.5, b=0.75):
        self.k1, self.b = k1, b
    def build(self, docs):
        self.N = len(docs)
        self.doc_len = [len(tokenize(d)) for d in docs]
        self.avgdl = sum(self.doc_len) / self.N
        postings = defaultdict(dict)                 # term -> {doc_id: tf}
        for doc_id, text in enumerate(docs):
            for term, tf in Counter(tokenize(text)).items():
                postings[term][doc_id] = tf
        self.postings = postings
        self.df = {t: len(p) for t, p in postings.items()}
        return self
    def idf(self, term):                              # 확률적 IDF (1.4)
        n = self.df.get(term, 0)
        return math.log(1 + (self.N - n + 0.5) / (n + 0.5))
    def score(self, query, doc_id):                  # BM25(D, Q) (1.1)
        s = 0.0
        dl = self.doc_len[doc_id]
        for term in tokenize(query):
            if doc_id not in self.postings.get(term, {}):
                continue                              # 그 문서에 없는 단어는 0 기여
            tf = self.postings[term][doc_id]
            denom = tf + self.k1 * (1 - self.b + self.b * dl / self.avgdl)
            s += self.idf(term) * (tf * (self.k1 + 1)) / denom
        return s
    def search(self, query, top_k=None):             # 역색인으로 후보만 채점 (1.5)
        cand = set()
        for term in tokenize(query):
            cand.update(self.postings.get(term, {}).keys())
        ranked = sorted(((self.score(query, d), d) for d in cand), reverse=True)
        return ranked if top_k is None else ranked[:top_k]

idx = BM25Index(k1=1.5, b=0.75).build(corpus)
print(f"N={idx.N}  avgdl={idx.avgdl:.2f}  문서길이={idx.doc_len}")
print(f"idf(brilliant)={idx.idf('brilliant'):.3f}  "
      f"idf(acting)={idx.idf('acting'):.3f}  idf(the)={idx.idf('the'):.3f}")

print("\n=== BM25 랭킹 (k1=1.5, b=0.75) ===")
for q in ["brilliant acting", "wonderful amazing", "the film"]:
    print(f"\n[질의] {q!r}")
    for rank, (sc, d) in enumerate(idx.search(q), 1):
        print(f"  {rank}. doc{d}  score={sc:.4f}  | {corpus[d]}")

# ── tf 포화 직관: acting 의 tf 가 5인 doc5 vs tf 1인 doc1 ──
print("\n=== tf 포화(saturation) 확인: 질의 'acting' ===")
single = BM25Index(k1=1.5, b=0.0).build(corpus)   # b=0 으로 길이효과 제거, tf효과만
for d in [1, 5]:
    tf = single.postings["acting"][d]
    print(f"  doc{d}: tf(acting)={tf} → score={single.score('acting', d):.4f}")
print("  → tf가 5배(1→5)라도 점수는 5배가 아니다(포화). k1을 키우면 격차가 커진다.")

# ── 손잡이 흔들기: k1, b 를 바꾸면 순위가 어떻게 변하나 ──
print("\n=== 손잡이(k1,b) 민감도: 질의 'acting brilliant' ===")
for k1, b in [(0.0, 0.75), (1.5, 0.0), (1.5, 0.75), (2.0, 0.75)]:
    rk = BM25Index(k1=k1, b=b).build(corpus).search("acting brilliant", top_k=3)
    order = " > ".join(f"doc{d}({sc:.2f})" for sc, d in rk)
    print(f"  k1={k1}, b={b:<4}: {order}")
```

```
  4. doc0  score=0.6814  | the movie had brilliant acting and a wonderful story

[질의] 'the film'
  1. doc4  score=0.9781  | i loved the wonderful music and the great cast
  2. doc3  score=0.9436  | the plot was confusing but the visuals were truly amazing
  3. doc2  score=0.7588  | an amazing soundtrack made this film wonderful
  ...

=== tf 포화(saturation) 확인: 질의 'acting' ===
  doc1: tf(acting)=1 → score=0.6931
  doc5: tf(acting)=5 → score=1.3330
  → tf가 5배(1→5)라도 점수는 5배가 아니다(포화). k1을 키우면 격차가 커진다.

=== 손잡이(k1,b) 민감도: 질의 'acting brilliant' ===
  k1=0.0, b=0.75: doc0(2.23) > doc5(0.69) > doc1(0.69)   ← k1=0이면 tf 무시, 도배 무력
  k1=1.5, b=0.0 : doc0(2.23) > doc5(1.33) > doc1(0.69)
  k1=1.5, b=0.75: doc0(2.20) > doc5(1.32) > doc1(0.72)
  k1=2.0, b=0.75: doc0(2.19) > doc5(1.47) > doc1(0.72)
```

> 부동소수점 표기 차이를 빼면 위 수치 그대로 재현됩니다. **순위와 경향이 핵심** 입니다.

### 4.3 결과를 정직하게 읽기 (관찰 포인트)

1. **tf 폭증이 1등을 보장하지 않는다.** doc5 는 `acting` 을 5번이나 쓰지만, `brilliant acting` 질의에서 **doc0(각 1번)에 밀립니다**(1.32 vs 2.20). 두 가지가 겹쳐 일어난 결과입니다 — (a) tf 포화($k_1$)가 `acting` 반복의 효용을 깎았고, (b) doc0 은 *희소한* `brilliant`(idf=1.54)까지 함께 가져 점수를 크게 받았습니다. *한 단어 도배보다 질의어를 고르게 덮는 문서가 이긴다* — 키워드 도배(keyword stuffing)에 강한 BM25 의 핵심 미덕입니다.
2. **tf 5배 ≠ 점수 5배.** `acting` 단독 질의에서 doc5(tf=5, 0.6931→1.3330)는 doc1(tf=1)의 약 **1.9배** 점수일 뿐 5배가 아닙니다 — tf 포화를 *숫자로* 확인.
3. **$k_1 = 0$ 이면 도배가 무력화된다.** $k_1 = 0$ 일 때 `acting brilliant` 에서 doc5(0.69)가 doc1(0.69)과 **같아집니다**(등장 여부만 봄, tf 완전 무시) — 반면 doc0 은 희소어 `brilliant` 덕에 2.23 으로 굳건합니다.
4. **$b$ 의 효과는 길이 편차에 비례한다.** 이 코퍼스는 문서 길이(7~10)가 고르게 모여 있어 $b=0$↔$0.75$ 의 순위 변화가 작습니다(doc5 1.33→1.32). 길이가 *크게* 다른 코퍼스라면 효과가 또렷해집니다 — 4.4 의 $b$ 스윕으로 직접 확인하세요.
5. **작은 코퍼스에서는 `the` 도 idf 가 0 이 아니다.** `the film` 에서 `the`(df=3)와 `film`(df=3)의 idf 가 **둘 다 0.693 으로 같아**, `the`×2 를 가진 doc4·doc3 가 `film`×1 문서들을 앞섭니다. *진짜* 대규모 코퍼스라면 `the` 의 idf≈0 이 되어([[Day-016]] "흔한 단어의 무력함") 이런 역전이 사라집니다 — **불용어 제거([[Day-006]])와 큰 코퍼스가 왜 중요한지** 보여주는 소규모 코퍼스의 *artifact* 입니다.

### 4.4 (실험) 직접 흔들어 보기

- **$k_1$ 스윕**: `k1` 을 `0, 0.5, 1.2, 2.0, 5.0` 으로 바꿔 `acting` 단독 질의에서 doc5/doc1 점수비가 어떻게 변하는지 표로 만들어 보세요. $k_1$↑ → tf 에 민감(포화 느림).
- **$b$ 스윕**: `b` 를 `0, 0.5, 1.0` 으로 바꿔 가장 긴 문서의 순위 변동을 관찰하세요.
- **rank-bm25 와 대조**: `uv add rank-bm25` 후 `BM25Okapi` 로 같은 코퍼스를 돌려 우리 from-scratch 점수와 *경향이 일치* 하는지 검증하세요(재현성 — [[Day-012]] Pass 3 정신).
- **[[Day-015]] TF-IDF 와 비교**: 같은 질의로 TF-IDF 코사인 순위와 BM25 순위를 나란히 두고, 어디서 갈리는지(특히 tf 도배 문서 doc5)를 분석하세요.

## 5. 용어집 (Glossary)

| 용어 | English | 한 줄 뜻 |
|------|---------|---------|
| BM25 | Best Matching 25 | 확률적 IR 에서 나온 사실상 표준 랭킹 함수 |
| 랭킹 | ranking | 후보 문서를 관련도 점수로 줄 세우는 일 |
| tf 포화 | term-frequency saturation | tf 가 커져도 점수가 무한히 안 오르고 점근함($k_1$) |
| 문서길이 정규화 | document-length normalization | 긴 문서의 우연한 tf 이점을 깎음($b$) |
| avgdl | average document length | 컬렉션 평균 문서 길이(정규화 기준) |
| IDF (BM25) | inverse document frequency | $\ln(1+\frac{N-n+0.5}{n+0.5})$, 희소 단어를 키움 |
| $k_1$ | k1 parameter | tf 민감도 손잡이(보통 1.2–2.0) |
| $b$ | b parameter | 길이 정규화 강도(보통 0.75, 0=끔/1=최대) |
| term-at-a-time | term-at-a-time | 단어별 포스팅을 돌며 점수를 누적하는 채점 방식 |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **왜 tf 를 포화시키나**: 단어가 100번 나온 문서가 1번 나온 문서보다 정말 100배 관련 있을까요? 포화가 *없는* TF-IDF([[Day-015]])는 어떤 문서에 취약할까요? (힌트: 키워드 도배.)
> 2. **$b$ 의 양면**: 길이 정규화를 강하게($b=1$) 걸면 짧은 문서가 유리해집니다. 어떤 검색(예: 트윗 vs 논문)에서 $b$ 를 낮추거나 높여야 할까요?
> 3. **BM25 의 맹점**: BM25 는 결국 *단어가 겹치는지* 만 봅니다(lexical match). "car" 로 검색했을 때 "automobile" 만 든 문서를 못 찾습니다 — 이 **어휘 불일치(vocabulary mismatch)** 문제가 Phase 5 의 **밀집 검색(dense retrieval)·임베딩** 으로 가는 동기입니다. BM25 가 여전히 강력한 *베이스라인* 인 이유는 무엇일까요?

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **BM25** 는 [[Day-016]] 역색인으로 추린 후보를 줄 세우는, IR 의 사실상 표준 랭킹 함수다. 질의 단어마다 **IDF × (포화·길이정규화된 tf)** 를 더해 점수를 낸다. 핵심 손잡이는 둘 — **$k_1$**(tf 포화: 반복의 효용을 깎아 키워드 도배를 막음)과 **$b$**(문서길이 정규화: 긴 문서의 우연한 이점을 상쇄). IDF 는 [[Day-007]] idf 의 확률적 형제로 희소 단어를 키우고 흔한 단어(`the`)를 0 으로 깎는다. [[Day-016]] 에 저장해 둔 `tf`·`df`·`문서길이` 가 그대로 입력이 되어, "역색인으로 좁히고 → BM25 로 줄 세운다" 는 미니 검색엔진의 골격이 완성된다. 단, BM25 는 *어휘가 겹쳐야* 작동하는 한계가 있어(→ Phase 5 밀집 검색의 동기), 다음은 이 순위를 *평가* 하는 지표(MAP·nDCG·MRR)로 넘어간다.
