> [!abstract] 🎯 오늘의 목표
> 지금까지의 MLP([[Day-026]])는 입력을 **한 덩어리(고정 크기 벡터)** 로 받았습니다. 하지만 문장·시계열·음성은 **길이가 제각각** 이고 **순서가 의미를 바꿉니다**("개가 사람을 물다" ≠ "사람이 개를 물다"). 오늘은 이런 **시퀀스(sequence)** 를 다루는 첫 신경망, **순환 신경망(RNN, Recurrent Neural Network)** 을 배웁니다. 핵심은 단 하나의 순환식 **`h_t = tanh(Wₓₕ·x_t + Wₕₕ·h_{t-1} + b)`** — 매 시점 **같은 가중치** 를 재사용하며, 직전 상태 `h_{t-1}`(=지금까지의 요약 기억)을 다음 단계로 넘깁니다. [[Day-029]] 의 `nn.Module`·`torch.optim` 골격 위에 `nn.RNN` 을 얹어, "**첫 비트를 끝까지 기억하기**" 과제를 정확도 1.00 으로 학습시켜 봅니다. 이 hidden state 아이디어가 이후 LSTM·Attention·Transformer([[Phase-3]])로 이어집니다.

## 0. 들어가며 (Motivation)

[[Day-026]]~[[Day-029]] 에서 만든 MLP 는 입력이 **고정 크기 벡터** 였습니다 — XOR 는 항상 `[x1, x2]` 2차원. 하지만 우리가 궁극적으로 다룰 **텍스트** 는 그렇지 않습니다:

- **길이가 다르다.** "안녕"은 2글자, 이 문장은 수십 글자. MLP 의 입력층은 크기가 고정이라 곤란합니다.
- **순서가 의미를 만든다.** [[Day-007]] 의 Bag-of-Words 는 단어를 **세기만** 해서 "개가 사람을 물다"와 "사람이 개를 물다"를 **구별하지 못했습니다**. 순서 정보를 버렸기 때문입니다.

그래서 필요한 건 **입력을 한 스텝씩 순서대로 읽으면서, 지금까지 본 것을 요약한 "기억"을 이어가는** 구조입니다. 사람이 문장을 왼쪽에서 오른쪽으로 읽으며 머릿속에 맥락을 쌓아가듯이. 이 "이어지는 기억"을 **hidden state** 라 부르고, 그것을 매 스텝 갱신하는 신경망이 바로 **RNN** 입니다.

> An MLP takes one fixed-size vector; text does not fit — sequences vary in length and **order changes meaning** (Bag-of-Words threw order away, [[Day-007]]). We need a network that reads one step at a time while carrying a running summary — the **hidden state**. That network is the **RNN**.

## 1. 핵심 개념 (Core Concepts)

### 1.1 시퀀스 데이터와 hidden state

**시퀀스(sequence)** 는 `x_1, x_2, …, x_T` 처럼 순서가 있는 입력의 나열입니다(단어들, 시각별 주가, 음성 프레임…). RNN 은 이걸 **한 시점씩** 읽으며, 매 시점 **hidden state** `h_t` 를 갱신합니다. `h_t` 는 "**시점 t 까지 본 내용의 요약**" 이라고 생각하면 됩니다. 마지막 `h_T` 는 시퀀스 전체를 압축한 벡터가 되어, 분류·예측의 입력으로 쓰입니다.

### 1.2 RNN 의 순환식 — 오늘의 한 줄 심장

$$h_t = \tanh(W_{xh}\,x_t + W_{hh}\,h_{t-1} + b), \qquad h_0 = \mathbf{0}$$

- `x_t` : 현재 시점의 입력 (예: 지금 단어의 임베딩)
- `h_{t-1}` : **직전** 시점의 hidden state (= 지금까지의 기억)
- `W_{xh}` : 입력을 hidden 으로 (입력 → 기억)
- `W_{hh}` : **기억을 다음 기억으로** — RNN 을 RNN 이게 하는, 자기 자신으로 도는 가중치
- `tanh` : 비선형성([[Day-026]]), 상태를 `[-1, 1]` 로 눌러 폭주 방지

핵심은 `h_{t-1}` 이 다시 입력으로 들어간다는 **순환(recurrence)** 입니다. 그래서 `h_t` 안에는 `h_{t-1}` 이, 그 안에는 `h_{t-2}` 가… **과거 전체가 재귀적으로 포개져** 있습니다.

### 1.3 파라미터 공유 (parameter sharing) — 왜 중요한가

결정적으로, **`W_{xh}`, `W_{hh}`, `b` 는 모든 시점에서 똑같은 하나** 입니다. 시점마다 새 가중치를 쓰는 게 아닙니다. 이 **파라미터 공유** 덕분에:

1. **길이에 무관** — 시퀀스가 3스텝이든 300스텝이든 파라미터 수는 그대로. 어떤 길이든 처리합니다.
2. **위치 불변 패턴 학습** — "부정 표현이 나오면 기억을 뒤집는다" 같은 규칙을 **위치와 상관없이** 한 번 배우면 어디서든 적용됩니다. ([[Day-009]] Word2Vec, CNN 의 가중치 공유와 같은 철학.)

### 1.4 시간축으로 펼치기(unrolling)와 BPTT

RNN 은 그림상 "자기 자신으로 도는 고리" 지만, 학습할 땐 시간축을 따라 **쭉 펼쳐(unroll)** `h_1→h_2→…→h_T` 로 이어진 **깊은 하나의 계산 그래프** 로 봅니다. 그러면 [[Day-027]]·[[Day-028]] 의 **역전파를 그대로** 적용할 수 있는데, 시간축을 거슬러 흐른다는 뜻에서 이를 **BPTT(Backpropagation Through Time, 시간펼침 역전파)** 라 부릅니다. PyTorch 의 autograd([[Day-028]])가 `loss.backward()` 한 줄로 알아서 해 줍니다.

### 1.5 `nn.RNN` — PyTorch 표준 부품

[[Day-029]] 의 `nn.Linear` 처럼, PyTorch 는 위 순환식을 `nn.RNN` 한 줄로 제공합니다:

```python
self.rnn = nn.RNN(input_size=1, hidden_size=8, batch_first=True)
out, h_n = self.rnn(x)   # x:(N, L, input) → out:(N, L, H), h_n:(1, N, H)
```

- `out` : **모든 시점의** hidden state `h_1…h_L` (시퀀스 라벨링·언어모델용)
- `h_n` : **마지막** hidden state `h_L` (= `out[:, -1, :]`) — 문장 분류처럼 "전체 요약 하나" 가 필요할 때
- `batch_first=True` : 입력 모양을 `(배치, 길이, 특성)` 순으로 (직관적이라 권장)

### 1.6 vanishing/exploding gradient — 다음 편으로 가는 문턱

순환식이 `W_{hh}` 를 **매 시점 반복 곱** 하기 때문에, 시퀀스가 길면 BPTT 의 기울기가 뒤로 갈수록 **0으로 사라지거나(vanishing)** **폭발(exploding)** 합니다. 그 결과 바닐라 RNN 은 **먼 과거를 오래 기억하지 못합니다**(장기 의존성 학습 실패). 이 한계를 **게이트(gate)** 로 푸는 것이 다음 편 [[Day-031]] 의 **LSTM·GRU** 입니다.

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 한 줄씩 읽으며 적어 두는 '메모지 한 장'
> RNN 은 책을 읽는 사람과 같습니다. 단, 손에 든 것은 **딱 한 장짜리 메모지(hidden state `h`)** 뿐입니다. 한 단어(`x_t`)를 읽을 때마다, **직전 메모(`h_{t-1}`)를 보고** 새 단어를 반영해 **메모를 고쳐 씁니다**(`h_t`). 페이지를 넘겨도 규칙(펜을 쥔 손, 즉 `W_{hh}`)은 **바뀌지 않습니다**(파라미터 공유). 다 읽고 나면 그 마지막 메모 한 장(`h_T`)에 줄거리가 요약돼 있어, 그걸 보고 "이 글은 긍정/부정" 같은 판단을 내립니다. 문제는 메모지가 **한 장뿐** 이라, 뒤로 갈수록 앞부분이 덧칠돼 **희미해진다**(vanishing gradient)는 것 — 그래서 [[Day-031]] 에서 "무엇을 지우고 무엇을 남길지" 정하는 **게이트** 가 등장합니다.
>
> An RNN is a reader with a single sticky note (`h`): each new word rewrites the note using the previous note; the pen (the shared `W_{hh}`) never changes; the final note summarizes the whole text. But one note fades as it is overwritten — hence gates in [[Day-031]].

## 3. 한 장의 그림 (One Picture)

```text
   MLP (Day-026~029)                RNN (Day-030) — 시간축으로 펼침(unroll)
   ───────────────                  ───────────────────────────────────────────
   x ─▶ [Linear→tanh→Linear] ─▶ y     x_1     x_2     x_3          x_L
        (입력은 고정크기 한 덩어리)    │       │       │            │
                                       ▼       ▼       ▼            ▼
                              h_0 ─▶ (RNN) ─▶ (RNN) ─▶ (RNN) ─▶ … ─▶ (RNN) ─▶ h_L ─▶ [head] ─▶ y
                                       ▲       ▲       ▲            ▲
                                    같은 Wxh,Whh,b 를 매 시점 재사용 (파라미터 공유)

   순환식:  h_t = tanh(Wxh·x_t + Whh·h_{t-1} + b)
            h_t 안에는 h_{t-1} 이, 그 안에 h_{t-2} 가 … 과거가 포개져 있다.
   ┌─────────────────────────────────────────────────────────────────────────┐
   │ 마지막 기억 h_L 하나가 시퀀스 전체의 요약 → 분류·예측에 사용            │
   └─────────────────────────────────────────────────────────────────────────┘
```

> Unrolled through time, an RNN is a deep chain that reuses the **same** weights at every step; the final hidden state `h_L` summarizes the whole sequence.

### 4.2 1부 — 순환식을 손으로 펼치기 (발췌)

```python
import torch
torch.manual_seed(0)
H, D = 4, 1
Wxh = torch.randn(H, D) * 0.5    # 입력 -> hidden
Whh = torch.randn(H, H) * 0.5    # hidden -> hidden (여기로 '기억'이 흐른다)
bh  = torch.zeros(H)

def rnn_step(x_t, h_prev):
    return torch.tanh(Wxh @ x_t + Whh @ h_prev + bh)   # 오늘의 순환식 그대로

seq = [torch.tensor([1.]), torch.tensor([0.]), torch.tensor([1.])]
h = torch.zeros(H)               # h_0 : 빈 기억
for t, x_t in enumerate(seq, 1):
    h = rnn_step(x_t, h)         # h_{t-1} 을 다시 넣는다 = 순환
    print(f"x_{t}={x_t.item():.0f} -> h_{t} =", [round(v,3) for v in h.tolist()])
```

출력(시드 고정 `torch.manual_seed(0)`):

```
  h_0 = [0.0, 0.0, 0.0, 0.0]  (초기 상태, 아직 아무것도 안 봄)
  x_1=1 -> h_1 = [0.647, -0.146, -0.797, 0.277]
  x_2=0 -> h_2 = [-0.355, 0.306, -0.393, -0.344]
  x_3=1 -> h_3 = [0.685, -0.066, -0.893, 0.31]
```

`x_1` 과 `x_3` 은 똑같이 `1` 인데도 `h_1 ≠ h_3` 입니다 — `h_3` 은 "그 전에 `1, 0` 을 봤다" 는 **맥락(=`h_2`)** 을 반영하기 때문입니다. 이것이 hidden state 가 "기억" 이라는 증거입니다.

### 4.3 2부 — nn.RNN 으로 '첫 비트 기억하기' (발췌)

과제: 길이 `L=6` 의 0/1 시퀀스를 주고, **첫 번째 비트** 를 맞히게 합니다. RNN 은 첫 비트를 hidden state 에 담아 **6스텝 뒤까지 실어 날라야** 정답을 냅니다.

```python
class FirstBitRNN(nn.Module):
    def __init__(self, hidden=8):
        super().__init__()                                 # Day-029 규칙
        self.rnn  = nn.RNN(1, hidden, batch_first=True)
        self.head = nn.Linear(hidden, 1)
    def forward(self, x):
        out, h_n = self.rnn(x)          # out:(N,L,H)
        return self.head(out[:, -1, :]).squeeze(1)         # 마지막 h_L -> logit

loss_fn = nn.BCEWithLogitsLoss()                            # Day-029 안정판
opt = torch.optim.Adam(model.parameters(), lr=0.01)
for epoch in range(1, 301):
    loss = loss_fn(model(Xtr), ytr)
    opt.zero_grad(); loss.backward(); opt.step()           # ← Day-029 세 줄 그대로
```

실행 결과(재현 가능, `torch.manual_seed(7)`):

```
  epoch   1 | loss 0.7027 | test acc 0.535   ← 처음엔 동전 던지기 수준
  epoch  50 | loss 0.0224 | test acc 1.000   ← 첫 비트를 '기억'하기 시작
  epoch 300 | loss 0.0014 | test acc 1.000
```

정확도가 `0.5`(찍기)에서 `1.00` 으로 오른다는 건, RNN 이 **첫 비트를 6스텝 동안 hidden state 로 보존** 하는 법을 학습했다는 뜻입니다. 학습 루프는 [[Day-029]] 의 세 줄 관용구(`zero_grad → backward → step`)와 손실(`BCEWithLogitsLoss`)을 **그대로** 썼다는 점에 주목하세요 — 모델만 `nn.RNN` 으로 바꿨을 뿐입니다.

### 4.4 3부 — 순서가 의미를 바꾼다 (발췌)

```python
a = torch.tensor([[[1.],[0.],[0.],[1.],[1.],[0.]]])   # 첫 비트 = 1
b = a.clone(); b[0,0,0] = 0.0                         # 뒤는 동일, 첫 비트만 0
# p(첫비트=1):  a -> 0.999 (예측 1) ,  b -> 0.001 (예측 0)
```

뒤쪽 5개가 똑같아도 **첫 원소 하나** 만 바뀌면 결과가 뒤집힙니다 — 모델이 **순서(위치)** 를 실제로 쓰고 있다는 확증입니다.

### 4.5 (실험) 직접 흔들어 보기

- **길이 늘리기**: `L=6` 을 `L=20`, `L=50` 으로 키워 보세요. 길이가 길수록 학습이 느려지고 정확도가 흔들립니다 — 1.6절의 **vanishing gradient**(먼 과거 기억의 한계)를 몸으로 느낄 수 있습니다. [[Day-031]] 의 LSTM 예고편.
- **마지막 대신 평균**: `out[:, -1, :]`(마지막 상태) 대신 `out.mean(dim=1)`(전체 평균)을 head 에 넣어 보세요. "첫 비트" 과제에선 성능이 떨어질 수 있습니다 — 순서 정보를 뭉개기 때문.
- **hidden 크기**: `hidden=8` 을 `2`, `1` 로 줄이면 기억 용량이 줄어 수렴이 어려워집니다.
- **직접 셀 짜기**: 1부의 `rnn_step` 을 `requires_grad=True` 파라미터로 만들고 [[Day-028]] 처럼 학습 루프를 돌려, `nn.RNN` 없이 같은 과제를 풀어 보세요(BPTT 를 손으로 체험).

## 5. 용어집 (Glossary)

| 용어 | English | 한 줄 뜻 |
|------|---------|---------|
| 순환 신경망 | RNN | 시퀀스를 한 스텝씩 읽으며 hidden state 를 갱신하는 신경망 |
| 시퀀스 | sequence | 순서가 있는 입력의 나열 `x_1…x_T` (단어·시계열 등) |
| 은닉 상태 | hidden state | 시점 t 까지 본 내용을 요약한 벡터 `h_t` — RNN 의 "기억" |
| 순환식 | recurrence | `h_t = tanh(Wxh·x_t + Whh·h_{t-1} + b)` — RNN 의 심장 |
| 파라미터 공유 | parameter sharing | 모든 시점에서 같은 `Wxh, Whh, b` 를 재사용 (길이 무관) |
| 펼치기 | unrolling | 순환 고리를 시간축으로 늘여 깊은 계산 그래프로 보기 |
| 시간펼침 역전파 | BPTT | 펼친 그래프에 역전파를 적용해 기울기를 시간축으로 전파 |
| 기울기 소실/폭발 | vanishing/exploding gradient | `Whh` 반복곱으로 먼 과거 기울기가 사라지거나 폭주 |
| nn.RNN | nn.RNN | 위 순환식을 제공하는 PyTorch 표준 층 |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **왜 MLP 로는 안 되나.** 길이가 다른 문장들을 MLP 에 넣으려면 어떤 문제가 생기나요? (힌트: 입력층 크기가 고정) 그리고 "패딩해서 길이를 맞추면 되지 않나?" 에 대한 반론은? (순서·위치 정보와 파라미터 수)
> 2. **`h_t` 안에 무엇이 들어 있나.** 순환식을 `h_3` 까지 손으로 풀어 대입하면 `h_3` 이 `x_1, x_2, x_3` 를 모두 포함함을 보일 수 있습니다. 그렇다면 왜 **먼 과거일수록** 영향이 약해질까요? (힌트: `tanh` 와 `Whh` 의 반복곱)
> 3. **`out[:, -1, :]` vs `h_n`.** 2부에서 마지막 시점 hidden 을 두 방법(`out[:, -1, :]`, `h_n[-1]`)으로 꺼낼 수 있습니다. 둘은 왜 같은 값일까요? 그리고 문장 분류가 아니라 **매 단어에 품사 태그** 를 붙이는 과제라면 `out` 전체가 왜 필요할까요?

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **RNN 은 시퀀스를 한 스텝씩 읽으며 "기억"(hidden state `h_t`)을 이어가는 신경망이다.** 심장은 순환식 **`h_t = tanh(Wₓₕ·x_t + Wₕₕ·h_{t-1} + b)`** 하나로, `h_{t-1}` 을 다시 넣는 **순환** 덕에 `h_t` 에는 과거 전체가 재귀적으로 포개진다. 결정적으로 `Wₓₕ, Wₕₕ, b` 를 **모든 시점에서 공유** 하므로 길이에 무관하고 위치 불변 패턴을 배운다. 학습은 시간축으로 **펼쳐(unroll)** [[Day-027]]·[[Day-028]] 의 역전파를 적용하는 **BPTT** 이며, PyTorch 에선 [[Day-029]] 의 골격 위에 `nn.RNN` 한 줄이면 된다 — "첫 비트 기억하기" 를 정확도 **1.00** 으로 학습해, hidden state 가 실제 "기억" 임을 확인했다(그리고 첫 원소만 바꿔도 결과가 뒤집혀 **순서를 쓰고 있음** 을 봤다, [[Day-007]] BoW 가 못하던 일). 다만 `Wₕₕ` 반복곱 탓에 먼 과거는 **기울기 소실** 로 잊혀지는데, 이를 게이트로 푸는 것이 다음 편 [[Day-031]] 의 LSTM·GRU 이고, 그 흐름이 Phase 3 마무리 🛠️ **B3 빌드(미니 Transformer)** 로 이어진다.
