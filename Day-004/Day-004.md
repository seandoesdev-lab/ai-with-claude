- **평균(mean)·분산(variance)·표준편차(std)** — 데이터가 어디에 모여 있고(중심), 얼마나 흩어져 있는지(퍼짐).

- **거리 vs 유사도** — 두 벡터가 "얼마나 먼가" 와 "얼마나 비슷한가"는 다르다.

> Today we build the *minimum* statistical intuition needed for IR/NLP/RAG: center & spread (mean/variance/std), the normal distribution and its 68-95-99.7 rule, standardization (z-score) to put features on the same scale, and the crucial difference between **distance** and **similarity**. ## 1. 핵심 개념 (Core Concepts)

### 1.1 확률 한 입 — 분포(distribution)라는 사고방식

**확률(probability)** 은 "어떤 일이 일어날 가능성"을 **0(절대 안 일어남) ~ 1(반드시 일어남)** 사이 숫자로 나타낸 것입니다. AI 에서 중요한 건 개별 확률값보다 **분포(distribution)** 라는 사고방식입니다.

- **분포** = "어떤 값들이, 각각 얼마나 자주(혹은 어떤 가능성으로) 나타나는가"의 전체 그림.
- LLM 이 다음 단어를 고를 때도 사실은 **어휘 전체에 대한 확률 분포**를 만든 뒤 거기서 하나를 뽑습니다(Day 4X 디코딩에서 정식으로 다룸).
- 그러니 "데이터를 분포로 본다"는 습관이 트랙 전체를 관통합니다.

> Probability is a number in [0, 1]. But the mindset that matters most for AI is the **distribution** — the full picture of which values occur and how often. Even an LLM picks its next token by sampling from a probability distribution over the vocabulary.

### 1.2 중심(center) — 평균과 중앙값

데이터 한 무더기를 대표하는 "가운데 값"입니다.

| 통계량 | 영어 | 뜻 | 특징 |
|--------|------|-----|------|
| 평균 | mean (average) | 모든 값을 더해 개수로 나눔 | 직관적이지만 **이상치(outlier)에 약함** |
| 중앙값 | median | 정렬했을 때 한가운데 값 | 이상치에 **강함**(robust) |

> 💡 예: 연봉 `[3000, 3200, 3500, 3300, 90000]`(만 원). 평균은 약 2.26억처럼 보이지만, 중앙값은 3300 만 원으로 현실에 가깝습니다. **한 명의 극단값**이 평균을 끌어올린 것. 데이터를 볼 때 평균과 중앙값을 **둘 다** 보는 습관이 중요합니다.

### 1.3 퍼짐(spread) — 분산과 표준편차

평균만으로는 부족합니다. `[50, 50, 50]` 과 `[0, 50, 100]` 은 평균이 같지만(50) 전혀 다른 데이터죠. 이 **흩어진 정도**를 재는 것이 분산과 표준편차입니다.

- **편차(deviation)** = 각 값 − 평균. (평균에서 얼마나 떨어졌나)
- **분산(variance, σ²)** = 편차를 **제곱해서 평균** 낸 것. (제곱하는 이유: 부호 없애기 + 큰 차이를 강조)
- **표준편차(standard deviation, σ)** = 분산의 **제곱근**. (단위가 원래 데이터와 같아져 해석이 쉬움)

$$\text{분산 } \sigma^2 = \frac{1}{N}\sum_{i=1}^{N}(x_i - \mu)^2, \qquad \text{표준편차 } \sigma = \sqrt{\sigma^2}$$

> ⚠️ **모분산 vs 표본분산 (ddof).** NumPy 의 `.var()`/`.std()` 는 기본값이 **모분산**(`ddof=0`, N 으로 나눔)입니다. 표본에서 모집단을 **추정**할 때는 `N−1` 로 나누는 **표본분산**(`ddof=1`)을 씁니다. 입문 단계에선 `ddof=0` 으로 충분하되, "둘이 다르다"는 것만 기억하세요(실습에서 확인).

### 1.4 정규분포(Normal / Gaussian) — 종 모양과 68-95-99.7 규칙

세상의 수많은 데이터(키, 몸무게, 측정 오차, 신경망 초기 가중치…)는 **정규분포**라는 좌우대칭 "종 모양(bell curve)"을 따릅니다. 정규분포는 단 두 숫자, **평균 μ(중심)** 와 **표준편차 σ(폭)** 로 완전히 결정됩니다.

가장 유용한 실전 규칙이 **68-95-99.7 규칙(empirical rule)** 입니다.

```
        평균 μ 를 중심으로...
   μ-3σ   μ-2σ   μ-σ    μ    μ+σ   μ+2σ   μ+3σ
    │      │      │      │      │      │      │
    └──────┴──────┴──┐ ▁▄█▄▁ ┌──┴──────┴──────┘
                ◀── 68% ──▶            (±1σ 안)
          ◀──────── 95% ────────▶      (±2σ 안)
    ◀──────────── 99.7% ────────────▶  (±3σ 안)
```

- 데이터의 약 **68%** 가 평균 **±1σ** 안에,
- 약 **95%** 가 **±2σ** 안에,
- 약 **99.7%** 가 **±3σ** 안에 들어옵니다.

> The normal (Gaussian) distribution is the symmetric "bell curve," fully described by its mean μ and standard deviation σ. The empirical rule: ~68% of data lies within ±1σ, ~95% within ±2σ, ~99.7% within ±3σ. This is why a value 3σ away is "rare" (≈0.3%).

### 1.5 표준화(Standardization, z-score) — 단위가 다른 특징을 같은 잣대로

키(170cm 근처)와 몸무게(65kg 근처)를 그대로 한 벡터에 넣으면, 숫자가 큰 키가 계산을 지배해 버립니다. 그래서 각 특징을 **평균 0, 표준편차 1** 로 바꿔 **같은 스케일**로 맞춥니다. 이것이 **표준화(z-score)** 입니다.

$$z = \frac{x - \mu}{\sigma}$$

- z 값은 "이 데이터가 평균에서 **표준편차 몇 개만큼** 떨어졌나"를 뜻합니다. (`z=2` → 평균보다 2σ 큼)
- 머신러닝·딥러닝에서 입력을 표준화하면 **학습이 안정**되고 빨라집니다. 트랜스포머의 **층 정규화(Layer Normalization, Phase 3)** 도 이 발상의 친척입니다.

> ⚠️ 용어 주의: **표준화(standardization)** 는 "평균0·표준편차1 로 맞추기"(z-score). Day 003 에서 한 **정규화(normalization)** 는 보통 "벡터 길이를 1로 맞추기"(L2 normalize). 둘 다 "스케일을 맞춘다"는 점은 같지만 **목적과 공식이 다릅니다.** 한국어로는 둘 다 "정규화"로 뭉뚱그려 부르는 경우가 많아 혼동하기 쉽습니다.

### 1.6 거리(distance) vs 유사도(similarity) — 오늘의 하이라이트 ⭐

두 벡터가 "비슷하다"를 재는 두 가지 관점입니다.

**① 유클리드 거리 (Euclidean distance, L2):** 두 점 사이의 **직선 거리**. 작을수록 가깝다.

$$d(\mathbf{a},\mathbf{b}) = \sqrt{\sum_i (a_i - b_i)^2}$$

**② 코사인 유사도 (Cosine similarity):** 두 벡터가 이루는 **각도**가 얼마나 작은가(방향이 얼마나 같은가). −1 ~ 1, 클수록 비슷하다. **크기(길이)는 무시**하고 **방향만** 본다.

$$\cos(\mathbf{a},\mathbf{b}) = \frac{\mathbf{a}\cdot\mathbf{b}}{\lVert\mathbf{a}\rVert\,\lVert\mathbf{b}\rVert}$$

핵심 차이를 한 문장으로:

> **거리는 "얼마나 멀리 떨어졌나(크기 포함)", 코사인은 "얼마나 같은 방향을 보나(크기 무시)".**

검색(IR)에서 **문서 길이**를 떠올리면 왜 보통 코사인을 쓰는지 분명해집니다. 같은 주제를 길게 쓴 문서는 단어 등장 횟수가 커서 벡터가 "길어"집니다 → 유클리드 거리로는 질문에서 **멀어 보입니다.** 하지만 **방향(주제)** 은 같으므로 코사인은 "비슷하다"고 제대로 알아챕니다.

그리고 둘을 잇는 아름다운 관계가 있습니다 — **단위벡터(길이 1)** 끼리는:

$$d_{\text{유클리드}}^2 = 2\,(1 - \cos)$$

즉 **정규화만 해두면 "거리로 정렬"과 "코사인으로 정렬"이 똑같은 순위**를 줍니다. 그래서 벡터 DB 들은 내부적으로 둘을 자유롭게 오갑니다(Phase 5).

> Euclidean distance measures how far apart (magnitude matters); cosine similarity measures how aligned the directions are (magnitude ignored). For document search, a long document has a "big" vector and looks far in Euclidean terms even on-topic — cosine fixes this. And for **unit vectors**, `dist² = 2(1 − cos)`, so ranking by distance and by cosine agree.

## 2. 직관 / 비유 (Intuition)

**과녁 비유 (The Dartboard Analogy) — 평균과 표준편차:**
다트를 여러 번 던졌다고 합시다.
- **평균** = 화살들이 모인 **무게중심**의 위치 (조준이 어디로 쏠렸나 = 편향, bias).
- **표준편차** = 화살들이 그 중심에서 **얼마나 흩어졌나** (손이 얼마나 떨렸나 = 분산, variance).
- 좋은 사수는 중심도 정확하고(평균↔정답) 흩어짐도 작습니다(작은 σ).

**키 비교 비유 — z-score:**
"철수는 키 182cm, 영희는 농구선수 중 키 182cm." 같은 182cm 라도 의미가 다르죠. **z-score 는 "그 집단 안에서 몇 σ 만큼 큰가"** 로 바꿔, 서로 다른 집단·단위를 **공정하게 비교**하게 해줍니다.

**방향 vs 거리 비유 — 코사인:**
두 사람이 **같은 방향**으로 걷고 있다면, 한 명이 100m 앞서 있어도 "**같은 길을 간다**"고 말합니다(코사인=1). 반면 **직선 거리**로 재면 100m 나 떨어져 있죠. 검색에서 "같은 주제"를 찾는 건 **같은 방향**을 찾는 일에 가깝습니다.

> Mean/std = the center and scatter of darts on a board (bias vs. variance). z-score = "how many σ above the group's average," making different scales comparable. Cosine = two walkers heading the *same direction* are "on the same path" even if one is 100m ahead — that's what topic-search wants.

## 3. 직관을 굳히는 한 장의 그림 (One picture to remember)

```
   같은 방향, 다른 크기            →  코사인은 "같다", 거리는 "다르다"

        b=(0,1)
          ▲
          │            a 와 c 는 한 직선 위(같은 방향)
          │              cos(a,c)=1.00  ✅ 비슷
          │              d(a,c)=1.00    ❌ 멀다(길이차)
          │
          └────────●─────────●────▶
                 a=(1,0)   c=(2,0)
        cos(a,b)=0 (직각) · d(a,b)=√2
```

### 4.2 평균·분산·표준편차·정규분포 — `stats_basics.py`

아래를 프로젝트 폴더에 **`stats_basics.py`** 로 저장합니다. (옆에 함께 제공됩니다.)

> 💡 맨 위 `sys.stdout.reconfigure(encoding="utf-8")` 한 줄은 **Windows 콘솔에서 한글·특수기호(`—`, `σ` 등)가 깨지거나 `UnicodeEncodeError` 가 나는 것을 막는** 작은 안전장치입니다. 한 번 써 두면 마음이 편합니다.

```python
# stats_basics.py — 중심/퍼짐, 정규분포, 표준화(z-score)
import sys
import numpy as np

# Windows 콘솔(cp949)에서 한글·특수기호가 깨지거나 에러 나는 것을 막아 줍니다.
sys.stdout.reconfigure(encoding="utf-8")

rng = np.random.default_rng(42)   # seed 고정 → 재현 가능

print("=" * 56)
print("[1] 중심과 퍼짐: 평균(mean)·분산(var)·표준편차(std)")
print("=" * 56)
scores = np.array([60, 70, 80, 90, 100], dtype=float)
print("점수            :", scores)
print("평균   mean     :", scores.mean())          # 80.0
print("분산   variance :", scores.var())           # 200.0 (편차 제곱의 평균)
print("표준편차 std    :", round(scores.std(), 4)) # 14.1421...

# '직접' 계산해서 공식 확인하기
dev = scores - scores.mean()                        # 편차(deviation)
print("편차            :", dev)
print("분산(직접 계산) :", (dev ** 2).mean())       # == scores.var()

# 모분산(ddof=0, 기본) vs 표본분산(ddof=1)
print("표본분산 ddof=1 :", round(scores.var(ddof=1), 4))  # N-1 로 나눔 → 더 큼

print("\n" + "=" * 56)
print("[2] 정규분포(normal) 만들고 통계 재기")
print("=" * 56)
# 평균 170, 표준편차 6 인 '키' 데이터 10,000개를 정규분포에서 샘플링
heights = rng.normal(loc=170, scale=6, size=10_000)
print("표본 평균    :", round(heights.mean(), 3))   # ~170
print("표본 표준편차:", round(heights.std(), 3))     # ~6

# 68-95-99.7 규칙을 '비율'로 직접 검증
mu, sd = heights.mean(), heights.std()
theory = [68.3, 95.4, 99.7]
for k in (1, 2, 3):
    inside = np.mean(np.abs(heights - mu) < k * sd) * 100
    print(f"  평균 ±{k}σ 안의 비율: {inside:5.1f}%  (이론값 {theory[k-1]}%)")

print("\n" + "=" * 56)
print("[3] 텍스트 히스토그램으로 '종 모양' 직접 보기")
print("=" * 56)
counts, edges = np.histogram(heights, bins=15)
peak = counts.max()
for c, left, right in zip(counts, edges[:-1], edges[1:]):
    bar = "#" * int(round(c / peak * 40))
    print(f"{left:6.1f} ~ {right:6.1f} | {bar} {c}")

print("\n" + "=" * 56)
print("[4] 표준화(standardization, z-score) — 스케일 맞추기")
print("=" * 56)
# 특징 2개: 키(cm, ~170)와 몸무게(kg, ~65). 단위·스케일이 전혀 다름!
feat = np.array([[172, 68],
                 [160, 55],
                 [185, 90],
                 [168, 60]], dtype=float)   # shape (4, 2): 4명 x 2특징
print("원본 (키, 몸무게):\n", feat)
mu = feat.mean(axis=0)          # 열별(특징별) 평균  → (2,)
sd = feat.std(axis=0)           # 열별 표준편차       → (2,)
z = (feat - mu) / sd            # 브로드캐스팅: (4,2) - (2,) / (2,)
print("표준화 후 z (평균0, 표준편차1):\n", z.round(2))
print("표준화 후 열별 평균    :", z.mean(axis=0).round(6))  # ≈ [0, 0]
print("표준화 후 열별 표준편차:", z.std(axis=0).round(6))   # ≈ [1, 1]
```

### 4.3 거리 vs 유사도 — `distance_vs_similarity.py`

오늘의 하이라이트입니다. 같은 데이터를 **유클리드 거리**와 **코사인 유사도** 두 잣대로 비교하고, "검색은 왜 보통 코사인인가"를 직접 봅니다. 아래를 **`distance_vs_similarity.py`** 로 저장합니다.

```python
# distance_vs_similarity.py — 유클리드 거리 vs 코사인 유사도
import sys
import numpy as np

# Windows 콘솔(cp949)에서 한글·특수기호가 깨지거나 에러 나는 것을 막아 줍니다.
sys.stdout.reconfigure(encoding="utf-8")

def euclidean(a, b):
    """두 벡터 사이의 유클리드 거리(L2): 직선 거리. 작을수록 가깝다."""
    return np.sqrt(((a - b) ** 2).sum())

def cosine(a, b):
    """코사인 유사도: 방향(각도)이 얼마나 같은가. -1~1, 클수록 비슷."""
    return (a @ b) / (np.linalg.norm(a) * np.linalg.norm(b))

print("=" * 56)
print("[1] '거리'는 멀고가까움, '코사인'은 방향(크기 무시)")
print("=" * 56)
a = np.array([1.0, 0.0])
b = np.array([0.0, 1.0])
c = np.array([2.0, 0.0])   # a 와 '같은 방향', 길이만 2배
print(f"a={a}, b={b}, c={c}")
print(f"  d(a,b)={euclidean(a,b):.3f}   cos(a,b)={cosine(a,b):.3f}  (직각->0)")
print(f"  d(a,c)={euclidean(a,c):.3f}   cos(a,c)={cosine(a,c):.3f}  (같은 방향->1)")
print("-> a와 c는 방향이 같아 코사인=1.0, 하지만 길이가 달라 거리는 1.0")

print("\n" + "=" * 56)
print("[2] 검색에서 왜 보통 '코사인'을 쓰나 — 문서 길이 문제")
print("=" * 56)
# 단어 [고양이, 강아지, 자동차] 의 등장 횟수(term frequency) 벡터
query = np.array([1, 1, 0])      # 짧은 질문: 고양이 강아지
short = np.array([1, 1, 0])      # 같은 주제, 짧은 문서
long_ = np.array([10, 10, 1])    # 같은 주제, 긴 문서 (횟수만 큼)
junk  = np.array([0, 0, 5])      # 다른 주제: 자동차
for name, d in [("short(같은주제·짧음)", short),
                ("long (같은주제·긺)", long_),
                ("junk (다른주제)", junk)]:
    print(f"  {name:20s} 거리={euclidean(query,d):6.2f}  코사인={cosine(query,d):.3f}")
print("-> 거리로는 'long'이 멀어 보이지만(길이 탓!), 코사인은 주제가 같음을 알아챔")

print("\n" + "=" * 56)
print("[3] 정규화하면: (유클리드 거리)^2 = 2 * (1 - 코사인유사도)")
print("=" * 56)
def unit(v):
    return v / np.linalg.norm(v)
x = np.array([3.0, 4.0, 0.0])
y = np.array([4.0, 0.0, 3.0])
xu, yu = unit(x), unit(y)
d2  = euclidean(xu, yu) ** 2
rel = 2 * (1 - cosine(xu, yu))
print(f"  단위벡터 거리^2 = {d2:.4f}")
print(f"  2 * (1 - cos)   = {rel:.4f}   <- 같다!")
print("-> 정규화된 임베딩에서는 '거리 정렬'과 '코사인 정렬'이 같은 순위를 준다")

print("\n" + "=" * 56)
print("[4] 같은 데이터, 두 랭킹 (거리 작을수록 가까움 vs 코사인 클수록 비슷)")
print("=" * 56)
docs = np.array([[1.0, 1.0, 0.0],
                 [10.0, 10.0, 1.0],
                 [0.0, 0.0, 5.0],
                 [2.0, 1.0, 0.0]])
q = np.array([1.0, 1.0, 0.0])

dist = np.sqrt(((docs - q) ** 2).sum(axis=1))            # 각 문서까지 거리
qn = q / np.linalg.norm(q)
dn = docs / np.linalg.norm(docs, axis=1, keepdims=True)  # 행별 정규화 (Day 003!)
cos = dn @ qn                                            # 코사인 유사도

print("문서별 거리   :", dist.round(3), " -> 가까운 순:", np.argsort(dist))
print("문서별 코사인 :", cos.round(3), " -> 비슷한 순:", np.argsort(cos)[::-1])
print("문서 1(긴 같은주제)이 거리 랭킹에선 밀리고, 코사인 랭킹에선 상위인지 비교!")
```

실행:

```powershell
uv run python distance_vs_similarity.py
```

**관찰 포인트:**
- `[1]`: `a`(길이1)와 `c`(길이2)는 **방향이 같아 코사인=1.0**, 그러나 **거리는 1.0**. "크기 무시 vs 크기 포함"의 차이를 한눈에.
- `[2]`: `long` 문서는 **거리로는 질문에서 가장 멀어** 보입니다(횟수가 커서). 그런데 **코사인으로는 short 와 거의 같은 점수** — 길이에 속지 않고 주제를 봅니다. 이것이 IR 에서 코사인이 표준인 핵심 이유입니다.
- `[3]`: `dist² = 2(1−cos)` 가 소수점까지 일치합니다. **정규화 한 번이 두 잣대를 통일**시킨다는 사실.
- `[4]`: `np.linalg.norm(docs, axis=1, keepdims=True)` 는 어제 배운 **행별 정규화** 그대로입니다. 거리 랭킹과 코사인 랭킹이 **다른 순위**를 줄 수 있음을 직접 확인하세요.

## 5. 용어집 (Glossary)

| 용어 | 영어 | 한 줄 뜻 |
|------|------|---------|
| 확률 | probability | 사건의 가능성, 0~1 사이 수 |
| 분포 | distribution | 어떤 값들이 각각 얼마나 자주 나타나는지의 전체 그림 |
| 평균 | mean / average | 값들의 합 ÷ 개수 (중심, 이상치에 약함) |
| 중앙값 | median | 정렬 후 한가운데 값 (이상치에 강함) |
| 편차 | deviation | 각 값 − 평균 |
| 분산 | variance (σ²) | 편차 제곱의 평균 (퍼짐의 정도) |
| 표준편차 | standard deviation (σ) | 분산의 제곱근 (단위가 데이터와 같음) |
| 정규분포 | normal / Gaussian distribution | μ·σ로 결정되는 좌우대칭 종 모양 |
| 68-95-99.7 규칙 | empirical rule | ±1σ/±2σ/±3σ 안에 약 68/95/99.7% |
| 표준화 | standardization (z-score) | (x−μ)/σ 로 평균0·표준편차1 만들기 |
| 정규화 | normalization (L2) | 벡터 길이(norm)를 1로 만들기 |
| 유클리드 거리 | Euclidean distance (L2) | 두 점의 직선 거리 √Σ(aᵢ−bᵢ)² |
| 코사인 유사도 | cosine similarity | 두 벡터의 방향 일치도(크기 무시), −1~1 |
| 이상치 | outlier | 다른 값들과 동떨어진 극단값 |

## 7. 오늘의 한 줄 요약 (One-line takeaway)

> 데이터는 **중심(평균)** 과 **퍼짐(표준편차)** 으로 요약되고, 그 모양은 흔히 **정규분포(68-95-99.7)** 를 따른다. 단위가 다른 특징은 **z-score 표준화**로 같은 잣대에 올리고, 두 벡터의 닮음은 **거리(크기 포함)** 가 아니라 보통 **코사인 유사도(방향)** 로 잰다 — 정규화해 두면 둘은 같은 순위를 준다. 이것이 어제 임베딩을 정규화한 이유다.
