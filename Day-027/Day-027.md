## 0. 들어가며 (Motivation)

지난 편([[Day-026]])의 마지막 장면을 떠올려 봅시다. ReLU 은닉 뉴런 2개로 XOR 를 완벽히 풀었지만, 그 가중치 $W_1,b_1,W_2,b_2$ 는 **우리가 직접 골라 넣은 값** 이었습니다. 4개짜리 장난감 문제라 가능했지, 진짜 신경망(GPT 는 수천억 개)에서는 상상도 못 할 일입니다.

그래서 신경망의 진짜 마법은 구조(순전파)가 아니라 **학습(learning)** 에 있습니다: *"틀린 답을 보고, 어떤 가중치를 얼마나 움직여야 덜 틀릴까"* 를 자동으로 알아내는 것. 놀랍게도 우리는 이 도구를 이미 절반쯤 갖고 있습니다 — [[Day-014]] 에서 로지스틱 회귀를 **교차엔트로피 손실 + 경사하강** 으로 학습시켰죠. 오늘 배울 역전파는 그 경사하강을 **여러 층으로 확장** 하는 딱 한 가지 재료 — **연쇄법칙(chain rule)** — 를 더한 것입니다.

> Last time we solved XOR by *hand-picking* the weights. Real networks have millions of parameters, so we need them to be found *automatically* from data. That is what **backpropagation** does: forward-pass to predict, measure the error with a **loss**, use the **chain rule** to push that error backward into every weight as a **gradient**, then nudge the weights with **gradient descent** — the same optimizer we already used for logistic regression in [[Day-014]], now extended across layers.

## 1. 핵심 개념 (Core Concepts)

### 1.1 학습 = 손실을 줄이는 일

"학습(learning)" 을 수학으로 옮기면 이렇습니다: 예측이 정답에서 얼마나 벗어났는지를 하나의 숫자 — **손실(loss)** $L$ — 로 재고, 그 $L$ 을 **가장 작게 만드는** 파라미터 $\theta=\{W_1,b_1,W_2,b_2\}$ 를 찾는 **최적화(optimization)** 문제입니다. 오늘은 [[Day-014]] 와 같은 **이진 교차엔트로피(binary cross-entropy, BCE)** 를 씁니다:

$$
L = -\frac{1}{N}\sum_{i=1}^{N}\Big[\,y_i\log \hat y_i + (1-y_i)\log(1-\hat y_i)\,\Big]
$$

여기서 $\hat y_i$ 는 출력 뉴런의 시그모이드 확률입니다. 정답에 가까우면 $L\!\to\!0$, 크게 틀리면 $L$ 이 커집니다.

### 1.2 기울기(gradient) — "어느 쪽으로 움직여야 손실이 주나"

각 파라미터 $\theta_j$ 를 **아주 조금** 키웠을 때 손실이 얼마나 변하는가 = 편미분 $\partial L/\partial \theta_j$. 이 값들을 모은 벡터가 **기울기(gradient)** $\nabla_\theta L$ 이고, 손실이 **가장 빨리 커지는 방향** 을 가리킵니다. 따라서 그 **반대 방향** 으로 한 걸음 내려가면 손실이 줄어듭니다 — 이것이 **경사하강(gradient descent)** 입니다([[Day-014]]):

$$
\theta \;\leftarrow\; \theta \;-\; \eta\,\nabla_\theta L
\qquad(\eta=\text{학습률, learning rate})
$$

문제는 단 하나: **여러 층을 거친** 파라미터(특히 은닉층의 $W_1$)에 대해 $\partial L/\partial \theta_j$ 를 **어떻게 계산하느냐** 입니다. 답이 연쇄법칙입니다.

### 1.3 연쇄법칙(chain rule) — 역전파의 심장

신경망은 함수의 **합성(composition)** 입니다: $x \to z_1 \to a_1 \to z_2 \to \hat y \to L$. 미적분의 **연쇄법칙** 은 합성함수의 미분을 "국소 미분들의 곱" 으로 쪼갭니다. 예를 들어 은닉층 가중치 $W_1$ 에 대한 손실의 미분은:

$$
\frac{\partial L}{\partial W_1}
= \underbrace{\frac{\partial L}{\partial \hat y}}_{\text{출력 오차}}
\cdot \frac{\partial \hat y}{\partial z_2}
\cdot \frac{\partial z_2}{\partial a_1}
\cdot \underbrace{\frac{\partial a_1}{\partial z_1}}_{\text{활성함수 미분}}
\cdot \frac{\partial z_1}{\partial W_1}
$$

즉 **출력 쪽 오차** 를 시작점으로, 각 층의 **국소 미분** 을 곱해 가며 입력 쪽으로 되짚어 내려갑니다. 이 "뒤에서 앞으로 미분을 전파" 하는 과정이 **역전파(backpropagation)** — 순전파([[Day-026]])와 정확히 **반대 방향** 입니다.

### 1.4 순전파 ↔ 역전파, 한 쌍으로 보기

| 방향 | 흐르는 것 | 하는 일 |
|------|----------|---------|
| **순전파 forward** | 입력 → 출력 (활성값 $a$) | 답 $\hat y$ 와 손실 $L$ 계산 |
| **역전파 backward** | 출력 → 입력 (기울기 $\partial L/\partial \cdot$) | 각 가중치의 기울기 계산 |

핵심 통찰: 역전파는 순전파에서 계산해 둔 값($a_1,\hat y$ 등)을 **재활용** 하기 때문에, 층이 아무리 많아도 순전파와 **비슷한 비용** 으로 모든 기울기를 한 번에 구합니다. 이 효율성이 딥러닝을 가능하게 했습니다.

### 1.5 아름다운 단순화 — BCE + 시그모이드

출력층에 **시그모이드 + BCE** 를 함께 쓰면, 복잡해 보이던 출력 오차 미분이 놀랄 만큼 깔끔해집니다([[Day-014]] 에서 이미 등장):

$$
\frac{\partial L}{\partial z_2} = \hat y - y
$$

"예측 − 정답" 그 자체입니다. 실습 코드에서 `dz2 = (a2 - y)/N` 한 줄이 바로 이것 — 여기서부터 기울기가 은닉층으로 거꾸로 흘러갑니다.

### 1.6 한 번의 학습 스텝 = 4단계 순환

$$
\text{① forward} \;\to\; \text{② loss} \;\to\; \text{③ backward(연쇄법칙)} \;\to\; \text{④ update(경사하강)}
$$

이 한 바퀴를 데이터 전체에 대해 반복하는 단위가 **에폭(epoch)** 이고, 수천 에폭을 돌리면 손실이 점점 줄며 가중치가 "정답을 내는" 값으로 스스로 수렴합니다.

## 2. 직관 / 비유 (Intuition)

> [!tip] 비유 — 책임 소재를 거슬러 추궁하기 (blame assignment)
> 회사가 잘못된 보고서(=틀린 예측)를 냈습니다. 사장(출력층)은 먼저 **"최종 답이 정답에서 얼마나 벗어났나"** 를 잽니다(=손실). 그다음 **책임을 거슬러 추궁** 합니다: "이 오차 중 **네 몫** 은 얼마냐?" 를 바로 아래 부서(은닉층)에 묻고, 각 부서는 다시 자기 팀원(가중치)에게 같은 질문을 내려보냅니다. 이때 **영향력이 컸던 사람일수록 더 큰 책임(=큰 기울기)** 을 배정받고, 그만큼 더 크게 행동을 고칩니다(=경사하강 갱신). 역전파란 바로 이 **"출력의 오차를, 각자가 기여한 만큼 거꾸로 나눠 주는"** 책임 분배 과정입니다. 연쇄법칙은 "부서를 거칠 때마다 그 부서의 민감도(국소 미분)를 곱해 책임을 전달" 하는 규칙이고요.
>
> A wrong final report (bad prediction) is traced backward: each layer asks the layer below "how much of this error is *your* fault?" Whoever had more influence gets more blame (a bigger gradient) and adjusts more. Backprop is credit/blame assignment; the chain rule is how blame is passed layer by layer (multiply by each layer's local sensitivity).

## 3. 한 장의 그림 (One Picture)

```text
                 ┌──────────────── 순전파 (forward, Day-026) ───────────────►
   입력 x ──▶ z1 = W1·x + b1 ──▶ a1 = φ(z1) ──▶ z2 = W2·a1 + b2 ──▶ ŷ = σ(z2) ──▶ L(ŷ, y)
                                                                                    │
   ◀──────────────────────── 역전파 (backward, 오늘) ───────────────────────────────┘
        ∂L/∂W1        ∂L/∂a1        ∂L/∂z2 = ŷ − y   (BCE+sigmoid 의 깔끔한 시작점)
          ▲             ▲              │
          └─ ×φ'(z1) ───┴── ×W2ᵀ ─────┘     각 화살표에서 '국소 미분'을 곱한다(연쇄법칙)

   그리고 갱신:  θ ← θ − η·(∂L/∂θ)      (경사하강, Day-014 와 같은 규칙)
   ────────────────────────────────────────────────────────────────────────────
   이 [forward → loss → backward → update] 한 바퀴 = 1 step,  전체 데이터 1회 = 1 epoch
```

> Forward computes the prediction and loss left-to-right; backward sends the gradient right-to-left, multiplying by each layer's local derivative (the chain rule). The starting point $\partial L/\partial z_2=\hat y-y$ is remarkably simple thanks to the BCE+sigmoid pairing. Then gradient descent nudges every parameter.

### 4.2 핵심 코드 — 역전파 4단계 (발췌)

```python
import numpy as np

def sigmoid(z):      return 1.0 / (1.0 + np.exp(-z))
def sigmoid_grad(a): return a * (1.0 - a)      # a=σ(z) 일 때 dσ/dz = a(1-a)

X = np.array([[0,0],[0,1],[1,0],[1,1]], float)
y = np.array([[0],[1],[1],[0]], float)         # XOR

rng = np.random.default_rng(7); H = 4          # 은닉 뉴런 4개, seed 고정
W1 = rng.normal(0,1,(2,H)); b1 = np.zeros((1,H))
W2 = rng.normal(0,1,(H,1)); b2 = np.zeros((1,1))
lr, N = 0.5, X.shape[0]

for epoch in range(1, 8001):
    # ① 순전파
    a1 = sigmoid(X @ W1 + b1)
    a2 = sigmoid(a1 @ W2 + b2)                 # 예측 확률 ŷ
    # ② 손실은 로깅용 (BCE) — 생략 가능
    # ③ 역전파 (연쇄법칙)
    dz2 = (a2 - y) / N                         # ∂L/∂z2 = ŷ − y  (핵심!)
    dW2 = a1.T @ dz2;  db2 = dz2.sum(0, keepdims=True)
    dz1 = (dz2 @ W2.T) * sigmoid_grad(a1)      # 오차를 은닉층으로 전파 × 활성 미분
    dW1 = X.T @ dz1;   db1 = dz1.sum(0, keepdims=True)
    # ④ 경사하강 갱신
    W2 -= lr*dW2; b2 -= lr*db2; W1 -= lr*dW1; b1 -= lr*db1
```

### 4.3 결과를 정직하게 읽기 (관찰 포인트)

1. **학습 전은 '동전 던지기'.** 초기 손실 0.6994 ≈ $-\ln(0.5)$ 이고 예측이 죄다 0.5 언저리 — 아직 아무것도 모릅니다. 무작위 가중치에서 시작했으니 당연합니다.
2. **손실이 계단처럼 꺾여 내려간다.** epoch 500 까지는 거의 평평(0.69)하다가 1000 부근에서 0.27 로 **급락** 합니다. 신경망 학습에서 흔한 모습 — 한동안 헤매다(plateau) 유용한 표현을 찾는 순간 빠르게 무너집니다. 8000 에폭에서 0.0025 까지 떨어졌습니다.
3. **XOR 를 완전히 학습.** 최종 예측이 $(0,0)\!\to\!0.002,\ (0,1)\!\to\!0.998,\ (1,0)\!\to\!0.997,\ (1,1)\!\to\!0.003$ — 정확도 1.00. [[Day-026]] 에서 **손으로 넣던** 가중치를 이번엔 **데이터가 스스로** 찾아냈습니다.
4. **`dz2 = (a2 - y)/N` 한 줄이 출발점.** 1.5 절의 $\partial L/\partial z_2 = \hat y - y$ 그대로입니다. 여기서 `@ W2.T` 로 오차를 은닉층에 전파하고 `* sigmoid_grad(a1)` 로 활성함수 미분을 곱하는 두 줄이 **연쇄법칙의 실체** 입니다.

### 4.4 (실험) 직접 흔들어 보기

- **학습률 바꾸기**: `lr` 을 0.5 → 5.0 으로 키우면 더 빨리(때론 불안정하게) 수렴하고, 0.05 로 줄이면 8000 에폭으로 부족합니다. **학습률이 학습 속도·안정성의 손잡이** 임을 체감하세요.
- **비선형 제거**: 은닉 활성함수를 없애 `a1 = X @ W1 + b1` (선형)으로 바꾸면 손실이 0.69 근처에서 **안 내려갑니다** — [[Day-026]] 의 "선형 붕괴" 가 재현됩니다. 역전파가 있어도 **비선형이 없으면** XOR 는 못 풉니다.
- **기울기 수치검증(gradient check)**: $W_1$ 의 한 원소를 $\pm\epsilon$ 흔들어 손실 차이로 근사한 기울기 $\frac{L(\theta+\epsilon)-L(\theta-\epsilon)}{2\epsilon}$ 를 역전파가 준 `dW1` 과 비교해 보세요. 거의 같으면 backward 구현이 맞다는 강력한 증거입니다.
- **[[Day-014]] 와 잇기**: 은닉층을 아예 빼고 1층(=로지스틱 회귀)으로 만들면 XOR 는 못 풀지만 AND/OR 는 학습됩니다. "은닉층 없는 신경망 = 로지스틱 회귀" 를 코드로 확인.

## 5. 용어집 (Glossary)

| 용어 | English | 한 줄 뜻 |
|------|---------|---------|
| 역전파 | backpropagation | 출력의 오차를 연쇄법칙으로 각 가중치까지 거꾸로 전파해 기울기를 구하는 알고리즘 |
| 손실함수 | loss function | 예측이 정답에서 얼마나 벗어났는지를 하나의 수로 재는 함수 |
| 교차엔트로피 | cross-entropy (BCE) | 확률 예측용 손실 — 정답에 가까울수록 0, 틀릴수록 커짐 |
| 기울기 | gradient | 각 파라미터에 대한 손실의 편미분을 모은 벡터, 손실이 가장 빨리 커지는 방향 |
| 연쇄법칙 | chain rule | 합성함수의 미분을 국소 미분들의 곱으로 쪼개는 미적분 규칙 |
| 경사하강 | gradient descent | 기울기의 반대 방향으로 파라미터를 조금씩 갱신하는 최적화 |
| 학습률 | learning rate ($\eta$) | 한 걸음의 크기 — 학습 속도·안정성을 좌우 |
| 에폭 | epoch | 학습 데이터 전체를 한 번 훑는 단위 |
| 순전파 | forward pass | 입력→출력 방향으로 예측·손실을 계산(어제, [[Day-026]]) |

## 6. 생각거리 (Reflection)

> [!question] 스스로 답해 보기
> 1. **왜 '거꾸로'인가.** 은닉층 가중치 $W_1$ 의 기울기를 구하려면 출력 오차를 먼저 알아야 합니다. 이 "의존성의 방향" 때문에 계산이 왜 출력 → 입력 순서로 흘러야 하는지, 연쇄법칙의 곱 구조로 설명해 보세요.
> 2. **손실 곡선의 평지(plateau).** 실습에서 epoch 100~500 동안 손실이 0.69 에 머물다 1000 부근에서 급락했습니다. 이 "한동안 안 배우다 갑자기 배우는" 현상이 왜 생기는지, 무작위 초기화·비선형 은닉층의 역할과 엮어 상상해 보세요.
> 3. **자동화로 가는 다리.** 우리는 각 층의 미분($\partial L/\partial z_2=\hat y - y$, $\times W_2^\top$, $\times\phi'$)을 **손으로** 유도해 코드에 적었습니다. 층이 50개인 Transformer 라면 이걸 사람이 다 유도할 수 있을까요? "미분을 자동으로 계산" 해 주는 도구가 왜 필요할지 — 다음 편 [[Day-028]](PyTorch·autograd) 로 이어 상상해 보세요.

## 7. 오늘의 한 줄 요약 (Takeaway)

> [!summary]
> **신경망의 '학습'은 손실 $L$ 을 줄이는 최적화이고, 그 엔진이 역전파(backpropagation)다.** 네 단계가 한 바퀴를 이룬다: ① 순전파로 예측 $\hat y$ 와 손실(교차엔트로피)을 계산 → ② 출력 오차 $\partial L/\partial z_2=\hat y - y$ 에서 시작해 → ③ **연쇄법칙** 으로 각 층의 국소 미분($\times W^\top$, $\times\phi'$)을 곱하며 기울기를 입력 쪽으로 "거꾸로" 전파 → ④ **경사하강** $\theta\leftarrow\theta-\eta\nabla_\theta L$ 로 갱신([[Day-014]] 와 같은 규칙). 순전파에서 구한 값을 재활용하므로 층이 많아도 순전파와 비슷한 비용으로 모든 기울기를 한 번에 얻는다 — 이 효율이 딥러닝을 가능하게 했다. NumPy 만으로 [[Day-026]] 의 XOR MLP 를 *데이터로* 훈련해 정확도 1.00 을 손수 확인했다(손코딩 가중치를 이제 데이터가 스스로 찾았다). 🛠️ 이 backward 는 Phase 3 마무리 **B3 빌드(NumPy MLP → 미니 Transformer)** 의 심장이며, 다음 편 [[Day-028]] 은 이 미분을 자동화하는 **PyTorch·autograd** 다.
