> [!abstract] 🎯 오늘의 목표
> 앞으로 배울 모든 주제(NLP·IR·LLM·RAG)가 **하나의 지도** 위 어디에 있는지 이해하고, **RAG = IR + LLM** 이라는 큰 그림을 잡는다. 코드보다 **개념의 지도**를 그리는 날.

## 0. 들어가며 (Why start here?)

심화 주제로 바로 뛰어들고 싶겠지만, **용어들이 어떻게 연결되는지**에 대한 지도가 없으면 나중 지식이 머릿속에서 떠다닙니다. 오늘은 그 지도를 그립니다 — 앞으로 매일 다시 꺼내 볼 기준점입니다.

> It's tempting to jump straight into advanced topics, but without a map of how the terms connect, later knowledge floats around untethered. Today we draw the map.

## 1. 핵심 개념 (Core Concepts)

### 1.1 큰 동심원: AI ⊃ ML ⊃ DL

```text
┌─────────────────────────────────────────────┐
│ AI (인공지능)  "지적 작업을 기계가 수행"          │
│  ┌─────────────────────────────────────────┐ │
│  │ ML (머신러닝) "규칙 대신 데이터로 학습"       │ │
│  │  ┌─────────────────────────────────────┐ │ │
│  │  │ DL (딥러닝) "다층 신경망"  → Transformer → LLM │ │
│  │  └─────────────────────────────────────┘ │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

- **AI**: 가장 넓은 개념(규칙 기반도 포함). **ML**: 데이터로 패턴을 학습. **DL**: 다층 신경망 기반 ML — 오늘날 LLM은 모두 여기 위에 있습니다.

### 1.2 가로축: 응용 분야

| 분야 | 한 줄 정의 | 트랙 위치 |
|------|-----------|-----------|
| **NLP** (자연어 처리) | 사람의 언어를 기계가 이해·생성 | Phase 1 |
| **IR** (정보 검색) | 문서 더미에서 질문에 맞는 것을 찾아 순위 매김 | Phase 2·5 |
| **LLM** (대규모 언어 모델) | 방대한 텍스트로 학습한 언어 생성 신경망 | Phase 3·4 |
| **RAG** (검색 증강 생성) | **IR + LLM** — 먼저 검색해 근거를 찾고, 그 근거로 생성 | Phase 6 |

### 1.3 핵심 통찰: RAG = IR + LLM

> [!important] 이 트랙을 관통하는 한 문장
> **RAG는 검색(IR)으로 찾은 근거를 LLM에 건네 답하게 하는 구조다.** "먼저 찾고(retrieve), 그다음 생성한다(generate)."

LLM은 (1) 모르는 최신/사내 정보를 지어내고(**환각, hallucination**), (2) 출처를 못 댑니다. IR이 그 약점을 메웁니다. 그래서 NLP·IR·LLM을 따로 배운 뒤 마지막에 RAG로 합칩니다.

```text
질문 ──▶ [IR: 관련 문서 검색] ──▶ 근거
질문 + 근거 ──────────────────▶ [LLM: 답 생성] ──▶ 답변(+출처)
```

## 2. 직관 / 비유 (Intuition)

> [!tip] 도서관 비유
> - **LLM** = 박식하지만 **기억에만 의존해 답하는 사서** (가끔 자신 있게 틀림 = 환각).
> - **IR** = 질문에 맞는 **책을 빠르게 찾아주는 색인/검색** 시스템.
> - **RAG** = 사서가 답하기 전에 **관련 책을 펴 근거를 확인하고** 답하는 방식 → 더 정확하고 출처를 댈 수 있음.
>
> *LLM = a well-read but memory-only librarian; IR = the catalog that finds the right books; RAG = checking those books before answering.*

## 3. 한 장의 그림 (One Picture)

```text
        [ 잘 찾아서(retrieve) ]  +  [ 잘 답한다(generate) ]
                 IR                         LLM
                   \_________  RAG  _________/
```

### 4.2 "검색 vs 생성"을 코드로 — `mini_search.py`
IR의 가장 원초적 형태(단순 키워드 매칭). 설치 없이 순수 Python:

```python
# mini_search.py — 가장 단순한 "검색" (IR의 출발점)
documents = [
    "고양이는 포유류이고 야옹 하고 운다",
    "강아지는 포유류이고 멍멍 하고 짖는다",
    "트랜스포머는 어텐션 기반의 신경망 구조다",
    "RAG는 검색과 생성을 결합한 기법이다",
]

def naive_search(query, docs):
    q_words = set(query.split())
    scored = []
    for i, doc in enumerate(docs):
        overlap = len(q_words & set(doc.split()))
        if overlap > 0:
            scored.append((overlap, i, doc))
    return sorted(scored, reverse=True)

for score, idx, doc in naive_search("RAG는 무엇인가", documents):
    print(f"[score={score}] {doc}")
```
```powershell
python mini_search.py
```

> [!warning] 이 코드의 한계가 곧 다음 단계의 동기
> 단어가 **정확히 겹쳐야만** 찾습니다("검색 증강 생성"으로 물으면 못 찾음). 이 한계가 → 더 나은 랭킹(**BM25**, Phase 2)과 의미 기반 검색(**임베딩**, [[Day-003]]·Phase 5)으로 이어집니다.
